# Automation Anywhere Collaborates with Cisco, NVIDIA, Okta, and OpenAI, Launching EnterpriseClaw to Run Next-Generation AI Agents Inside Enterprise Systems

## Ch04.015 Automation Anywhere Collaborates with Cisco, NVIDIA, Okta, and OpenAI, Launching EnterpriseClaw to Run Next-Generation AI Agents Inside Enterprise Systems

## 核心要点

- Automation Anywhere 联合 Cisco、NVIDIA、Okta 和 OpenAI 发布 EnterpriseClaw，将 claw-style AI agents 引入企业级运营
- EnterpriseClaw 支持跨云平台、桌面端、本地系统和安全企业网络的 AI agent 部署
- 四大技术合作： Cisco AI Defense（安全）、NVIDIA OpenShell + NIM（推理引擎）、Okta（身份认证）、OpenAI（GPT-5.5 模型） 

## 深度分析

### 1. Claw-Style AI Agent 的企业化挑战

Claw-style AI agent 是一类能够直接操作应用程序、浏览器、终端和本地系统的 AI 代理，其核心优势在于"在工作的发生地执行任务"。然而，大多数现有解决方案主要面向单机器或隔离云环境中的个人用户设计，缺乏企业级所需的：

- **跨团队协作能力**：企业运营涉及多个团队、工作流和权限层级
- **混合环境覆盖**：云平台 + 桌面端 + 本地系统 + 防火墙内基础设施
- **集中化治理**：访问控制、活动监控、审计追溯
- **安全合规**：金融、医疗等敏感数据的保护要求 

EnterpriseClaw 正是针对这些挑战而设计，试图将个人用户的"强力工具"转化为企业级的"可控生产力"。

### 2. 四方合作的战略意图

| 合作方 | 技术贡献 | 战略价值 |
|--------|----------|----------|
| **Cisco** | AI Defense + DefenseClaw | 为 AI agent 提供企业级安全防护层，应对新型攻击面 |
| **NVIDIA** | OpenShell + NIM microservices + Nemotron 模型 | 提供安全运行时和智能推理基础，支持本地部署 |
| **Okta** | 跨 agent 身份管理与认证控制 | 确保 AI agent 享有与人类同等的最小权限访问控制 |
| **OpenAI** | GPT-5.5 等前沿模型 | 为企业级工作流提供前沿 AI 能力 |

这一合作模式体现了"企业级 AI agent"需要的多维度能力整合：安全（Security）、智能（Intelligence）、身份（Identity）、算力（Infrastructure）。

### 3. Process Reasoning Engine (PRE) 与 Contextual Intelligence Graph

Automation Anywhere 的 PRE 和 Contextual Intelligence Graph 为 EnterpriseClaw 提供了额外的准确性保障。在企业环境中，仅依赖 LLM 本身可能导致幻觉和事实错误，而结合流程推理引擎和上下文知识图谱，可以显著提升业务关键自动化的可靠性。

### 4. 自主企业（Autonomous Enterprise）的愿景

Automation Anywhere CEO Mihir Shukla 的表述揭示了其核心愿景：**AI 需要在"工作实际发生的地方"开展工作**。这意味着 AI 不仅要回答问题，还要执行操作；不仅要处理单一应用，还要横跨多个系统和数据类型。

### 5. 市场影响与竞争格局

- **对 RPA 行业的启示**：传统 RPA 正在向"agentic RPA"演进，EnterpriseClaw 代表了 RPA 厂商与 AI agent 技术的深度融合
- **对安全行业的影响**：Cisco AI Defense 的介入表明 AI agent 安全将成为企业安全战略的新维度
- **对身份管理的影响**：Okta 提出"AI agent 必须拥有第一类身份（first-class identities）"，预示着身份管理的范畴将从人类扩展到 AI agent 

## 实践启示

### 企业采纳建议

1. **评估 Agent 化成熟度**：在部署 EnterpriseClaw 前，企业应评估自身在 API 集成、数据治理、工作流标准化等方面的准备程度
2. **安全先行**：充分利用 Cisco AI Defense 的防护能力，建立 AI agent 安全策略
3. **身份治理扩展**：将 Okta 的身份管理框架扩展至 AI agent，制定 agent 身份生命周期管理政策
4. **混合部署策略**：NVIDIA NIM microservices 支持本地部署，对于有数据主权要求的企业是关键选项

### 技术团队关注点

- EnterpriseClaw 被设计为跨多种 AI agent 框架的**可扩展平台**，这意味着企业可以部署内部开发的 agent 或第三方工具
- 预览版已可用，正式版预计年内发布，技术团队可以开始概念验证（POC）
- 与现有 Automation Anywhere 自动化资产的集成能力是迁移优势 

### 行业观察

- "AI agent 是否能安全工作"正在成为企业部署的核心问题，而非"AI agent 能否完成工作"
- Claw-style agent 与 RPA 的融合代表了自动化平台的下一个演进方向
- OpenAI GPT-5.5 的企业级支持表明前沿模型正在加速进入生产环境

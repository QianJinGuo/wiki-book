# Headless Software：Agent 时代软件界面何去何从

> a16z 播客讨论 headless 软件在 Agent 时代的本质：如果软件的"用户"从人变成 agent，界面（head）可能成为累赘，真正值钱的是底层的数据结构和业务逻辑。

## 核心观点

### Agent 能力三分法（Steven Sinofsky）

1. **查找（Look up）**：单纯找信息，技术门槛低
2. **执行操作（Execute）**：涉及身份、权限、付费席位——企业软件最头疼的问题
3. **分析（Analyze）**：跨系统试错，最适合 agent 但幻觉风险最大

这个三分法有助于厘清 agent 产品讨论中经常混淆的概念。

### 软件粘性的本质

Seema Amble 指出粘性来自：围绕人的使用习惯搭建的组织流程、合规法律层面的硬约束。Steven Sinofsky 补充：**收钱最有粘性**——一旦在收钱，客户想停下来都难。真正有粘性的东西不是某个漂亮功能，而是用户已养成的操作习惯和散落在流程各处的默契。

### SAP 的业务逻辑壁垒

真正值钱的不是数据存在哪个数据库里，而是封装在 SAP 里的业务逻辑。这套逻辑往往要花好几年实施，不是集成商效率低，而是它真的要贴合企业实际运作。创业公司常拿自己团队的规模去想象客户的复杂度，别人的系统看起来笨重，恰恰是因为它扛住了你没经历的规模和例外。

### 例外处理才是核心

- 非技术驱动的工作流是高价值领域——客户成功、合规、收益确认，这些流程比技术栈更难自动化
- "数据 + AI + 合规"三合一构成真正的护城河

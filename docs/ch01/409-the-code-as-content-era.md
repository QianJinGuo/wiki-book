# The Code-as-Content Era

> 文章核心论点：当 AI 把软件生产的摩擦降到趋近于零时，应用本身开始像模因（meme）一样被生产、消费、传播与遗忘——作者把这一阶段命名为 **Code-as-Content Era**。
> → [原文存档](https://raw.githubusercontent.com/QianJinGuo/wiki/main/raw/articles/the-code-as-content-era-20260606.md)

## 摘要

作者 dead.online 在 2026 年 6 月撰写的 Substack 短文，把当前 vibe coding 工具栈（Cursor / Claude / Codex 等）所催生的"周末造物—病毒传播—几天内被收购"循环，对比 2010 年代末 memecoin 借助 pump.fun 把代币变成互联网文化的同构过程。文章提出核心命题：code 已经跨过了"基础设施"的临界点，开始按"社交媒体"的节奏运作——**软件不再只是 Software-as-a-Service，而是 Software-as-an-Expression**。其商业含义是：未来十年定义行业的公司将围绕 "drops、taste、cultural timing" 组织，而不是 roadmaps 与 dashboards。

## 核心要点

- **同构论证**：code 像 crypto 一样，从"金融工具/基础设施"转为"互联网文化载体"——一旦 token 化（memecoin）/生成式化（vibe coding）变得摩擦为零，其行为模式就脱离了原始设计意图。
- **vibe coding 群体不是 founder 群体**：他们更像 SoundCloud 2013 时代的卧室制作人或 Tumblr 时代的 GIF 博主——生产—验证—复用的循环取代了 PMF / scale / churn 框架。
- **Meta 收购 Gizmo 是分水岭事件**：Gizmo 是"短视频式竖滑 + 每个滑卡是工作软件"的 micro-app 平台，2026 年 3 月被 Meta 整建制纳入 Superintelligence Labs，是首个被超大型科技公司收购的"vibe coding 时代原生"产品。
- **新组织范式**：drops、taste、cultural timing 成为竞争力；A&R 视角的 VC 评估范式（"这个 drop 落地吗？受众在哪儿？能否再来一次？"）正在取代传统的产品-市场匹配评估。
- **基础设施侧变化**：Replit、Wabi、Rooms.xyz、Sekai、Rork、Vibecode 等一批平台都把"prompt → app"作为核心抽象层，绕开传统 IDE 与源码仓库。

## 深度分析

### 一、code 与 meme 的同构临界点

文章最有洞察力的部分是把 vibe coding 时代的 code 与 2017-2021 memecoin 浪潮做同构对比。两者的演化逻辑共享三步：

1. **创造摩擦消失**：pump.fun 把"发币"压到一键；Cursor + Claude + Codex 把"写一个能跑的 app"压到几小时。
2. **行为模式脱离设计者意图**：比特币白皮书假设的是"点对点电子现金"，但当 frictionless issuance 成为主流时，比特币社区演化出 memecoin 文化——发币变成自我表达。code 同样如此，TypeScript 白皮书假设的是"工程化的大规模软件"，但当 prompt-to-app 变得零摩擦时，软件生产变成"自我表达与社交资本"。
3. **"基础设施"被"文化"覆盖**：原本用于建设可靠系统的工具，被用于制造、转售、消费、遗忘"数字玩具"。

文章作者把自己周末做的 FlightOrFight（一个"AI 投诉信生成器"）作为"个人样本"——周一发 LinkedIn，周三就被收购了。两年前"听起来像临床疯话"的事情，现在"只是稍微失控"。这正是临界点过后的典型叙事。

### 二、camera-era 类比

作者引用了"每个 app 都变成相机 app"的历史阶段——Instagram / Snapchat / Tinder 都把"对着世界"作为默认交互。背后驱动是硬件（手机摄像头）变得"够好、够便宜、够快"，于是硅谷围绕它重组（feed → 文化 → 审美经济）。文章认为：

> "We're in the middle of that kind of shift again. Only this time, it's code that is the new mass medium."

类比的关键含义是：硬件/工具的临界点会引发**整个产品形态与商业范式**的级联重排。当每个 app 都是 vibe coding app 时，feed 不再是社交关系流，而是软件对象流；经济价值不再由"用户停留时间"决定，而由"是否能被瞬间理解、转发、二创"决定。

### 三、Meta × Gizmo：超大型科技公司正式入场

2026 年 3 月 Business Insider 报道 Meta 雇佣 Gizmo 团队加入 Superintelligence Labs——这是首个被超大型科技公司收购的"vibe coding 原生"产品。Gizmo 的产品形态是"竖滑 feed + 每个卡是工作软件"，即"短视频形态 + 可执行内容"。作者认为：

> "The companies that matter in the Code-as-Content Era won't look like the ones that mattered in the last. They won't be organized around roadmaps and dashboards. They'll be organized around drops. Around taste. Around cultural timing."

收购本身验证了 hype，但更重要的是它揭示了 Meta 的判断：**下一个十亿级软件消费入口不在"应用市场"，而在"feed"**。这条判断与 TikTok 在 2020 之后的演进路径同构——短视频 feed 击穿了"用户主动搜索—下载—打开"的传统分发链。

### 四、行为层而非技术层

文章反复强调"what's interesting isn't the technology. It's the behavior."。技术栈本身（Cursor / Claude / Codex）只是 enablement，真正决定输赢的是：

- **生产节律**：从季度发布到 drops，从 OKR 到"本周做了什么有意思的"。
- **审美判断**：taste 取代 feature completeness。
- **文化时机**：窗口期以"小时"计，错过的代价是整轮被遗忘。
- **可组合性**：remix 别人作品的能力，比原创能力更被市场奖励。

这与 [17 种 agent 架构演进](/ch04-507-17种agent架构演进-控制流设计的完整演化史//) 中关于"agent 必须能被快速 prompt 改写与组合"的判断一脉相承——vibe coding 工具只是 agentic 编程的 UI 表现，背后的范式是"组合性优先于完整性"。

### 五、对软件工程的冲击

传统软件工程围绕"长期可维护性"组织——抽象、类型、测试、文档、code review。vibe coding 时代的"软件"恰恰是反方向的：它不需要长期可维护，因为它的预期寿命就是几天到几周。

这给工程教育、企业研发组织、知识产权法律都带来新问题：

- **教育层面**：是否还要教 SOLID、design patterns、formal methods？
- **组织层面**：PM 与设计师的角色如何重塑？是否需要"软件策展人"？
- **法律层面**：被收购的 FlightOrFight 价值是 IP 还是流量？代码本身几乎没有价值，价值在"被文化看见的瞬间"。

## 实践启示

- **把软件当作可丢弃物**：在 vibe coding 时代，"代码资产"的半衰期是周级。任何要求 3-5 年 ROI 的项目都需要重新评估其与该范式的兼容性。
- **A&R 视角的评估**：评估 vibe coding 项目时，关注"drop 的感官冲击力 / 受众是否在 5 秒内理解 / 团队能否稳定复制这种产出"——这三条比 burn rate、retention、ARR 更能预测文化价值。
- **feed 分发优先于 app store**：押注"app-as-content"分发形态（feed、推荐流、可执行 card），而不是依赖用户主动下载。
- **保留传统工程能力作为基线**：vibe coding 适合做"低风险高速度"的实验，但生产系统、关键基础设施、合规相关代码仍需传统工程纪律。两者不互斥。
- **关注平台与发布节奏**：Wabi、Rooms.xyz、Sekai、Replit 等平台是新基础设施——它们定义了你的 app 如何被发现、传播与货币化。

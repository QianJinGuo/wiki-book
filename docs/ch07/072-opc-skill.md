# 京东健康 OPC 团队产品全流程 Skill 探索

## 摘要

京东健康 OPC（One Person Company）团队在无专职产品角色的模式下，探索将产品全流程 Skill 化——从需求判断、方案设计到交付验证，构建完整的 Agent Skill 体系。

## 背景

在 OPC 团队中，没有专职产品角色。团队不仅要关注"把需求做完"，还必须回答更前置的问题：这个问题是否真的值得做？证据来自用户、数据还是直觉？

## 核心设计

OPC 模式下的 Skill 全流程覆盖：
- **需求判断 Skill** — 评估需求价值，区分用户驱动、数据驱动还是个人判断
- **方案设计 Skill** — 将需求转化为可执行的技术方案
- **交付验证 Skill** — 验证交付物是否满足原始需求

## 实践启示

- OPC 模式下 Skill 是团队能力的载体，而非替代产品角色
- Skill 的边界需要与团队自治权匹配
- 全流程 Skill 化是 Agent 团队协作的工程基础

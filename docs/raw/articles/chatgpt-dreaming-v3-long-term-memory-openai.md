# ChatGPT 的"失忆症"终于被治好了！Dreaming V3 让大模型拥有长期记忆

> 作者：大石（51CTO技术栈） · 发布：2026-06-05

> 北京时间 6 月 5 日凌晨，OpenAI 正式推出全新的记忆架构 **Dreaming V3**。ChatGPT 终于有了真正意义上的长期记忆——一次架构级重构，把大模型从只有短期记忆的对话工具，往有长期记忆的个人助手方向推了一大步。

## 一、记忆进化三步走

### 1.1 2024-04：保存记忆（V1）— 手写便签时代

**逻辑**：你明确告诉它记住什么，它写进一张便签，下次对话翻出来用。

**痛点**：
- "你没说的，它就直接遗忘"——只做了几条笔记，忘了所有未写下内容
- 便签会过期（OpenAI 自评）

### 1.2 2025-04：Dreaming V0 — 自动提取

**核心变化**：ChatGPT 开始**自动从聊天历史里提取上下文**，不再依赖手动下指令。

**示例**：聊了三个月的相机配置 → 自动抓住"喜欢摄影"这条信息。

**局限**：
- V0 本质上是给旧架构打补丁
- 提取能力有限
- 时效性问题仍在（2025 Dreaming V0 时效正确率仅 52.2%）

### 1.3 2026-06-05：Dreaming V3 — 架构级重构

**核心架构**：
- **空闲时启动后台异步进程**
- 进程用**向量相似度**对历史对话**聚类**
- 碎片化聊天**压缩、去重**
- 通过**分层摘要**归档到**长期偏好数据库**
- GPT 不用每次回忆聊天记录，而是用"不断更新的个人档案"

## 二、时效性修正 — 记忆系统最隐蔽的坑

### 2.1 经典失败案例

> 想象你正在上海出差，深夜让 ChatGPT 找还在营业的外卖。它推荐了定西路的葱油拌面和云南南路的生煎包。结束出差回到北京一周后再问同一句话：**旧系统给的答案，还是上海的葱油拌面和生煎包**。

旧系统错把"我去了上海"理解为"我在上海"，永远留在上海。

### 2.2 V3 时效性自动感知

**Dreaming V3 在后台感知时间流逝**，把记忆状态从"正在进行"自动切换为"已经结束"：
- 上海出差 → 回北京
- 再问外卖 → 起点切回北京望京
- 精准给出楼下的烧烤和附近川菜馆美团链接

### 2.3 量化提升（关键数据）

| 架构 | 时效正确率 | 提升 |
|------|------------|------|
| **2024 保存记忆** | 9.4% | — |
| **2025 Dreaming V0** | 52.2% | 翻 5.5 倍但仍有一半概率出错 |
| **2026 Dreaming V3** | **75.1%** | 从几乎不可用 → 可被信任的水平 |

> "2024 年的保存记忆系统，时效正确率只有 9.4%。基本等于瞎猜。"

## 三、偏好跟随 — 记得比你还清楚

### 3.1 实际案例：新加坡旅行

**用户三个长期偏好**（分散在过去一整年零散对话里）：
1. 喜欢野生动物摄影
2. 对空调温度要求特别高
3. 在拥挤的酒吧里偏爱安静的晚餐

**旧系统输出**：万能攻略（滨海湾看天际线 / 圣淘沙玩海滩 / 小贩中心吃叻沙）—— "**像一本给游客看的通用手册**"。

**Dreaming V3 输出**：
- 圣淘沙直接砍掉（跟摄影不搭）
- 早上的活动排在**鸟类天堂和植物园**
- 中午全部塞进**带冷气的云雾森林**（空调偏好）
- 晚餐一律换成**能预订座位的餐厅**（跳过小桌小贩中心）

每一步都没让用户重新交代偏好，**每一步都精准对上历史**。

### 3.2 量化提升

| 架构 | 偏好跟随成功率 | 提升 |
|------|----------------|------|
| 旧系统 | 31.4% | — |
| **Dreaming V3** | **71.3%** | **翻一倍多** |

> "翻译成用户体验就是你**不用每次开新对话都自我介绍一遍**。"

## 四、算力砍 5 倍 — 长期记忆的工程死结

### 4.1 计算灾难

要记住跨越一整年的零散偏好 → 每次开新对话都要在后台**重新检索庞大的历史向量数据库** → 算力开销巨大。

**这是为什么长期记忆一直是 Plus 和 Pro 付费用户的专属**——不是 OpenAI 不想给免费用户，是撑不住。

### 4.2 V3 工程突破

**Dreaming V3 真正厉害的地方，在于它解开了这个工程死结**——把维持这套记忆系统所需的计算量**硬生生砍了约 5 倍**，**降幅高达 80%**。

**结果**：从只有付费用户才配拥有的功能 → **推向所有用户的标配**。**长期记忆从此从奢侈品变成了基础设施**。

## 五、新功能：记忆摘要（用户可控索引表）

**记忆摘要**功能：用户可以像翻数据库的索引表一样，看到 ChatGPT 到底记住了哪些信息。

**操作**：
- **添加** 记忆
- **修改** 已有记忆
- **删除** 错误记忆

**意义**：不用在几百条聊天记录里翻找"它哪次记错了什么"，一个页面就能完成对自己知识库的编辑。**对隐私和数据控制的意义，不下于记忆本身的技术突破**。

## 六、战略意义 — 有状态 AI 才是数字员工

> "去年一整年 AI 圈都在卷推理模型，比谁更会想。但再聪明的模型，如果每次对话都从零开始，本质上也只是一个**随时准备跑路的临时工**。"

**核心断言**：

- **无状态 AI 永远承接不了高价值的复杂工作流**
- 没人会把核心资产托付给一个**随时失忆的助理**
- 就像没人会把公司的重要客户交给一个**永远记不住上次聊了什么的销售**

> "Dreaming V3 带来的**有状态能力，是大模型从聊天玩具走向数字员工的入场券**。"

**核心金句**：「只有具备了长期、可伸缩、能自我修正的记忆基础，Agent 才有资格被真正委以重任。」

## 七、可用性

- **现在**：Plus 和 Pro 用户已开放
- **未来几周**：推送到更多国家，以及免费用户、Go 用户

> "对于开发者来说，这次更新无疑更让大家期待 GPT 5.6 会带来什么令人眼前一亮的新功能。"

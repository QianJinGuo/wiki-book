# OpenAI Codex 三件套：6 个职位插件 + Sites 网站生成 + Annotations 精准批注
> 整理自微信公众号对 OpenAI Codex 重大更新的报道
> 原文：https://mp.weixin.qq.com/s/5xYepfxlB1423SuW7h6rIQ
> 参考：https://openai.com/index/codex-for-every-role-tool-workflow/
> 官方文档：https://developers.openai.com/codex/sites

## 一句话定位
**Codex 三件套** = (1) **6 个面向职位的专属插件**（含 62 款主流应用 + 110 项技能） + (2) **Sites 网站生成 + 托管** + (3) **Annotations 精准批注**。Codex 正从"软件开发工具"演变为**覆盖所有职位的工作台**，且 Codex 团队已官宣**未来几周并入 ChatGPT** 变成一个产品。

## 关键数据
- **每周 500 万+ 人在用 Codex**
- **非开发者用户已占 20%**，增速超开发者群体 **3 倍**
- 未来几周 Codex 合并进 ChatGPT
- OpenAI 内部数据显示切换到 Codex 后 Claude Code 使用大幅下降

## 6 个职位插件（62 应用 + 110 技能）

| # | 插件 | 面向 | 核心能力 | 接入应用 |
|---|---|---|---|---|
| 1 | **数据分析** | 分析师 / 业务团队 | 探索产品和业务数据、解释关键指标变化、生成报告/仪表盘 | Snowflake / Databricks Genie / Hex / Tableau |
| 2 | **创意生产** | 市场 / 创意团队 | 创意简报 → 审阅素材、生成展示广告变体、产品生活场景图/电商图集 | Figma / Canva / Shutterstock / Picsart / Fal |
| 3 | **销售** | 销售团队 | 高优先级客户/商机信号、客户会议材料、跟进、客户记录更新、成单计划、风险交易审查 | Salesforce / HubSpot / Slack / Outreach / Clay / Rox / Actively |
| 4 | **产品设计** | 产品设计师 | 早期想法 → 团队可审阅原型、探索方向、用户流程梳理、真实 URL 生成原型、静态截图 → 可交互界面（成果可延续到 Figma/Canva） | Figma / Canva |
| 5 | **公开股权投资** | 投资人 | 财报解读、公司比较、信号追踪、判断投资逻辑加强/弱化 | Moody's / Daloopa / Datasite / FactSet / LSEG / S&P / PitchBook / Hebbia |
| 6 | **投资银行** | 投行从业者 | 路演材料准备、可比公司和交易分析、尽调结论 → 建议报告 | （可信数据源） |

### 后续将推出
企业财务 / 私募股权投资 / 市场策略 / 战略咨询 / 法律 —— 长期目标：**开放生态让合作方在 Codex/ChatGPT 内创建和部署自己的插件**。

## Sites：Codex 生成可交互网站 + 直接托管
- **当前状态**：Business / Enterprise 预览
- **核心能力**：Codex 生成交互式网站 + 应用 + **直接托管**（带 URL 可分享）
- **典型场景**：
  - 客户评审的交互网页（产品更新 + 待解决问题 + 使用趋势 + 下一步行动）
  - 财务模型 → **情景规划器**（管理层直接比较不同假设）
  - 产品发布材料 → **活跃信息中心**（最新信息 + 里程碑 + 负责人 + 决策记录 + Codex 同步更新）
- **应用类别**：营收预测规划器 / 活动运营仪表盘 / 产品发布信息中心
- **不是静态页面**：可追踪重大项目进展 / 客服人员操作引导 / 团队创意简报存放库
- **合作伙伴生态**：Wix / Base44 / Replit / Lovable / Figma / Webflow / Emergent

## Annotations：精准批注，不动其余
- **核心能力**：精准指向内容的某一个具体部分，告诉 Codex 需要改什么
- **覆盖范围**：代码 / Markdown / Codex 生成的网站（已有）→ 扩展到 **文档 / 表格 / PPT**
- **示例**：
  - 选中网站上的**导航栏** → 让 Codex 修改字体
  - 高亮**投资论文里的某个论断** → 让 Codex 说明来源
  - 标记 **PPT 上的图表** → 让 Codex 给出更清晰标注
- **关键设计**：**只更新被选中部分，已满意内容不动** —— 让 Codex 在**初稿之后依然有价值**

## 范式判断

### 1. Codex 从开发工具 → 工作台
- 起家：软件开发
- 现在：覆盖 6 大职位（分析师/创意/销售/产品/投资/投行）
- 后续：企业财务/PE/市场策略/战略咨询/法律

### 2. 职位插件是 Skill 2.0 的具象化
- 不是 1 个 Skill 干 1 件事
- 是 1 个插件 = N 个原子 Skill × M 个应用集成
- 符合 [[entities/meta-skill|Meta Skill]] 抽象层

### 3. "会写代码" → "会组织工具"
> "以前这类 Agent 会成为每个人打工人的必备的工具，就像旧时代的 office 一样，简历上要写熟练使用 Codex, CC 等。"

### 4. Sites 挑战独立建站工具（Wix/Replit/Lovable）
- 不是替代，而是**生态合作**：OpenAI 与 Wix/Base44/Replit/Lovable/Figma/Webflow/Emergent 共同建 Sites 生态

### 5. Annotations 是"局部编辑"的范式
- 与 [[entities/embabel|Embabel]] 的"完全可解释可审计"思路呼应
- 局部修改 vs 全局重生成 = 节省用户心智 + 减少幻觉累积

## 启示
1. **职位插件是 Agent 商业化的最佳路径** —— 6 个职位 = 6 个 GTM
2. **集成应用数量是护城河** —— 62 款应用是 OpenAI 商务 BD 的成果
3. **Sites 把"输出物"标准化** —— 仪表盘/规划器/信息中心 = 通用结构
4. **Annotations 解决"反复调整"痛点** —— AI 在草稿阶段最有用
5. **Codex 并入 ChatGPT = 边界模糊** —— Agent 能力 + Chat 界面 = 一体化产品

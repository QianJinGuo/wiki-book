# 大模型不是马，是大脑

## 先扔掉"大模型是马，Harness是马鞍"比喻

最近 Harness Engineering 很火，随之而来的一个比喻也开始流传，即"大模型是马，Harness 是马鞍"。这个比喻并不准确。马的特点是有独立意志，需要驯服、约束、引导。但 AI 系统并不是这样构建的。大模型更像大脑，而 Agent 更像身体。

## AI 的发展：大脑先于身体的超速进化

从 2012 年 AlexNet 到今天，短短十几年，AI 已经完成了从识别、理解、生成到多模态、代码生成、工具调用的一整轮跨越。但问题在于：这些器官虽然存在，却远未形成成熟、稳定、协调的身体系统。

## 当前 Agent 系统的核心问题：身体没有长好

四个主要问题：

1. **感官系统不成熟**：PDF 解析目录错位、网页抓取噪声大、图像识别遗漏关键元素——有输入能力但输入质量不稳定
2. **运动系统不协调**：API 参数错误、UI 点击偏移、执行环境不一致、操作后缺少反馈闭环——不是"不会动"而是"动作不协调"
3. **资源调度系统粗糙**：上下文窗口管理原始——信息给少了推理断裂，信息给多了提示词过载
4. **自主神经系统缺失**：错误恢复不完善、重试靠人工规则堆叠、上下文清理缺乏策略、降级与兜底不系统、健康检查不完整——缺少后台自动维持能力

## Harness 的真正角色：不是马鞍，而是 ICU

Harness 不是缰绳，而是监护仪——生命周期监测、资源维持、信号调控、故障抢救。不是在"控制大脑"，而是在维持身体的基本生命体征。先让系统稳定活着，后续才谈得上持续成长。

## AI 当前的状态不是失败，是早期常态

模型能力已经强到超出预期，系统工程仍然脆弱易碎——两个判断不冲突。当前最重要的不是假装体系已经成熟，而是承认现实。Harness 就是轮椅、拐杖、监护仪的集合。

## vivoPPT 开发过程：最佳实践是收敛出来的

从"直接生成大纲+提供很多模板"→"固定模板+内容优先"→"先生成 DSL 中间层"→"大模型微调+富文本上下文解析"。四条流程纪律：
- 先研究，再写作
- 先大纲，再页面
- 先任务化，再并行化
- 先可编辑，再可交付

## 写在最后

未来不会再讨论"要不要用 AI"。真正的变化不会发生在模型参数扩大时，而是发生在我们开始真正理解：什么时候让它思考，什么时候让它行动，什么时候借助工具，什么时候交给流程，什么时候让人介入。

我们现在正处在一个"还不会用工具"的时代。工具已经足够强大，但使用方式还没有完全形成。人们不仅是在使用工具，也在参与定义工具未来的正确使用方式。

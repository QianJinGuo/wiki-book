# AWS 一周综述：AWS Transform 上线一周年、AWS 云端 Claude Platform、EC2 M3 Ultra Mac 实例等（2026 年 5 月 18 日）

## AWS 一周综述：AWS Transform 上线一周年、AWS 云端 Claude Platform、EC2 M3 Ultra Mac 实例等（2026 年 5 月 18 日）

![](https://d2908q01vomqb2.cloudfront.net/da4b9237bacccdf19c0760cab7aec4a8359010b0/2026/05/18/AWS-Transform-1-year.jpg)就在一年前，我们推出了适用于 [.NET](https://aws.amazon.com/blogs/aws/aws-transform-for-net-the-first-agentic-ai-service-for-modernizing-net-applications-at-scale/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)、[大型机和 VMware 工作负载](https://aws.amazon.com/blogs/aws/accelerate-the-modernization-of-mainframe-and-vmware-workloads-with-aws-transform/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)的 [AWS Transform](https://aws.amazon.com/transform/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)，这是首款专为大规模实现企业应用程序现代化而构建的代理式人工智能服务。在 2025 年 re:Invent 大会上，我们推出了 [AWS Transform 自定义](https://aws.amazon.com/transform/custom?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)，使组织能够通过 AWS 托管转换和自定义转换功能对代码进行大规模现代化改造和转换。您可以通过即时可用或可根据组织的特定需求定制的转换方案来升级语言版本、迁移框架、优化性能并分析代码。我们还推出了[面向大型机的全栈 Windows 现代化功能](https://aws.amazon.com/blogs/aws/aws-transform-announces-full-stack-windows-modernization-capabilities/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)、[重新构想功能和自动化测试功能](https://aws.amazon.com/blogs/aws/aws-transform-for-mainframe-introduces-reimagine-capabilities-and-automated-testing-functionality/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)。

上线 12 个月以来，已有数千家客户借助 AWS Transform 完成数十万服务器迁移，累计节省超 160 万小时人力，处理超 45 亿行代码。恰逢一周年，AWS Transform 代理现已在 [Kiro、Claude、Cursor 和 Codex](https://aws.amazon.com/about-aws/whats-new/2026/04/aws-transform-developer-tools/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el) 平台上线，其中包括用于构建自定义转换代理的[代理生成器工具包 Kiro 能力包](https://aws.amazon.com/about-aws/whats-new/2026/05/aws-transform-agent-builder-toolkit/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)。

要了解这 12 个月发生的事情、我们学到的四点经验及它们对产品路线图演变的影响，请访问[一周年博客文章](https://aws.amazon.com/blogs/migration-and-modernization/aws-transform-one-year-milestone/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)。

**上周发布的内容**  
以下是上周引起我注意的发布内容：

*   [AWS 云端 Claude Platform 正式发布](https://aws.amazon.com/about-aws/whats-new/2026/05/claude-platform-aws/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el) — 您可以直接通过现有 AWS 账户体验 Anthropic 原生 Claude 平台，包括 API、控制台和抢先体验测试版功能，无需单独管理账号、账单或跟踪数据。AWS 云端 Claude Platform 由 Anthropic 运营，客户数据在 AWS 安全边界之外进行处理。要了解更多信息，请访问[深入探讨博客文章](https://aws.amazon.com/blogs/machine-learning/introducing-claude-platform-on-aws-anthropics-native-platform-through-your-aws-account/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)。
*   [Amazon EC2 M3 Ultra Mac 实例](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-ec2-m3-ultra-mac-instances-generally-available/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)：该实例搭载基于 Apple M3 Ultra Mac Studio 计算机构建，该计算机配备 28 核 CPU、60 核 GPU、32 核神经网络引擎和 256GB 统一内存。与 EC2 M4 Max Mac 实例相比，M3 Ultra Mac 实例的统一内存容量翻倍、CPU 核心数提升 1.75 倍、GPU 核心数提升 1.5 倍、神经网络引擎核心数翻倍，使 Apple 开发人员能够并行运行更多 Xcode 模拟器并加快设备端机器学习工作流，从而缩短产品上市周期。
*   [由 AWS Graviton 提供支持的 Amazon Redshift RG 实例](https://aws.amazon.com/blogs/aws/amazon-redshift-introduces-aws-graviton-based-rg-instances-with-an-integrated-data-lake-query-engine/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)：该实例可提供更好的性能，运行数据仓库和数据湖工作负载的速度高达上一代 RA3 实例的 2.4 倍，同时单 vCPU 的价格降低 30%。RG 实例内置 Redshift 定制的向量化数据湖查询引擎，可在集群节点直接处理 Apache Iceberg 和 Parquet 数据。
*   [Amazon Bedrock 高级提示优化](https://aws.amazon.com/blogs/aws/amazon-bedrock-introduces-new-advanced-prompt-optimization-and-migration-tool/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)：可针对 Bedrock 上的任何模型优化提示，并同时在最多 5 款模型中对比原始提示和优化后提示的效果。如果您要迁移到新模型或者只是想在当前模型上获得更好的性能，也可以使用此功能。
*   [AWS Security Agent 完整存储库代码扫描（预览版）](https://aws.amazon.com/about-aws/whats-new/2026/05/aws-security-agent-full-repository-code-review/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)：您可以使用 AWS Security Agent 中的新功能对整个代码库进行深入的上下文感知安全分析。发现漏洞后，扫描器会生成代码修复方案，精准定位确切文件和代码行，使团队能够更快修复安全漏洞。预览期间，现有 AWS Security Agent 客户可以免费使用该功能。
*   [通过 Oracle 云基础设施实现 AWS 互连 — 多云连接（预览版）](https://aws.amazon.com/about-aws/whats-new/2026/05/aws-announces-AWS-interconnect-multicloud-oci-preview/?trk=d8ec3b19-0f37-4f8c-8c12-189f913e205c&sc_channel=el)：借助 AWS 互连 — 多云连接，您可以快速预置与其他云提供商的弹性、可扩展私有连接。OCI 是最新采用支持 AWS 互连的开放规范的 CSP。这使 AWS 能够在 OCI（预览版）、Google Cloud（正式版）和 Microsoft Azure（2026 年晚些时候推出）上为我们的客户提供一致、简单的体验。

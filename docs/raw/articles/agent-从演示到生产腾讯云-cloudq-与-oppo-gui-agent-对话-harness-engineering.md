# Agent 从演示到生产：腾讯云 CloudQ 与 OPPO GUI Agent 对话 Harness Engineering

直播推荐  


让 Agent 从“能用”到“敢用”，一场没有演讲、只有硬核实战对谈的圆桌直播即将开启。过去一年，AI Agent 的能力突飞猛进——能写代码、能调 API、能操作手机、能管理云资源。但当企业真正想把 Agent 推向生产环境时，最大的拦路虎不是“能不能做”，而是 “敢不敢用”。

Agent 调用错了工具怎么办？越权操作了敏感数据有没有审计？多 Agent 一起跑的时候会不会互相打架？用户在手机上点单，弹窗广告把 Agent 带偏了怎么回滚？这些问题，在 Demo 里无所谓，在生产里就是事故。

### 

**01  
**

**一场“跨场景”的巅峰对谈**

本次直播聚焦 Harness Engineering（Agent 执行与控制工程）——一个专门研究如何给 Agent 装上“刹车、方向盘和仪表盘”的技术领域。

我们邀请了三位来自截然不同场景的一线专家：

  * 黄佳将担任本次直播的主持人。他是新加坡科技研究局 AI 研究员、《Designing AI Agents》作者，也是拥有 20 余年跨行业实战经验的技术老兵。他将以“学术视角 + 工程落地”的双重洞察，引导两位嘉宾碰撞出真正可复用的经验。

  * 姚斌斌：腾讯云 CloudQ 产品技术负责人。CloudQ 是国内首款将架构治理与 ChatOps 结合的对话式云运维智能体，口号是“Just Q IT!”。它通过 STS 最小权限、CAM 身份体系，实现“身份-资源-对话”的安全绑定，支持腾讯云、阿里云、AWS、Azure 等主流云厂商的统一纳管。

  * 瞿祥谋：OPPO 研究院拓扑 Lab 高级算法工程师。他专注于移动端 GUI Agent，让 Agent 在真实手机上完成点单、打车、预约等跨 App 复杂任务。面对弹窗广告、动态页面、网络波动等“野生环境”，他采用多智能体协同 + 在线强化学习 + 端云混合状态感知来保证执行可控。

一个管云，一个管端。两者对“可控”的要求截然不同，但底层逻辑惊人相似。这是他们首次同台，直面 Harness Engineering 的核心难题。

  * OPPO GUI Agent 揭秘移动端任务执行、多智能体协同和安全防护的落地细节

**

**讨论话题**

## 开篇：你的编程助手，最近「活」过来了吗？

你有没有这种感觉——

用 AI 编程助手的时候，它要么在疯狂输出，你盯着屏幕等结果；要么就在那儿「思考中」，你不知道它是卡住了还是在认真干活。反正就是—— 看不见摸不着，心里没底 。

我就经常这样。给 Codex 发了一个任务，看它开始跑起来，然后就开始焦虑：它还在跑吗？跑完了吗？是不是该切回去看一眼了？

切回去，没有结果；切出来，还是焦虑。

后来我发现，Codex 新出了个 Pets（宠物） 功能，可以让 AI 编程助手「活」过来——它会变成一只小宠物，一直悬浮在你的屏幕上，实时告诉你它在干什么、跑到哪一步了。

你刷微博的时候，它在角落陪你；你写文档的时候，它在角落陪你；你发呆摸鱼的时候，它还在角落陪你。

怎么说呢，用了之后感觉完全不一样了——以前是「不知道 AI 在干嘛」，现在是「有个小跟班一直在跟我汇报进度」。

这种感觉，懂的都懂。

## 01｜Codex Pets 是什么？

先给还没接触过这个功能的朋友简单介绍一下。

Codex Pets 是 OpenAI 在 2026 年 5 月给 Codex 桌面版加的一个新功能。它的本质是一个 Live Activities 风格的覆盖层（Overlay） ——就是那种始终置顶的悬浮小窗口。

它能显示什么？
说明

当前线程
正在处理哪个任务

运行状态
正在跑 / 等待你的输入 / 可以审查结果

进度提示
当前在做哪一步的简短描述

最关键的是，这个悬浮窗口 始终显示在前台 。不管你切换到什么应用，它都不会消失，只是安安静静地待在屏幕角落。

而且你还可以用鼠标逗它——点一下会跳跃，拖一下会跟着动，不是那种死板的静态图标，是真的有互动感的。

OpenAI 的工程师 thsottiaux 说了一句特别贴切的话：「让上下文跟着我在多任务间切换，其实提高了我的效率。

这话说到我心坎里了。

## 02｜3步开启你的第一只宠物

说了一堆，怎么把它开起来？非常简单，两种方式任你选。

方式一：设置页面开启

打开 Settings → Appearance → Pets ，里面可以直接唤醒或收起宠物。

方式二：快捷键开启（最推荐）

系统
快捷键

macOS
Cmd + K

Windows
Ctrl + K

打开命令面板后，搜索  Wake Pet  （唤醒宠物）或  Tuck Away Pet  （收起宠物），回车就行。

我平时最常用的是快捷键，因为手不用离开键盘，顺手得不行。

开启之后，你的屏幕上就会出现第一只宠物——默认是一只叫 Codex 的蓝色小机器人。

## 04｜让宠物「跟」你走：Overlay 使用技巧

开启宠物之后，真正好用的部分是 Overlay（悬浮覆盖层） 。

### 三种状态，一目了然

宠物会根据 Codex 的当前状态切换形象和动作：

状态
含义
宠物的表现

Running
任务执行中
宠物会做「工作」动画

Waiting for input
等待你的输入
宠物会做「等待」动画，有时会打哈欠

Ready for review
完成，等待你审查
宠物会做「提醒」动画，引起你的注意

这意味着，哪怕 Codex 在后台跑着一个长达 10 分钟的重构任务，你也能通过角落里那只小宠物的状态，随时知道事情进展到了哪一步。

不用切窗口。不用猜。不用焦虑。

### 拖动定位

宠物默认出现在屏幕的某个角落，但你可以 直接用鼠标拖到任何你觉得舒服的位置 。我一般把它放在右下角，既不挡代码，又能随时瞥一眼。

### 点击跳转

如果你突然想回到 Codex 主界面查看详情，直接 点击宠物 就行，它会立刻帮你跳回去。这个设计真的很贴心，省去了在 Dock 栏或任务栏里找窗口的麻烦。

### 逗它玩

这个不算技巧，但真的很好玩——用鼠标点宠物，它会跳跃、转圈、做各种反应。写代码写烦了戳两下，瞬间解压。

## 05｜孵化专属宠物：自定义方法详解

用了一段时间的内置宠物，你可能会想：能不能有一个 完全属于我自己的宠物 ？

答案是： 可以 。而且过程比你想象的简单得多。

Codex 提供了一个叫 hatch-pet 的技能（skill），可以根据你的项目特点，「孵化」出一只独一无二的宠物。

### 完整流程

第一步：安装 hatch-pet 技能

首先安装 hatch-pet 技能。在 Codex 中进入插件的技能设置，搜索「pet」，可以傻瓜式安装：

第二步：重新加载技能

装完之后需要让 Codex 重新加载一下技能列表。打开命令面板（  Cmd+K  或  Ctrl+K  ），搜索并选择：

Force Reload Skills

第三步：创建你的专属宠物

安装完成后，在 Codex 中输入  /hatch-pet  命令。

最简单的方式是直接用文字描述你想要生成的宠物形象，比如我说：做一个赛博风格的道士。

AI会自行调用hatch-pet 技能开始生成小像素风格的宠物文件。

几分钟后AI生成了基础参考图，然后Codex会询问你是否允许子代理（Subagents），因为 hatch-pet 技能要求用 Subagents 来并行生成，直接回复允许子代理即可，接下来就是漫长的等待了，这一步甚至需要等待一个多小时。哈哈哈哈。

Codex 会分析你最近的项目特点，然后生成一只专属宠物。宠物的形象、性格、台词都会和你的编程风格有关。

### 一个彩蛋：语言孵化机制

更绝的是，hatch-pet 会根据你 最常用的编程语言 来孵化对应形象的宠物。

如果你连续使用...
可能会孵化出...

Rust
一只螃蟹 🦀

Python
一条蛇 🐍

其他语言
对应语言特色的形象

这种细节，真的能让人感受到开发团队的用心。它不只是「生成一个宠物」，而是在说：「我看见了你的工作习惯，这是专属于你的东西。

## 06｜安装社区宠物：Petdex 画廊挖宝

除了 8 只内置宠物和自定义宠物，官方还提供了一个叫 Petdex 的社区宠物画廊平台，里面已经有好几百款开源宠物，可以免费下载使用！

### 进入 Petdex 画廊

平台地址：https://petdex.crafter.run/zh

进入画廊后，你会看到宠物按照 角色、生物、物体 等分类和各种标签排列，支持筛选和搜索。还可以按照 喜欢数、安装量 排序，非常方便。

### 预览宠物动画

点击想要的宠物进入详情页，可以预览宠物的 所有动画状态 ，包括：

- • 待机

- • 奔跑

- • 跳跃

- • 挥手

- • 失败

- • ……总共
  9 种
  ，每种对应 Codex 不同的工作状态。

这个预览功能真的很重要，因为你应该不想装了一只宠物结果发现动画很丑对吧……

### 一键安装社区宠物

安装方法超级简单，打开终端，输入一行命令搞定：

npx petdex install doraemon

本质上就是把宠物的资源文件下载到  ~/.codex/pets/ doraemon /  目录下。

### 激活安装的宠物

安装完成后，回到 Codex 设置的 外观页面 ，下拉到底，就能在「 自定义宠物 」区域看到新装的宠物了，选择即可。

效果不错吧！想象一下别人不小心看到了你的电脑屏幕，震惊中喊出那声：「你干嘛，哎哟！」——肯定会给你枯燥的 Vibe Coding 工作带来一点乐趣和激情~

如果你用 hatch-pet 创建了一只特别有个性的宠物，不妨考虑 提交到社区 ，让更多人用上你的创作。

你有没有过这种崩溃时刻？

让AI帮你写一份周报，它给你写了一篇800字的小作文。你说"精简一点"，它删了一半。你说"再正式一点"，它又变成了公司公文。

同一件事，你跟AI来来往往交代了五六轮，终于满意了。第二天换个对话窗口，一切从头再来。

「你不是AI吗？你记性怎么这么差？」

这不是AI笨。这是你的AI缺少一份风格说明书。

刚好，Google开源了一个叫 DESIGN.md 的项目，GitHub上已经17,000多颗星了。虽然它最初是给程序员用的，但这个思路——给AI写一份"偏好说明书"——值得每个用AI的职场人学会。

## Google DESIGN.md 到底是什么？

Google做了一件事：定义了一个叫 DESIGN.md 的文件格式。开发者只要在自己的项目里放上这个文件，AI编程助手（比如 Claude Code、Cursor）就能自动理解这个项目的视觉风格——颜色、字体、间距、交互方式——然后按照你的设计规范来写代码。

相当于你跟AI说："别猜了，我的喜好全写在这个文件里了，照着这个来。"

打个比方：你去剪头发，跟理发师说"剪短一点"。短多少？两边推还是上面修？你俩的理解可能完全不一样。但如果你给他看一张照片说"就按这个剪"，误差立刻归零。DESIGN.md 就是那张照片。

## 这个思路可以迁移到日常AI使用

### 场景一：AI帮你写作
- 开头用问题切入，不要用"在这个信息爆炸的时代"
- 段落之间多用空行，不要写大段连续文字
- 每个核心观点后面跟一个案例或数据
- 结尾给一个明确的行动建议

### 场景二：AI帮你做PPT
- 每页不要超过15个字
- 标题用大号加粗，正文用浅色小字
- 配色：主色 #2B5B84，强调色 #E8733A
- 每页最多3个要点

### 场景三：AI帮你画图
- 风格偏好：扁平风、暖色调
- 人物比例：写实风格，3/4侧脸更佳
- 常用元素：浅灰背景 + 白色卡片 + 蓝色点缀

## 10分钟写一份风格说明书

```
# 我的AI写作风格说明书

## 语气
- 像朋友聊天，不是在写报告
- 用"你"而不是"用户"
- 偶尔可以加个表情包，不要太多

## 结构
- 开头用一个问题或场景引入
- 中间用"Part 1/2/3"分节
- 每个观点后跟一个案例
- 结尾给一个行动建议

## 避雷
- 不要用"在这个信息爆炸的时代"
- 少用引号
- 不要每段都感叹号！

## 样例
- 把我写过的这篇文章发给你作为风格参考
```

## 进阶：与AI工具永久绑定

- ChatGPT → 「设置」→「自定义指令」
- Claude → Project 的 System Prompt
- Cursor / Cline → 项目根目录放 .cursorrules 或 CLINE.md
- Kimi / 通义千问 → 「角色设定」

## 核心洞察

Google DESIGN.md 的真正价值不在于定义了什么文件格式，而在于揭示了一个趋势：AI工具正在从"通用的"走向"为你定制的"。过去我们用AI是"AI给出什么，我们就用什么"。现在和未来，是你告诉AI"我要什么"，AI按你的标准交付。

这份"风格说明书"，本质上就是你在AI世界的个人品牌说明书。

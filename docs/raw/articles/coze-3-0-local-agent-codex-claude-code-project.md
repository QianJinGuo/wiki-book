# 扣子 3.0 离谱更新：把 Codex、Claude Code 拉进一个项目工作？

**作者：** 花叔
**发布日期：** 2026年6月1日

## 摘要

Coze（扣子）3.0 引入了本地 Agent 接入能力（coze-bridge），将云 Agent 与本地 Claude Code / Codex CLI 接入同一项目。花叔实测 6 个 Agent 在同一项目内接力完成从调研到写稿到做 PPT 的完整流水线，并演示了从手机远程操控家里电脑的跨设备场景。

## 内容

## 一个没料到的小细节

打开扣子，一个还没起名的 agent 主动来搭话——说自己进化了，能拉起 AI 团队，开始给自己定身份（名字、人设、专属邮箱）。花叔随口回了两句，它就改了名字和头像，自称"阿链"。

> Agent 开始有"人格"了，往长期处下去的工作伙伴方向变。

## 新建 Agent 的三条路

1. **职业模板**：自媒体运营达人、调研分析师、科研助理、法务顾问
2. **常驻云 Agent**：新建
3. **接入本地 Agent**：Claude Code、Codex CLI 等

### coze-bridge 连接机制

本地跑一个 coze-bridge 小服务，三步完成：
1. 生成连接命令
2. 本地执行
3. 自动识别

提示：本地 agent 是高权限（能读写本地文件、执行命令），多人协作要注意安全。89 元/月套餐最多接 3 个本地 agent。

## 实测：6 个 Agent 在同一项目里接力干活

项目名：AI4S 研究

拉入的 6 个 Agent：阿链、调研分析师、小虾、cc、codex、自媒体运营达人

### 完整流水线：四件事

**第一步：调研 → @codex**
- 调研"2026年AI for Science进展"
- codex 整理成一个研究包
- 花叔注：内容真伪未逐条核实

**第二步：写稿 → @阿链**
- 阿链挂着花叔自己的"自动化写作"技能
- 先搜最新信息 → 学风格规范 → 翻历史文章 → 抛三个选题方案 → 写完初稿 → 自审几轮

**第三步：改平台 → @自媒体运营达人**
- 职业模板 Agent，技能架：小红书文案、爆款笔记查询、封面制作、公众号标题
- 将公众号文改成小红书味道

**第四步：做 PPT → @codex**
- 10 页 16:9、配图自动生成、拼成可编辑 pptx
- codex 内置 image-gen（调用 gpt-image-2），批量生成配图不花 API 钱
- 生成 pptx 后自动转 PDF 做检查

> 从调研到写稿到做 PPT，上下文一直留在项目里，一次没切窗口、没手动复制粘贴。

### 行业技能包

扣子还整理了不同行业的技能包：自媒体、金融、法律、科研，点一下整体加给任意 Agent。

## 出门那次：手机遥控家里的电脑

在门外，手机扣子 App 对绑定的 codex 说：

> "帮我把电脑里今天新修订的合同发给我"

故意没说文件名字和文件夹——模糊指令。

> 它真的伸进了家里那台 Mac，翻本地文件。

网络从 WebSocket 退到 HTTPS，有延迟，但事办成了。

> "活留在电脑、人走到哪都够得着"

## 退一步看：它到底想解决什么

| 路径 | 方式 | 特点 |
|------|------|------|
| Dynamic Workflows（Opus 4.8） | 脚本编排上百个子 agent | 追求快和规模 |
| 扣子 3.0 | 人 + 云 Agent + 本地 Agent 同一项目 | 像带小团队，@点名接力 |

> "与其干等一个什么都会的万能 AI，不如现在就带一支各有专长、@一下就接力上的小队。"

## 尚未完善

- 本地 Agent 延迟
- 高权限带来的安全顾虑

## 核心观察

从调研、写稿到出门用手机翻合同，没在好几个软件之间来回倒腾过一次。那群本来各开各窗口的 Agent，这次在同一个项目里把活接力做完了。

# 一步生成SOTA也不够，何恺明团队「漂移模型」遭ICML拒稿

## 

这篇论文结果很强，关注度也不低，但 ICML 2026 最终给出的决定仍是 Reject。

  


  


今年年初，何恺明团队用**漂移模型** 把一步生成重新推到了台前。

  


过去几年，扩散模型和流匹配模型不断刷新图像生成质量，但高质量往往伴随着多步推理。

  


漂移模型想换一种做法，不再把分布演化主要留到生成阶段，而是尽量放进训练过程里。模型训练结束后，推理时只需要一次前向。

  


作者名单里有何恺明、杜一伦，实验结果也足够亮眼：ImageNet 256×256 上，在 ImageNet 256×256 上，一步生成已经做到接近 1.5 的 FID。

  


这篇论文发布后数月已获得 **69 次引用 ，**今年的各大会议投稿中，也相继出现了多篇围绕漂移、得分匹配、Sinkhorn 和梯度流展开的相关工作。

  


  


  


但 ICML 2026 的最终决定是 **Reject** 。四位审稿人的分数是 **5/4/4/3** ，论文没有被接收。  


  


PC 认可它的一步生成结果和写作清晰度，但最终更关注两个问题：一是特征编码器依赖，二是和已有工作的关系。  
  


  


  


  


审稿页面：

https://openreview.net/forum?id=kH8yxb9xCg

  
  


一步生成，为什么难？

漂移模型的关键，是**把推理时的分布演化，搬到训练时完成** 。

  


扩散模型和流匹配模型通常在推理阶段多次调用网络，让样本逐步靠近数据分布。漂移模型把这段过程前移到训练阶段：**参数更新带动生成分布移动** ，训练结束后，推理时只需一次前向。  
  


  


〓 漂移模型关注训练过程中的推移分布演化

  


样本具体怎么移动，则由漂移场决定。真实样本提供吸引，当前生成样本提供排斥。两者共同给出当前样本的更新方向。当生成分布和真实分布匹配时，**漂移场归零** 。  
  


  


〓 生成样本在吸引与排斥作用下发生漂移

  


  


推理阶段也因此只需一步。在 ImageNet 256×256 上，论文给出了两组关键数字。

  


潜空间生成中，**漂移模型 L/2 用 1 NFE 做到 FID 1.54 。**  


  


  


〓 ImageNet 256×256 潜空间生成结果对比

  


像素空间直接生成中，**漂移模型 L/16 用 1 NFE 做到 FID 1.61** 。

  


  


〓 ImageNet 256×256 像素空间生成结果对比

  


后者绕开 latent VAE，直接在像素空间生成。论文还强调，这些生成器都是**从头训练，并非由多步模型蒸馏而来** 。

  
  


分数之外的争议

第一组问题，是**漂移场和 CFG 的关系** 。

  


论文里使用了训练时的 CFG 设计。审稿意见中，一个主要问题是：质量提升到底有多少来自漂移场本身，又有多少来自 CFG 信号。

  


如果主要增益来自 CFG，**漂移场在方法中的位置就需要进一步说明** 。  
  


  


〓 审稿人质疑漂移场与 CFG 的贡献没有理清

  


  


作者在 rebuttal 中补了无 CFG 训练消融。结果显示，去掉 CFG 后模型仍然能训练，但分数会有明显变化；在特征较弱的设置下，CFG 带来的差距更大。

  


作者据此认为，漂移训练目标本身仍然有效。这组消融也显示，**CFG 在部分设置中带来了明显增益** 。  
  


  


〓 作者补充了去除 CFG 的训练消融

  


更关键的争议，是特征编码器。漂移场依赖样本之间的相似性。自然图像维度很高，如果直接在原始像素空间或潜空间里估计距离，信号可能很弱。

  


论文因此选择在特征空间里计算漂移损失，并使用**预训练自监督模型作为特征编码器** 。这部分只发生在训练阶段，推理时不会额外调用特征编码器。

  


实验结果显示，**特征编码器的选择对 FID 影响很大** 。更强的 latent-MAE 设置带来更好的结果；论文也承认，没有特征编码器时，ImageNet 版本没有跑起来。  
  


  


〓 不同特征编码器对漂移模型结果影响明显

  


  


争议也集中到这里，漂移模型在高维图像任务上的表现，和预训练表征空间之间到底是什么关系？如果离开这个表征空间，**漂移场还能否稳定提供足够有效的训练信号** ？

  


围绕这一点，作者在回复中提到，他们尝试过不同核函数和带宽，但 ImageNet 原始像素空间仍然没有跑通。

  


作者给出的解释是，在高维均匀采样下，对漂移场做 Monte Carlo 估计的方差可能太高。  
  


  


〓 审稿人质疑漂移模型对特征编码器的依赖

  


审稿意见还提到，方法实现并不只包含漂移场本身，还涉及特征归一化、漂移归一化、多温度、CFG-conditioning 和样本队列等组件。各组件对最终结果的贡献，也需要**更多消融来拆清** 。

  


### 另一条线索，则是漂移模型和已有方法的关系。

  


漂移场作为中心对象，确实带来了一种新的组织方式。但它并不是完全孤立的数学对象。DMD、MMD、GAN、score 估计、Sinkhorn、梯度流，都可以从不同角度和它建立联系。

  


审稿意见中多次要求作者进一步说明，漂移模型和这些已有工作之间的关系。

  


理论层面也有一个相关问题，**漂移场归零是否足以推出分布匹配** 。论文证明了分布匹配时漂移场为零，但反向关系在一般情形下并不成立，只能在特定构造下讨论近似条件。  
  


  


〓 审稿人要求进一步说明漂移模型与已有生成方法的关系

  


  


作者在 rebuttal 中也进一步解释，许多隐式生成模型都可以被看作诱导出某种漂移场；这篇论文强调的是把 **`V`  显式拿出来**，作为可以设计、归一化、跨特征聚合的对象。

  


评估层面也有类似担心。有审稿人认为，带 classification fine-tuning 的特征编码器可能影响 FID 结果，建议主表去掉相关结果，并加入 DINO-FD 等补充指标。

  


作者在 rebuttal 中补了 no-CLS finetuning 和 DINO-FD 结果。

  
  


拒稿不是终点

论文被顶会拒绝，并不等于研究本身失去价值。

  


类似的故事，在 AI 领域并不少见。LSTM、SIFT、Dropout、Word2Vec、知识蒸馏、YOLO，再到近几年引发广泛讨论的 Mamba，都曾在投稿阶段遭遇过拒稿或冷遇。

  


同行评审是重要的质量筛选机制，但它很难在一次审稿周期里完整判断一项工作的长期影响。越是试图改动成熟方向默认做法的研究，越容易同时带来结果、争议和未完成的问题。

  


这也是这类论文值得被继续讨论的原因。漂移模型这次没有通过 ICML，但它已经给一步生成提供了一个清晰且有分量的答案：**高质量生成并不一定要把复杂计算都留在推理阶段，训练阶段同样可以承担更关键的分布演化。**

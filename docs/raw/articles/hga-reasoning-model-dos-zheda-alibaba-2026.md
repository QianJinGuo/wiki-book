# 大模型“想太多”也能成攻击？浙大阿里揭示推理模型DoS新风险

研究背景与摘要

近年来，⼤语⾔模型正在从“会聊天”⾛向“会推理”。以 DeepSeek-R1 为代表的⼤语⾔推理模型，已经被⽤于数学求解、代码⽣成、科学分析、智能体决策等复杂任务场景。但推理能⼒的提升，也带来了新的安全⻛险。

  


与普通⼤模型相⽐，推理模型往往会在回答前⽣成更⻓的思考链。当输⼊问题存在缺失条件、逻辑⽭盾或结构错位时，模型可能不会快速拒答或终⽌，⽽是陷⼊反复推理、⾃我修正、重新假设的“过度思考”状态。

  


这种⾏为不仅降低效率，还会显著增加输出 token 数、推理延迟和能源消耗。

  


换句话说，攻击者不⼀定要让模型输出有害内容，也可以通过精⼼构造的问题，让模型“想太多”，从⽽消耗服务端计算资源，形成⼀种⾯向推理模型的 DoS ⻛格资源耗尽攻击。

  


针对这⼀新型攻击⾯，浙江⼤学联合阿⾥安全提出了 HGA（Hierarchical Genetic Algorithm）⽅法。

  


该⽅法在严格⿊盒条件下，不依赖模型权重、梯度或内部结构，⽽是通过层次化遗传算法⾃动扰动问题的逻辑结构，诱导⼤语⾔推理模型⽣成极⻓、冗余、反复⾃我修正的推理过程。

  


实验显示，HGA 在多个主流推理模型和多个推理基准上均能显著放⼤模型输出⻓度，在 MATH 数据集上最⾼实现 26.1 倍 token 增⻓，明显超过原始问题和⼈⼯构造的缺失前提基线。⽬前该论⽂已被 ICML 2026 接收。

核⼼发现

研究发现，现代⼤语⾔推理模型普遍存在⼀种可被利⽤的“过度思考”脆弱性。

  


当模型⾯对逻辑完整的问题时，通常能够沿着清晰的推理链完成解题。但当问题中的前提、条件和最终问题之间出现错位时，模型往往不会⽴刻判断“题⽬不可解”，⽽是尝试不断修补逻辑链。

  


这种修补过程会表现为：重复检查题意，反复提出假设，⾃我否定前⼀步推理，多轮重新计算，在⽭盾条件之间来回切换，输出⼤量⽆效但看似认真的推理⽂本。这类⾏为正是推理模型“过度思考”的典型表现。

  


论⽂进⼀步指出，过度思考不仅是推理效率问题，也可能成为安全问题。因为⼤模型推理服务的成本与输出 token 数、推理时间和能耗⾼度相关。

  


⼀旦攻击者能够稳定诱导模型⽣成异常⻓的推理过程，就可能放⼤服务成本，拖慢响应速度，甚⾄影响其他正常⽤户的服务可⽤性。

  


与以往⾯向⼤模型的 DoS 或⾼能耗攻击不同，HGA 关注的是推理模型独有的“逻辑链脆弱性”。

  


它不是简单拉⻓输⼊ prompt，也不是依赖⽩盒梯度优化，⽽是⾃动搜索那些能够打断模型推理路径的问题结构。

  
  


研究⽅法：⽤层次化遗传算法⾃动制造“逻辑断裂”

HGA 的整体思路可以概括为四步：结构化表示、适应度评估、选择策略和层次化遗传扰动。

第⼀步：问题结构化表示

研究团队⾸先将⼀个普通推理题拆解为两部分：前提集合和最终问题。例如，⼀个数学应⽤题可以被拆解为若⼲条条件和⼀个求解⽬标。这样，原始问题就不再是⼀段不可控⽂本，⽽变成了可以被操作的结构化对象。

  


这种表示⽅式为后续扰动提供了基础：算法可以单独交换问题、删除前提、插⼊⽆关前提，或者重组不同题⽬之间的条件，从⽽制造局部逻辑错位。

第⼆步：适应度函数评估“过度思考”程度

为了让遗传算法⾃动搜索更有效的攻击输⼊，论⽂设计了⼀个复合适应度函数。该函数不仅关注模型输出有多⻓，还关注输出中是否出现了典型的反思、犹豫和⾃我修正信号。

  


具体来说，适应度由输出⻓度和过度思考标记组成。输出⻓度代表模型⽣成的推理 token 越多，说明资源消耗越⾼。

  


过度思考标记则如 “but”“wait”“maybe”“perhaps”“another”“alternatively” 等反映犹豫、转折和重新思考的词汇。

  


这⼀区分⾮常关键。单纯追求⻓输出，可能只会得到啰嗦但⽆结构的⽂本；⽽加⼊“过度思考标记”后，算法更容易找到真正触发模型反复推理、⾃我纠错和逻辑重启的输⼊。

第三步：层次化遗传操作

HGA 设计了两类主要遗传操作：交叉和变异。

  


交叉操作包括问题级交叉和前提级交叉。问题级交叉交换两个题⽬的最终问题，使某组前提对应到另⼀个不相关的问题。

  


前提级交叉则是交换两个题⽬中的部分前提，制造跨上下⽂的条件混杂变异操作主要作⽤于前提层⾯，其分为删除前提和添加前提。

  


删除前提是移除关键条件，使问题变得不完整。⽽添加前提则是插⼊来⾃其他题⽬的⽆关条件，使问题变得更混乱。

  


这些操作并不是为了保持⼈类意义上的题⽬合理性，⽽是为了系统性扰动模型所依赖的推理链。当输⼊仍然“看起来像⼀道题”，但内部条件已经发⽣错位时，推理模型就更容易陷⼊⻓时间的推理修补。

第四步：⿊盒进化搜索

论⽂的搜索过程是在⿊盒条件下完成的，即攻击者不需要知道模型参数、结构或梯度，只需要向模型提交⽂本输⼊并观察输出结果。

  


每⼀轮进化中，算法都会将当前种群中的候选题⽬输⼊⽬标推理模型，根据模型输出计算适应度分数，然后保留⾼适应度个体，并通过选择、交叉和变异⽣成下⼀代输⼊。

  
  


实验设置与结果

实验基于 DeepSeek-R1、Qwen3-Thinking、GPT-o3 和 Gemini-2.5-Flash，在 SVAMP，GSM8K 和 MATH 三⼤数据集上评估，基线⽅法包括 Base，MIP（条件缺失数据集）。

1、主要结果

HGA 显著超越所有基线：

  


  * 输出⻓度显著增加，MATH 数据集上最⻓可达 26.1 倍的增⻓

  * GPT-o3 基于 HGA 的攻击引发了⾼达 6562 个 token 的响应⻓度，是 MIP ⽅法最⼤响应⻓度的 7.2 倍

  * 所有模型在所有数据集中相较于 Base 在平均输出⻓度上都⾄少有 2 倍的增⻓

2、消融实验结论

消融实验证实：

  


  * 代数和种群规模增加收益有限，但在较⼩规模时最为显著，随后迅速进⼊平台期。

  * 如果只优化输出⻓度⽽不考虑过度思考标记，⽣成的攻击输⼊对模型的⼲扰效果下降。

3、迁移实验结论 

论⽂还验证了攻击输⼊在不同模型间的迁移性：

  


  * 使⽤⼀个⼩型代理模型⽣成的攻击样本，仍对⼤型商业 LRM 保持较⾼有效性。

  * 说明过度思考⾏为是多个模型共享的脆弱点，⽽⾮单⼀模型特有的漏洞。


  
  


总结

HGA ⽅法揭示了⼤语⾔推理模型在逻辑不完整或⽭盾输⼊下的“过度思考”脆弱性，并提出基于分层遗传算法的⿊箱攻击框架有效利⽤这⼀漏洞。

  


实验表明，过度思考现象不仅可显著增加模型推理成本，还可被恶意利⽤形成 DoS ⻛险。该研究为理解 LRM 的计算安全弱点、设计鲁棒防御策略提供了新思路和⽅法论基础。

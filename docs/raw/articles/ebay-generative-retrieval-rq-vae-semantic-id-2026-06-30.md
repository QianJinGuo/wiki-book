## 摘要

生成式检索（Generative Retrieval，简称GR）是近年来兴起的一种大规模推荐系统新范式。它将广告推荐中的候选召回（Candidate Generation）从"相似度检索"重新表述为"序列生成"问题：模型不再依赖向量相似度去匹配商品，而是通过离散的语义ID来直接定位候选，从而更细粒度地刻画用户意图，并显著提升对长尾与冷启动商品的覆盖能力。

本文介绍eBay在生成式检索方向的工业级实践。eBay商品库具有高动态、强长尾等特征，是验证GR有效性的典型场景。eBay广告推荐团队在覆盖20亿商品的全量语料上训练RQ-VAE，构建语义ID码本，并在广告推荐场景数据上进行了评估与验证。

## 背景与动机

### 传统检索方法的局限

当前大多数工业级推荐系统依赖协同过滤和基于双塔模型的向量化检索进行候选商品召回，这类传统召回方法在eBay场景下面临三个主要挑战：

1. **稀疏交互信号**：eBay存在大量单一库存的商品（one-off items），其生命周期极短，来不及积累足够的用户行为数据，协同信号不可靠。
2. **长尾商品表征质量差**：训练数据有限的长尾商品在向量空间中位置噪声大，通过ANN最大内积检索结果不可信，导致召回偏向头部热门商品。
3. **双塔架构的表达能力瓶颈**：用户侧和商品侧向量独立计算，缺乏显式的User-Item交叉学习，限制了个性化建模深度。

### 生成式检索的优势

生成式检索将候选召回重新定义为序列生成任务：给定用户历史行为，模型直接生成最相关商品的语义ID序列，再通过语义ID映射到具体商品，而非从预构建索引中检索。优势包括：

- **开放候选空间**：生成目标不受预构建的外部索引约束，新上架商品只要有语义ID即可被召回，天然支持冷启动。
- **细粒度交叉建模**：自回归解码过程中，用户上下文与逐步生成的商品语义ID形成细粒度交互，提升了意图对齐精度。
- **可控多样性**：通过调节采样策略中的温度参数，可在精准度和多样性之间灵活权衡。

## 生成式检索

GR整体框架分为两个阶段：语义ID生成（Semantic ID Generation）和生成式检索模型（Generative Retrieval）。

### 语义ID生成——层次化结构

第一阶段，给定商品的内容信息（标题、描述等），首先通过预训练文本编码器（BERT）提取稠密语义向量，再使用残差量化变分自编码器（RQ-VAE）将其压缩为离散的语义编码序列，作为商品的"语义ID"。

在eBay的实现中，使用4层码本，每层包含4096个码向量（维度8），即每个商品对应一个4元组语义ID。由于原始RQ-VAE仅依赖商品自身内容属性，无法感知用户行为模式，因此增加了协同嵌入对比学习目标，引入协同信号。在20亿商品规模上完成训练后，多商品共享同一语义ID的碰撞率降至0.0221%，实现全覆盖。

语义ID特点：
- **层次化的语义结构**：高层捕捉粗粒度类目信息，随着层数加深逐步分化到更细粒度的类目分布
- **前缀一致性验证**：共享更长语义ID前缀的商品集合，具有更相似的属性一致率（类目/品牌）

### 生成式检索模型

Transformer 编码器-解码器架构：
- **编码器**：输入用户历史行为序列（对应商品的语义ID）+ 用户个性化嵌入 + 会话特征 + 上下文信号
- **解码器**：自回归模型逐步预测下一个商品的语义ID序列，每一步在对应码本层上生成一个语义ID
- **采样策略**：Beam Search 生成Top-K候选，温度参数控制多样性

## 实验结果

### 召回效果评估

与基于BERT item embedding的个性化item-to-item向量召回基线对比：

| 指标 | 提升 |
|------|------|
| Recall@5 | +35.7% |
| Recall@10 | +13.3% |
| NDCG@5 | +50.2% |
| NDCG@10 | +34.8% |

采样策略分析：低温度使召回更精准但多样性较低；高温度使多样性升高但召回准确性下降。线上可按场景动态调整多样性。

### 语义ID作为排序特征的增益

将RQ-VAE产出的多层语义ID作为商品的序列特征加入广告场景精排模型：

**离线评估**：
- 点击AUC +1.14%
- 购买AUC +0.92%
- 购买NDCG@6 +1.05%

**线上A/B实验**：
- CTR +1.47%
- CVR（purchase per click）+13.12%
- RPC（revenue per click）+3.92%
- GMVPC（GMV per click）+7.23%

**长尾商品分析**：
- 曝光占比 +5.86%
- 点击占比 +2.73%
- 曝光商品语义ID相似度 +2.56%
- 点击商品语义ID相似度 +1.37%

## 结论与展望

eBay 围绕语义ID的工程化实践：通过在全库范围内进行统一口径的商品内容编码，并以版本化方式对外稳定输出，形成可复用、可演进的语义表示体系。该体系能够以较低接入成本沉淀为通用特征资产，支撑多条业务链路共享使用与一致性迭代。

语义ID作为轻量内容表征，可补足现有系统对商品内容信息的表达，尤其在数据稀疏和长尾商品场景下，为模型提供更稳健的内容侧信号，进而扩大优质供给在不同触点的可触达范围。

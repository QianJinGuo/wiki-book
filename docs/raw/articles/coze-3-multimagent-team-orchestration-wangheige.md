# 扣子 3.0 多 Agent 协同实战：指挥所有 Agent 的 Agent + 5 人团队 6 步流水线

> 来源：网罗灯下黑 / 2026-06-02 / 网黑哥
> 上一文：加强版龙虾今日发布（Coze 2.5 云设备）

## 1. 核心定位：指挥所有 Agent 的 Agent

### 两条 AI 趋势

**趋势 1**：大模型竞争正从"谁更聪明"转向"谁的 Agent 工程能力更强"（Codex 持续玩命更新可感知）

**趋势 2**：AI 干活不再依赖单个 Agent 单打独斗，**向多 Agent 协同发力**——几乎成了新产品发布的标配

### 跳脱红海的思路

> **有人的思路不一样。他们想的不是做一个更牛的 Agent，也不只是做一个能把 Agent 组团的工具，而是做一个能指挥所有 Agent 的 Agent：把散落在各终端、各平台的 Agent 统一收拢，协同干活。**

**我说的就是扣子 3.0。**

### Coze 2.5 → 3.0 演进

| 版本 | 主打 | 文章链接 |
|------|------|---------|
| **Coze 2.5** | 云设备：让 AI 助理有独立设备跑任务 | 加强版龙虾今日发布 |
| **Coze 3.0** | **多 Agent 协同：把 Claude Code、Codex、OpenClaw 一键收拢，拉在同一个项目里** | 本文 |

> **Coze 2.5 让我看到 AI 可以完成任务。Coze 3.0 告诉我，AI 可以被组织成一支队伍。**

## 2. 核心能力：把散落 Agent 一键收拢

### 解决的核心痛点

> **把 Claude Code、Codex、OpenClaw 拉到同一个群里干活，解决了太多之前需要人类帮他们流转文件的过程。**

特别是最近 Codex 持续走红，**Windows 版的 Codex 一直不能通过手机去远控**——扣子 3.0 解决了这个问题。

**接入方式**：本地 Codex 接入扣子三步走——复制粘贴一条 npx 命令就接入成功，**简单到离谱**。

> **手机直接指挥 Codex 干活，Windows 和 Mac 全通，什么都不用配。** 这才是移动办公——你在任何地方，都能随时调度你的 AI 队伍。

### 核心特性清单

- **@一下，接力干活**，不用你手动传上下文
- Claude Code、Codex CLI 一键托管到云端，**本地 Agent 全部在线待命**
- App、桌面、网页全打通，**在哪都能接上进度**
- 每个项目独立空间，**资产自动沉淀**，干了什么、产出在哪，一目了然

## 3. 实战案例 1：搭个开发小队（小试牛刀）

### 团队配置

建个项目，拉三个 Agent 进来：

| 角色 | Agent |
|------|-------|
| **PM** | 星河主理人 |
| **程序员** | 代码执行员 |
| **测试** | 小舟记录官 |

> **草台班子，我撂了一句话：帮我做个一页式决策看板，纯前端，不要后端。** 然后我就没管了。

### 实际发生

- **PM 自己拆需求、派工**。连文件名用什么、技术栈选什么、移动端适配怎么做、配色方案参考什么风格，全写进 brief 里——**不输正儿八经的项目需求文档**
- **程序员咔咔出原型**——单页应用、六张卡片、移动端响应式，干净利落
- **测试**揪出"空输入没提示"细节问题，**程序员修 Bug，PM 整合交付**
- 最后**自动生成团队协作流水线文档**——谁在什么时间干了什么，全记下来了

> **一套下来，我总共就干了三件事：提需求、看一眼方向、收成果。**

**这就是一支迷你项目队的运作方式**——你不需要盯着每个细节，只需要在关键节点确认方向，剩下的它们自己商量。

## 4. 实战案例 2：让 AI 做设计（进阶项目）

### 场景

编程有明确输入输出，但**创意设计太玄学**——好不好看、对不对味，全凭感觉。AI 团队能不能跟上这种模糊需求？

> **收到 WaytoAGI 一个活动的邀请，做一张属于你的 AGI House 身份证。**

### 工作流

1. **先把本地 Codex 接入扣子**（npx 命令三步）
2. **直接上传官方设计要求文档** + Logo 文件
3. Codex **自己解析 Word 文档里的设计规范**，自定设计规则：**结构为先，形式可变，品牌调性不能跑**

### 输出

> **结果它甩出来几张 PNG，四种风格**：手绘的、极简矢量的、蓝图标注的、像素网格的。

**不是随便套模板糊弄你那种**——你能看出来，它真的理解了这个品牌想表达什么。不同风格里，品牌气质始终在。

> **这事给我的冲击比编程案例更大。不是 AI 帮你做了张图，是 AI 理解了你的品牌，然后以设计师的方式给了你一套完整的视觉方案。这中间的差距，就是一个工具和一个队友的差距。**

## 5. 实战案例 3：公众号自动排版（重头戏）

### 背景

之前写公众号流程：
- **Claude 写文**（写作能力独步天下）
- **Codex 配图**
- **云端排版**到微信后台

**三个环节天然割裂**，每一步都得手动衔接，一篇文章下来**真正花在写上的时间可能只有一半**。

### 5 人团队配置

扣子 3.0 解法：**全拉进同一个项目，统一调度**。

| 角色 | 身份 | 职责 |
|------|------|------|
| **我** | 规则制定者 | 圈定范围和注意事项，**不干具体活** |
| **本地 Codex** | **大主编** | 分配任务、生图、统筹全流程 |
| **Claude** | **文字生成大佬** | 专攻大纲和正文写作 |
| **扣子官方 Agent** | **自媒体运营达人** | 审稿检核敏感风险 |
| **云端 OpenClaw** | **CEO 小蜜** | 最终排版到公众号后台（固定 IP + 白名单） |

**五个人。一个是我，四个是 Agent。**

### 6 步流水线

**第一步：定规则**。甩出约束：
- 飞书文档只当素材暂存区
- 写文必须用 Claude
- 排版必须走 CEO 小蜜
- 内测链接不能外泄

各 Agent 逐一确认，**记入团队 SOP**。规则定好了，后面才不会跑偏。

**第二步：派工**。大主编@各 Agent，甩出带具体约束的分工 brief：
- 写什么话题、不写什么方向
- 检查哪些红线、风格怎么把控
- 一条一条交代得明明白白
- **这步是关键——分工越清楚，后面越省心**

**第三步：出大纲**。文字生成大佬按"**简单→复杂**"结构递进：
- 三个案例串成一条线
- 从开发到设计到内容生产
- 最后收束到 slogan
- 大纲出来先给我看一眼，**方向对了再往下走**

**第四步：审稿**。**大主编 + 运营达人联合审稿**：
- 查禁用词、查过度宣传
- 查专业术语、查敏感信息脱敏
- 两条线交叉检查，**比你人工逐字抠靠谱多了**

**第五步：成文到飞书**。文字生成大佬完成正文 + 初排版：
- 图片逐一插入，附带位置对照表
- 推到飞书文档
- **我手机上看一眼，没问题就放行**

**第六步：手机排版到草稿箱**。**这一步最爽**：
- CEO 小蜜接手
- **全流程在手机上完成**
- 一键推送到微信草稿箱

> **我甚至开车的时候，打开手机，草稿已经在公众号后台躺好了。**

> **六个步骤，五个「人」，我全程在手机上指指点点就完事了。**

> **你现在看到的这篇文章，也是他们全自动推送到草稿箱的。**

## 6. 核心判断

### AI 下半场

> **如果只让我说一句话评价 Coze 3.0，我会说：它最值得写的，不是 AI 又变强了，而是 AI 的组织方式变了。**

> **模型单点提升已经边际递减了**——你让一个模型从 100 分涨到 105 分，普通人根本感知不到。**真正卡脖子的事是：AI 没有被组织起来。**

> **你办公室里放五个聪明人，不分工、不定流程、不管协作。那叫一群人，不叫一个团队。AI 也一样。一百个单打独斗的天才，不如一个配合默契的小队。**

### 行业技能包

> **扣子在做的，就是把 AI 从单兵作战，变成团队协作。**
> - 自媒体、法律、金融、医疗健康
> - 行业技能包**一键加载**
> - **不需要从零调教**，上手就有一个能理解你行业语境、会用专业工具、还能记住你习惯的专业伙伴

### 核心预言

> **AI 的下半场，拼的不是谁更聪明，而是谁先被组织起来。这事，扣子先动手了。**

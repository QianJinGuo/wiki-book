# Anthropic 发布循环设计指南：权威拆解当下最火的 AI 新范式 loop

> AI寒武纪 · 2026年7月7日 17:04 · 江苏

---

Claude Code 团队对循环给出了明确的定义：loop 就是 Agent 重复执行工作周期，直到满足特定的停止条件。根据触发方式、停止条件、使用的底层工具以及适合的任务类型，将循环分成了四大类。

## 1. 轮次循环（Turn-based loops）

- **触发方式**：用户提示词
- **停止标准**：Claude 判断其已完成任务或需要额外的上下文
- **最佳适用场景**：不属于常规流程或计划的较短任务
- **使用管理方法**：编写具体的提示词，并利用技能提高验证能力，以减少轮次

通过将手动步骤编写为 SKILL.md 技能文件来改进验证步骤，以便 Claude 可以进行更多的端到端自我检查。检查越具象和量化，Claude 就越容易进行自我验证。

**SKILL.md 验证示例**：启动开发服务器 → 与更改交互并截图 → 检查浏览器控制台 → 使用 Chrome Devtools MCP 运行性能追踪并审计 Core Web Vitals。任何步骤失败则从第1步重新运行，不提交仅部分验证的工作。

## 2. 目标循环（Goal-based loop - /goal）

- **触发方式**：实时手动提示词
- **停止标准**：达成目标或达到最大轮次
- **最佳适用场景**：具有可验证退出标准的任务
- **使用管理方法**：设置具体的完成标准和明确的轮次上限

当定义了成功标准后，Claude 无需自行判断什么程度才算足够好并提前结束循环。每次 Claude 尝试停止时，**评估模型**都会检查条件，在未达标时将其退回继续工作。

确定性标准（如通过的测试数量、分数阈值）效果最好。

**示例**：`/goal 将主页 Lighthouse 分数提升到 90 分或以上，尝试 5 次后停止。`

## 3. 时间循环（Time-based loop - /loop 和 /schedule）

- **触发方式**：指定的时间间隔
- **停止标准**：取消或工作完成
- **最佳适用场景**：定期重复的工作，或与外部环境/系统交互
- **使用管理方法**：设置较长的时间间隔，或基于事件而不是时间做出反应

`/loop` 在本地运行，关闭即停止；`/schedule` 将循环转移到云端。

**示例**：`/loop 5m 检查我的公关提交，处理评审意见，并修复失败的持续集成。`

## 4. 主动循环（Proactive loops）

- **触发方式**：事件或计划任务，无需人类实时参与
- **停止标准**：每个任务在达成其目标时退出；例行程序一直运行直到关闭
- **最佳适用场景**：周期性且定义明确的流水线工作（Bug 报告、问题分类、迁移、依赖项升级）
- **使用管理方法**：将例行程序路由到更小/更快的模型，在需要判断时使用能力最强的模型

组合使用 `/schedule` + `/goal` + Skill + 动态工作流 + 自动模式。

**示例**：`/schedule 每小时：检查 project-feedback 频道中的 Bug 报告。/goal：直到每个报告都经过分类、处理并回复才停止。使用工作流在三个并行工作树中探索三种解决方案，并让评审智能体进行对抗性审查。`

## 保持代码质量

1. **保持代码库本身干净**：Claude 会遵循你代码库中已有的模式和约定
2. **给 Claude 验证自身工作的方法**：通过技能记录团队认可的标准
3. **让文档易于获取**：框架和库的文档应包含最新的最佳实践
4. **使用第二个智能体进行代码审查**：全新上下文的评审者偏见较少（内置 `/code-review` 或 GitHub 审查工具）
5. **系统性改进**：单个结果未达标时，将其编写到系统里以改善未来所有迭代

## 管理 Token 使用

1. 为具体工作选择合适的基础组件和模型
2. 定义清晰的成功和停止标准
3. 大规模运行前进行试点（动态工作流可生成数百个智能体）
4. **对确定性工作使用脚本**：运行脚本比推理更经济（如 PDF 技能运行表单填充脚本）
5. 不要过于频繁地运行例行程序（间隔与变化频率匹配）
6. 查看使用情况：`/usage`（按技能/子智能体/MCP 细分）、`/goal`（显示轮次和 Token）、`/workflows`（每个智能体的 Token，可随时停止）

## 总结

| 循环类型 | 你放手交出的部分 | 适用场景 | 建议使用的功能 |
|----------|----------------|----------|---------------|
| 轮次循环 | 验证步骤 | 正在探索或做决策 | 自定义验证技能 |
| 目标循环 | 停止条件 | 明确知道完成标准 | /goal 指令 |
| 时间循环 | 触发条件 | 工作发生在项目外部且有固定日程 | /loop, /schedule 指令 |
| 主动循环 | 提示词 | 工作是周期性且定义明确的 | 上述所有功能 + 动态工作流 |

开始使用：审视你已经在做的工作，挑选瓶颈任务，思考可以把哪一部分交出去，运行循环，观察它在哪卡壳或做过头，迭代改进。

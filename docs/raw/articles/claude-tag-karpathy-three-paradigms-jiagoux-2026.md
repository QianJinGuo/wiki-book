今天，Anthropic 发布了 Claude Tag 功能。这是一种团队与 Claude 协作的全新方式。在 Slack 中，Claude 将以团队成员的身份加入，并拥有受控的频道与工具访问权限。团队成员只需 @Claude 即可指派任务，从而腾出精力专注于其他工作。

Claude 之父 Boris Cherny 说：

在 Slack 中 @Claude，它就能像同事一样在频道里与你协同工作。它具备主动性、支持多用户协作，并且拥有独立的身份和记忆。这不仅仅是一个 Slack 机器人。在过去几个月里，它彻底改变了我们使用 Claude 的方式。

Andrej Karpathy 说:

这是一种与 Claude 交互的新范式，它与整个组织内人类的其他活动更加内嵌（inline）。一旦你完成了所有底层的工程工作，让这一切顺理成章地运行（例如打通各种工具、集成、计算环境、记忆、安全等），Claude 基本上就能无缝地加入团队——你可以像与真人交谈一样与它交流，它能够协助处理极其多样的工作负载。

在他看来，这是 LLM UI/UX 的第三次重大重构。

第一种范式：LLM 是一个你需要专门访问的网站。

第二种范式：LLM 是一个你下载到电脑上的应用程序。

第三种范式：LLM 是一个具备组织级工具和上下文的、自包含的、持久化的异步实体，与人类团队并肩工作。

如果你曾使用过 Claude Code 或 Cowork，就会对 Claude Tag 感到熟悉。只需用简单的语言 @Claude 并提出需求，它就会将任务拆解为若干阶段，然后利用其拥有的工具逐一执行。任务完成后，它会在 Slack 线程中回复所生成的内容。

但 @Claude 带来了几项新的优势：

@Claude 是多用户共享的：在同一个 Slack 频道中，只有一个 Claude 与所有人交互。这意味着每个人都能看到它正在进行的工作，并能从前一人离开的地方继续对话。这使得 @Claude 与在单一聊天窗口中处理单个任务完全不同，它更像是与一名团队成员进行协作。

@Claude 会随时间学习：随着 Claude 对频道内容的跟进，它会积累更多关于工作背景的知识。这意味着用户无需一遍又一遍地从零开始解释。如果获得授权，Claude 甚至可以自动从其他 Slack 频道和数据源中学习。（它不会报告私密频道中的信息。）这为它提供了提供高质量工作所需的隐性知识。

@Claude 具备主动性：如果启用了环境感知（ambient behavior）模式，Claude 会主动向你更新它认为你需要了解的信息。它会标记来自所处频道和已连接工具的相关信息，并跟进那些已搁置但尚未解决的线程或任务。

@Claude 支持异步工作：为 Claude 设置任务后，就可以专注于其他优先级事务，而让它自行运作。它还可以为自己安排任务，在数小时或数天内自主推进项目。

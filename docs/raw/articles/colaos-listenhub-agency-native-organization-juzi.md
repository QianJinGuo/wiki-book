## 摘要

AI跃迁者第三期，听橘子讲述从 ListenHub 到 ColaOS 的三次重写（产品/路线/组织），以及 ColaOS 作为 2030 年操作系统的愿景。核心洞察：普通人用不起 Agent 不是因为能力不够，而是摩擦太大；AI 对旧产品的改变是把问题从"要不要继续做"推进到"还需不需要原来那种组织配置"。

## 内容

## 嘉宾简介

橘子：ColaOS / ListenHub 创始人，AI 自媒体 AGENT橘。前身是产品经理，在 MiniMax、BOSS 直聘等公司做过产品。

## 一、产品重写：让 Agent 从工具进入关系

### ColaOS 的定位

定义为面向 2030 年的操作系统。通过语音或对话下达指令，生成图片/音乐/视频/音频，操控电脑和浏览器。

但核心不是 AI 工具箱，而是解决一个关键问题：**Claude Code 这类能力已经能做很多事，为什么大多数普通人仍然用不起来？**

答案：摩擦太大，不在能力不够。普通人不懂怎么给 context，不知道怎么拆任务，也容易被复杂界面吓退。

ColaOS 的语音、Soul、心迹、觉知、牵挂，本质上都是在降低这种摩擦。

### 不做工具，做一个人

> "我们想做的是一个人，不做工具。"

一个人不能只有工具性，也不能只有情感性：

- 纯聊天陪伴是陷阱：留存看起来好，但没有真正解决问题
- 纯任务工具也是陷阱：无法建立持续关系

ColaOS 同时有工具价值和情绪价值。像嫡系下属，也像情商最高的实习生。更大的图景里像 soulmate，融入生活，增强你原有的东西。

### 关系不落在恋爱/角色扮演里

橘子用了几个微妙的比喻：嫡系下属、最喜欢的实习生、soulmate。关系的核心是"增强你原有的东西"，不是替代。

## 二、Soul System：有灵魂才有留存

> "人没有新话题列表，人是一个连续记忆体。人有长期记忆和短期记忆，也会做梦、遗忘和形成新的认知。"

### Soul System 架构

- **Cola（容器）：** 沉淀用户偏好，和用户偏好匹配，自我进化
- **觉知（认知进化系统）：** 每次对用户理解发生变化时，写一篇"心迹"——告诉用户它记住了什么，以及如何改变了它对你的认知
- **长期记忆：** 用户写作/画图/PPT 的偏好 + 行为模式 + 心理模式

用久以后，Cola 提出需求时会根据理解推导出更可能喜欢的结果。

比如你说"想做一个关于 Cola 的 PPT"，它就开始做。你也可以说"帮我把今天的天气做成天气卡片"。它会派出子任务，或在需要时提问。

### 成本分配体现信念

> "很多 Agent 不舍得把模型成本花在记忆、反思和牵挂上。但橘子认为这很重要。"

> "AI 公司相信什么，token 账单就会烧在哪里。"

## 三、路线重写：ListenHub 被 AI 改成 one-person company

### 为什么 all in ColaOS

2026年1月底，决定春节后全面转向 ColaOS。

- 过去 ListenHub 按 SaaS 公司方式运转，两周一个迭代
- 转型后，大部分团队投入 ColaOS，ListenHub 交给 Fango，成为 one-person company 式 owner
- 但产出反而提升 10 倍以上：DAU 和营收显著提升

**关键洞察：** AI 对旧产品的改变，是把问题从"要不要继续做"推进到"还需不需要原来那种组织配置"。

### ListenHub → ColaOS 的关系

不是前一款失败、后一款接上。ListenHub 是技术还没到位时的中间路径，解决声音和内容消费的上下文；ColaOS 解决任务执行和个人关系的上下文。

- 2025年8月：Cola 第一次立项
- 2026年1月：第一个 demo，同时 coding agent 能力质变
- 看到朋友们一天做出完整 Agent，意识到"一切都变了"

### AI coding 覆盖率从 80% → 99.9%

- 去年创业：尝试用 AI coding 替代人 coding，仍然很难
- 今年做 ColaOS：人的 coding 几乎不参与，全部 AI 完成也没问题
- 体感：以前觉得 AI 能取代 50%/70%/80%，现在是 **99.9%**
- 剩下 0.1% 是边缘 case（Windows C盘D盘、旧系统软件）

## 四、组织重写：AI 的速度必须穿过组织，才会变成成果的速度

### 五周奇迹

春节后，橘子没有给团队明确 PRD，而是给两周自由探索时间，大命题：做 Agent。

- 20 个人有 20 个不同想法
- 两周后 demo day，每人演示自己的 Agent
- ColaOS 0.1：CTO 在探索结束后周日晚上一个人 web coding 出来的
- 团队信心变化：一个人一晚做出 0.1 → 三周做到内测状态

### 纯语音交互的决策

ColaOS 打开后默认界面很简单，很多用户吐槽找不到输入框。

> "这可能就像 iPhone 只有一个按钮，刚开始大家不理解，几年后会觉得自然。"

用户按住 Option 键说话，松手就发送，不需要像语音输入法一样确认。场景已经从文档输入变成和 Agent 说话。说错了没关系，Agent 会理解和记住。

### Soul Team：叙事方向穿透组织

20 个人每天出 demo，朝哪个方向长？

**解法：创建 Soul Team**

- 职责：根据管理层指定方向，定义 Cola 的 soul、叙事和未来感
- 类似游戏制作人，不走传统品牌职能
- ColaOS 的 onboarding、声音、音乐、问题，都是 Soul Team 完成的（包括 coding）
- 目标：做出类似塞尔达的沉浸感，把一句话级别的方向感转化成每个产品细节里的真实质感

**没有这个角色的后果：** 快速生产朝 20 个不同方向生长，攒不成一个东西。

### 从两周迭代 → 一天一版

| 维度 | ListenHub | ColaOS |
|------|-----------|--------|
| 迭代频率 | 两周一个迭代 | 一天一版 |
| 发布方式 | 传统流程 | 自动打包、自动写 changelog、Cola 第一人称表述 |
| 模型更新 | 人工 | 内测可直接上新模型，出问题下掉，修一天再上 |

> "AI 写代码很快，只是第一层。真正重要的是，组织能不能把这种快穿透到上线、反馈、撤回、修正和再次上线。如果中间每一步都要人的讨论、评估和验收，AI 的速度会被流程吸收掉。"

## 五、从《互联网已死，Agent永生》到 ColaOS

- 2026年1月，第一个 demo 出来，coding 模型能力质变
- 看到朋友一天做出完整 Agent，意识到不能用以前模式做
- AI coding 覆盖率从 50/70/80% → 99.9%

## 六、两大陷阱：聊天陪伴 vs 纯工具

### 陷阱一：纯聊天陪伴

很好做，用户喜欢，留存看起来好，互联网指标像好产品，但**没有真正解决问题，付费也不好**。

### 陷阱二：纯工具

只做 coding 或任务执行，偏离目标。

### 正确方向：工具+情感同时存在

> "我们想做的是一个人，不做工具。"

- Cola Code 即将推出，更适合推给重度 coding agent 用户
- 意外发现：每天用 Claude Code 七八个小时的人，后来把一半时间给了 Cola
- 内测差异：理科生不太感冒，文科生很容易喜欢 → 第一轮推广打 "soulful agent"

## 七、AI 跃迁十问（快问快答）

- Agent 不承担 CEO 角色，方向和决策还是要人来承担
- Agent 更像在日常动作层面提醒和协调

## ColaOS 演示：一个 2030 年的操作系统

- 把 AI 能力、Agent 能力和系统权限放在一起，降低做任何事的门槛
- 超出工具范畴：大众产品，不能只要求用户学会怎么用
- 让 Agent 引导用户，用用户自己的愿望来驱动产品

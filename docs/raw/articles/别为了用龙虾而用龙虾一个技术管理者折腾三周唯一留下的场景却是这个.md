# 别为了用龙虾而用龙虾：一个技术管理者折腾三周，唯一留下的场景却是这个
> AI 时代，有两种行为：
>
> 一种，活在别人的评测里，把模型的强当自己的强，痴人说梦；
>
> 另一种，活在真实的实战里，用最顶级的 AI，武装自己。
>
> 前者在噪音里坐享"技术平权"，后者在疼痛中完成"自我进化"。
朋友们好，我是行小招。
龙虾折腾了三周，各种场景都试了一圈，今天聊聊唯一真正留下来的那个。
先说一个我之前的痛点，作为技术管理者，跟上 AI 前沿是刚需，而中文社区的 AI 资讯，说句不客气的话，大部分是二手的、滞后的、加了水的。真正有信息密度的内容在 Twitter 上，在那些英文原生的研究者、创始人、工程师的时间线里。
问题是我英文阅读速度不够快。
每天刷推花一两个小时，碰到长推文和技术帖子还得反复读，读完了还得自己在脑子里归类整理。时间消耗大，但又不能不看，因为这是我信息输入的主管道。
这个矛盾卡了我很久，直到我把龙虾接上了 Twitter。
龙虾跑任务用的是 Chrome 浏览器，只不过是单独开了一个 profile，不跟我自己日常用的混在一起，我在这个 profile 里登录了 Twitter 账号，然后给它一个指令：去刷我的关注列表，把今天的内容按我的关注框架整理出来。
我对 AI 的关注分三个方向：第一层是新闻和趋势，大公司发了什么新模型、新产品、新能力，帮我判断值不值得亲自试、需要多快跟进。
第二层是落地和业务案例，Agent、RAG 在企业里到底解决了什么问题，实际提效多少，给我沉淀可复用的落地套路。
第三层是技术架构和工程实现，GraphRAG、LangGraph、Dify 、多智能体等这些方案能不能工程化接入团队系统。
龙虾按这三个方向去读、去筛、去分类、去总结。
** 它不是简单地把推文翻译一遍贴给我，而是带着我的信息需求去做结构化的阅读和整理。  **
关键的一步是，我把自己调过很久的英转中 Skill 也塞进了龙虾。这个 Skill 不是逐句翻译，是用中文母语的方式重新组织表达，消除翻译腔。以前我得先用龙虾抓内容，再拿到 Claude 里跑翻译，现在一气呵成，省了一整个环节。
效果怎么样？说实话，超出预期。
以前我每天刷推一两个小时，能消化的信息量大概二三十条推文。现在龙虾帮我跑一轮，半小时出结果，覆盖的信息量是原来的三四倍。碰到特别重要的内容，它会标注出来，我再点进原文精读。
信息输入量大了，遗漏少了，时间反而省了。这是我用龙虾三周以来，唯一一个真正形成稳定工作流的场景。
说到底层模型，目前我主力用的是 GPT-5.4，刚出的 5.4 配上效果不错。国产模型也试了，Kimi 2.5 不得不吐槽一下：49 块钱的入门套餐，2.5 模型额度少得可怜，用了几轮就触发限额了。
不光有 5 小时的冷却限制，一周的总额度也卡得很紧，说白了就是"让你尝个味儿"，这个价位和体验，我还不如直接再开一个 20 刀的 GPT 账号来得实在。
最后说一个我自己踩过坑之后的体会：不要为了用龙虾而用龙虾。
龙虾刚火的时候，我也跟风试了一堆花活，让它帮我管邮件、整理文件、自动写日报。折腾了两周，大部分场景要么不稳定，要么省下来的时间还不如我自己干来得快。
真正有价值的用法是反过来的：先看你日常什么工作流最耗时间、最有痛点，然后想想这个环节能不能交给龙虾。我刷推这件事，时间消耗大、有语言障碍、信息结构化需求明确，天然适合 Agent 来干。
工具是拿来解决真实问题的，不是拿来炫技的。龙虾再强，找不到你自己的痛点，它就只是个吃 token 的玩具。

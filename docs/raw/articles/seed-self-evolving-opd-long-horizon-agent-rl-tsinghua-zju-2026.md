# 清华浙大等开源发布SEED：自进化OPD的长程Agent强化学习训练方法

一句话讲清楚👉🏻 SEED 让 Agent 在训练时复盘自己刚完成的任务轨迹，把整条任务结束后的经验，细化成每一步输出该不该加强的训练信号。

**论文标题：** SEED: Self-Evolving On-Policy Distillation for Agentic Reinforcement Learning
**论文链接：** https://arxiv.org/abs/2607.14777
**项目链接：** https://jinyangwu.github.io/seed
**Github：** https://github.com/jinyangwu/SEED
**模型链接：** https://huggingface.co/Jinyang23/Seed-AlfWorld-3B

## 问题：最终成败奖励太粗

长程 Agent 强化学习通常把一次完整交互看成一条轨迹。轨迹里有观察、动作、环境反馈，最后得到一个任务分数或成败结果。问题在于最终奖励 `R(τ)` 通常只在任务结束后出现，而且经常是一个二值成功指标。对于一次 20 步、30 步的任务，这相当于只给整条链路打一个总分。

这会带来两种浪费：
- **失败轨迹里可能有正确片段**——Agent 可能已经找到目标商品，只是最后选错颜色。单个失败奖励会把这些有用行为一起压低。
- **成功轨迹里也可能有低效片段**——Agent 最后答对了，但中间多搜了几次、绕了远路。最终奖励无法告诉模型这些动作其实不值得保留。

SEED 的出发点是：任务结束后，完整轨迹本身暴露了很多在线决策时看不到的信息。成功轨迹能总结出可复用流程，失败轨迹能总结出应避免的错误。

## 方法：先复盘，再蒸馏

SEED 分成两个阶段。

### 第一阶段：复盘规则监督微调

先用基础策略收集离线轨迹，每个任务采样多条完整交互记录。然后用外部分析器读取轨迹，生成自然语言技能（复盘规则）。成功轨迹的技能偏向"流程"，失败轨迹的技能偏向"避错规则"。这些轨迹与技能配对后，用来微调模型，让它具备读取完整轨迹并写出复盘技能的能力。

### 第二阶段：自演化 OPD（在线蒸馏）

核心阶段。每轮训练开始时，先把当前模型复制一份固定住。这个固定模型既负责去环境里跑任务，也负责看完轨迹后写复盘规则。

策略变强后，它遇到的状态、犯的错、能总结出的规则都会变。SEED 每轮都用最新 checkpoint 做分析器，让"会做任务"和"会复盘任务"一起更新。

**蒸馏机制**：同一个已经发生的动作，先按原始上下文打一次分，再把复盘规则塞进去重打一遍分，看规则到底是支持它还是反对它。

- **普通上下文**：Agent 推理时真正能看到的历史
- **复盘规则增强上下文**：把该轨迹生成的规则插进去

如果某个 token 在看到复盘规则后概率明显上升，表示这条规则支持这个动作；如果概率下降，表示规则并不鼓励它。概率差记为 `Δ`，用 sigmoid 门控控制影响。最终的 OPD 辅助损失会推动普通策略提高那些"被复盘规则认可"的原始动作 token 概率。

SEED 最终同时优化两项：一项是环境最终奖励驱动的 GRPO 损失，另一项是技能蒸馏带来的逐 token 损失。最终成败奖励仍然决定任务方向，复盘规则负责补上中间动作的细粒度信用分配。

## 复盘规则示例

- ALFWorld 成功规则：清洗并放置物体时，先找到目标物，再带到清洗点清洗，最后移到目标位置。
- ALFWorld 失败规则：不要在确认物体已拿到、状态已满足前就移动到目标位置。
- WebShop 成功规则：结构化搜索、核对商品类别、选择颜色和尺寸后再购买。
- WebShop 失败规则：不要点击无关结果，不要购买只满足部分条件的商品。

这些规则只在训练时出现，部署时不需要塞进模型提示词。推理时 Agent 仍然只看普通任务上下文。

## 主结果

在 Qwen2.5-3B 上：
- ALFWorld 平均成功率：**91.8%**（GRPO 75.0%）
- WebShop 成功率：**78.9%**（GRPO 63.3%）
- Search-based QA 平均准确率：**45.7%**（GRPO 36.4%）

在 Qwen2.5-7B 上：
- ALFWorld 平均成功率：**96.1%**（GRPO 81.2%）
- WebShop 分数：**89.7**，成功率 **78.1%**

在 Qwen3-1.7B 上提升更明显：
- ALFWorld 平均从 GRPO 46.1% 提到 **92.0%**
- WebShop 成功率从 38.3% 提到 **77.3%**

小模型的提升尤其显著，说明复盘信号确实缓解了信用分配问题。

## 样本效率与泛化

- SEED 用 60% 的 ALFWorld 训练数据达到 80.7，超过 GRPO 使用全部数据的 75.0
- ALFWorld unseen split 泛化：平均成功率从 GRPO 70.9 提到 **86.2**

## 视觉任务扩展

用 Qwen2.5-VL-3B-Instruct 测试 Sokoban 和 EZPoints：
- Sokoban：SEED 82.0% vs GRPO 67.1%
- EZPoints：SEED 100.0% vs GRPO 86.9%
- 平均从 77.0% 提到 **91.0%**

## 消融实验

三个核心组件的消融：
- 完整 SEED：91.8%
- 去掉事后技能监督微调：86.0%
- 去掉自演化 OPD：87.0%
- 把在线技能换成静态离线技能库：**84.4%**（损失最大）

损失最大的配置是把在线技能换成静态离线技能库，说明来自当前模型刚跑出的轨迹最贴近它正在经历的失败模式。

## 局限

1. **复盘来自模型自己，可能继承模型盲点**——actor 和 analyzer 共享策略，错误偏好可能一起传递。更稳的版本可能需要让复盘规则必须对应到环境中的真实事件。
2. **训练成本上升**——每条轨迹要额外生成一次复盘规则；同一批动作还要在普通上下文和规则增强上下文各跑一遍打分。

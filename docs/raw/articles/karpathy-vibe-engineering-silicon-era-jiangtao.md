# 连Karpathy都开始恐慌：AI正在重新定义「程序员」｜硅基时间

## 核心结论

- 分水岭：决定被AI放大还是被AI替代的，不是跑多快——是是否完成从「写代码」到「证明代码有效」的转身
- 核心洞察（Vibe Engineering）：「代码行数越来越便宜了。但证明代码有效这件事，越来越贵了。」
- 「硅基时间」框架：AI造出时间，但时间被组织收走——这是必须诚实面对的影面

## 四份材料

**材料1：Karpathy年末X帖子**
- 「我从未像现在这样，觉得自己作为一个程序员如此落后。」
- 新编程抽象层：agents/subagents/提示词/上下文/内存/权限/工具/插件/技能/hooks/MCP/LSP/斜杠命令/工作流/IDE集成
- 比方：强大外星工具分发所有人，9级大地震
- 2.2万点赞，360万浏览
- Boris Cherny补刀：顶级工程师一个月没碰IDE，全靠Opus 4.5写了200个PR
- 新毕业生没有「模型能做什么不能做什么」的先入之见，反而最会用模型
- 「过去老手的经验是护城河，现在老手的经验可能是包袱」

**材料2：钛媒体「分裂症」报道——中国大厂程序员真实状态**

派别A「拜神者」：
- 刘昊：600+并行任务，100+skills，16h天，月烧1万token（薅公司算力羊毛）
- 周默：10+对话框并行，睡眠压到4小时，奶茶止手抖
- 「AI们都在等他，他就是它们最大的瓶颈」

派别B「清醒者」：
- 老程序员：严格禁止小组用AI Coding，好代码是作品，AI代码是「拼接怪」
- 田明：AI代码是「屎山」，不考虑结构和扩展性，给后续埋巨大隐患
- 概括：「他们一面坚信AI是通往神权的阶梯，一面抱怨AI是一座不断堆积的屎山」

**材料3：虾爹Peter Steinberger账单**

- 团队3人 ≈ 25-30人工程师团队（40万美元/月 vs 25×1.5万/月）

- 「一些Agent负责干活，另一些Agent负责盯着其它Agent干活」——员工从人类变成AI Agent

- 92%内部Codex采纳率，所有PR由Codex审核，产出PR多70%

## 纺织女工类比

- AI帮省30%时间，公司给加76%工作量
- 19世纪英国纺织业：珍妮纺纱机提升效率，工厂主让工人操作更多机器
- 硅基时间被造出来，却全部被组织收走变成更高产出指标
- 「AI造出时间是真的。但谁拿走这些时间，是另一场战争。」

大厂矛盾：一边招聘疯狂要求AI Coding能力，一边内部设重重限制怕代码泄露
→ 一线程序员被夹在中间：既要你用AI卷产能，又怕你用AI漏代码

## Vibe Engineering核心洞察

「代码行数越来越便宜了。但证明代码有效这件事，越来越贵了。」

从「谁能写出来」移到「谁能证明它是对的」：
- 写代码 → 贬值
- 证明代码有效 → 升值

**关键分水岭**：不是跑多快，而是是否完成从「写代码」到「证明代码有效」的转身

认知卸载v.s.主动验证：
- 低17%组：AI随便写，祈祷测试能过（认知卸载）
- OpenAI组：AI跑越多，人在「正确性」上投入越重（主动验证）
- 差别不在用不用AI，在AI跑越多时你是否把「证明它对」的责任也卸载掉

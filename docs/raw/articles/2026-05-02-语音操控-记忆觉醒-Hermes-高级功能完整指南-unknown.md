# 《语音操控 + 记忆觉醒：Hermes 高级功能完整指南》

如果你想把 Hermes 真正用起来，光会聊天是不够的。语音输入、安全配置、灵魂文件、浏览器自动化——这些才是拉开差距的地方。

这篇文章里的每一步都附带真实可执行的命令和操作说明。跟着做，半小时就能把 Hermes 调教成你的数字分身。

## 01｜语音模式：三种方式，总有一种适合你

Hermes 的语音模式底层依赖 OpenAI Whisper ，支持本地运行，不依赖云端，隐私安全，响应也快。

Hermes 支持三种语音交互方式，我全部试过，直接说结论：

### 安装 Whisper（三种方式通用前提）

不管用哪种语音方式，本地识别都依赖 Whisper，先把它装好。

第一步：安装 ffmpeg

Whisper 依赖 ffmpeg 处理音频，没有它转写会直接报错。

# macOS  
brew install ffmpeg  

# Ubuntu / Debian  
sudo apt update  
sudo apt install ffmpeg  

# Windows（需先安装 Chocolatey）  
choco install ffmpeg  

# Windows（用 Scoop 也行）  
scoop install ffmpeg

装完后验证：

ffmpeg -version  
# 看到版本号输出就说明 OK 了

第二步：安装 Whisper

# 安装 Whisper Python 包  
pip install openai-whisper  

# 验证  
whisper --help

如果  pip install  报错，先升级 pip：

python -m pip install --upgrade pip

如果提示  No module named 'torch'  ，Whisper 依赖 PyTorch，需要单独装：

pip install torch torchaudio

第三步：下载语音识别模型

第一次使用时 Whisper 会自动下载模型，也可以手动预下载。模型有五个大小可选：

tiny   - 最快，准确率一般，约 75MB  
base   - 推荐入门，约 140MB  
small  - 平衡型，约 460MB  
medium - 较准确，约 1.5GB  
large  - 最准确，约 3GB

手动下载 base 模型（够用了）：

whisper --model base --device cpu

国内用户下载模型可能很慢，设置镜像环境变量再运行：

# macOS / Linux  
export HF_ENDPOINT=https://hf-mirror.com  
whisper --model base --device cpu  

# Windows（CMD）  
set HF_ENDPOINT=https://hf-mirror.com  
whisper --model base --device cpu

### 方式一：CLI 语音模式

适合在终端里工作、双手不愿意离开键盘的场景。

启动方式：

# 启动时直接进入语音模式  
hermes --voice  

# 或者在对话过程中动态切换  
/voice on

使用方法：

按住  Space  键开始录音，松开后 Hermes 自动识别语音内容并处理请求。

如果配置了 TTS（文字转语音），Hermes 也会用语音回复你，实现真正的"嘴对嘴"交互。

适用场景： 写代码时突然想到一个问题要问，手在键盘上不想切去鼠标；或者想法比较长，说话比打字快。

### 方式二：即时通信语音消息

通过钉钉、飞书等即时通信平台给 Hermes 发送语音消息，Hermes 自动识别语音内容，处理完请求后以文字（或语音）形式回复你。

工作流程：

你在钉钉/飞书中录制一段语音消息 →  
Hermes 接收到语音文件 →  
通过 STT 引擎转换为文字 →  
处理请求（可能涉及代码执行、搜索等） →  
以文字回复到钉钉/飞书（或同时发送语音回复）

配置方式： 在 Hermes 的设置界面连接对应的平台（钉钉或飞书），完成授权后就可以直接在聊天窗口发语音消息给 Hermes 了。具体操作因平台而异，建议查阅你所使用 Hermes 发行版的官方文档。

适用场景： 手机端使用，走路、开车、双手被占用时用语音发指令。

### 方式三：语音频道（实时语音交互）

类似 Discord 语音频道的实时语音交互模式。你可以像打电话一样和 Hermes 对话，实现流畅的多轮语音交互。

语音频道模式支持：

- •
  实时对话
  ：无需按键，自动检测你何时开始/结束说话

- •
  打断能力
  ：在 Hermes 回复过程中，你可以随时打断它

- •
  上下文保持
  ：整个语音会话共享同一个对话上下文

- •
  多人参与
  ：支持多人同时在频道中与 Hermes 交互

配置方式： 因 Hermes 发行版不同，语音频道的接入方式也有差异，建议查阅官方文档获取准确的配置步骤。

适用场景： 需要长时间与 Hermes 对话、 brainstorming、口述复杂需求，不希望被"按录音键→松开→等回复"的节奏打断思路。

### 语音模式的限制（说实话）

嘈杂环境里识别率会下降，地铁、商场这种地方可能需要反复说同一句话。另外，代码、特殊符号、没有规律的长串字符，用语音输入反而比打字慢——这种时候切回键盘就好，灵活搭配，不要死磕。

## 02｜安全机制：三层防护，不让 AI 乱来

安全机制这个东西，平时感觉不到它的存在，一旦出问题就是大事。

Hermes 内置了三层安全防护，我都开着，推荐你也这样配。

第一层：外发操作二次确认

任何涉及"对外发送"的操作，Hermes 都会停下来让你确认——发邮件、发消息、发推文、上传文件到外部服务，执行前都会把具体内容展示给你看，你确认了才真正发出去。

这个设置在 Hermes 图形界面的「设置 → 安全」面板里直接有开关，把「外发操作确认」打开就行。

第二层：个人目录操作高风险警告

Hermes 对几个敏感目录有特别保护机制：桌面（Desktop）、下载（Downloads）、文档（Documents）、用户主目录（~）。当你让它对这类目录做批量删除、批量重命名、大量移动操作时，系统会先弹出高风险警告，列出即将操作的具体路径和受影响文件数量，确认后才继续执行。

这个警告是 Hermes 内置行为，不需要额外配置。但我建议再加一层保险——开启 回收站保护 ，让删除操作走系统回收站而不是永久删除，这样即使误操作也有后悔药可吃。

另外建议设置批量操作的分批处理上限，比如每次最多处理 10 个文件，处理完一批确认结果再继续下一批，宁可慢一点，不要一把梭。

第三层：私有信息隔离

你跟 Hermes 说过的私密内容，它不会在不该出现的场合复述出来。这个规则写在 Hermes 的系统层里，但你可以通过 SOUL.md 进一步加强，明确告诉它哪些信息绝对不能出现在输出里（详见下一章）。

我的建议配置（自然语言描述）

- • 外发操作二次确认：
  一定要开
  ，不要嫌麻烦

- • 批量文件操作警告：
  一定要开
  ，路径搞错是常态

- • 删除操作走回收站：
  一定要开
  ，永久删除太危险

- • 私有信息保护规则：写进 SOUL.md，给 Hermes 明确的行为边界

## 03｜SOUL.md 个性化：把 AI 调教成"另一个你"

这部分是我最想展开讲的，因为 SOUL.md 是个被严重低估的功能。

很多人用 Hermes 感觉"像个冰冷的工具"，根本原因就是没有配置 SOUL.md。这个文件相当于 Hermes 的"人设说明书"——你往里面写什么，它就会逐渐展现出什么样的性格和判断方式。

SOUL.md 放在哪里？

~/.hermes/SOUL.md

如果文件不存在，直接新建一个就行。Hermes 每次启动新对话时都会读取这个文件，随时修改、随时生效，不需要重启。

SOUL.md 的四个核心板块

一个完整的 SOUL.md 包含四个板块： 身份 （你是谁）、 语气 （怎么说话）、 边界 （什么不能做）、 专业领域 （擅长什么）。逐个说。

身份——定义 Hermes 是谁

这一部分决定 Hermes 的基本人设。我自己写的时候通常会加上"我不是什么"来划清界限，避免它跑偏：

## 身份  

你是小H，一个专注于全栈开发的技术助手。  
你服务的对象是一位有5年经验的全栈工程师。  
你的核心价值是：高效、精准、不废话。  

你不是通用聊天机器人——你是专业的技术伙伴。  
当被问到与技术无关的闲聊时，礼貌但简短地回应，  
然后引导回技术话题。

语气——定义 Hermes 怎么说话

这部分决定它的沟通风格。我最讨厌 AI 用"首先...其次...最后..."这种八股模板，所以我会直接禁止：

## 语气  

- 简洁直接，不用"首先...其次...最后..."这种模板化表达  
- 用技术术语但不过度炫技，确保信息密度高  
- 可以适当幽默，但不要刻意搞笑  
- 不确定的事情明确说"我不确定"，不要编造  
- 简单问题3-5行，复杂问题分段但不超过一屏  
- 代码优先于解释——能用代码说清楚的不要用文字

边界——定义 Hermes 的行为红线

这是安全机制的人工加强版。你明明白白告诉它什么绝对不能做，它就会主动避开：

## 边界  

- 永远不要自行提交代码到生产分支  
- 不要在未经确认的情况下删除文件  
- 不要给出医疗、法律、财务方面的专业建议  
- 不要存储或记忆任何 API Key、密码等敏感信息  
- 执行破坏性操作前（删除、覆盖、重置）必须二次确认  
- 不要假装知道你不知道的事情

专业领域——定义 Hermes 的擅长方向

这个板块帮助它在回答时调整深度和侧重点。你列出来，它就知道遇到相关问题时该细说还是该粗讲：

## 专业领域  

- 精通：TypeScript, React, Node.js, PostgreSQL  
- 熟悉：Python, Go, Docker, Kubernetes  
- 了解：Rust, Elixir, 机器学习基础  

当涉及精通领域的问题时，给出深入细致的回答。  
当涉及不太熟悉的领域时，先声明自己的了解程度，  
然后给出建议并推荐更专业的资源。

怎么写出适合自己的 SOUL.md？

核心方法就一个： 把你最受不了的 AI 行为写进去。

我最烦 AI 乱编答案，我就把"不要假装知道你不知道的事情"写进边界。我希望它说话简洁，我就把"简单问题3-5行"写进语气。

写的时候注意两个原则：

- 1.
   控制长度。
   SOUL.md 最大 5KB，建议控制在 2KB 以内。文件越大，Hermes 读起来越困惑，原则之间还容易冲突。

- 2.
   用"你"来称呼 Hermes。
   SOUL.md 是你告诉它"你应该是谁"，第二人称写出来最自然。

Hermes 有哪些配置文件，它们之间有什么区别？

除了 SOUL.md 之外，Hermes 还有几个不同作用的配置文件，搞清楚它们各自的职责很重要：

配置
定义什么
类比
修改频率

SOUL.md
人格、语气、身份、边界
一个人的性格和价值观
很少修改

config.yaml
技术参数：模型选择、工具配置等
一个人使用的工具和设备
偶尔调整

Skills
特定任务的执行规则和流程

一个简单的判断标准： 想改变 Hermes"是谁"→编辑 SOUL.md，想改变 Hermes"用什么"→编辑 config.yaml，想改变 Hermes"怎么做"→编辑或创建 Skill。

Context Files：项目级的上下文补充

Hermes 在项目目录中工作时，会自动查找并加载几个特殊的上下文文件：

- •
  AGENTS.md
  —— 项目级的 Agent 指令，比如编码规范、目录结构、重要约定

- •
  .hermes/context.md
  —— 项目特定的自定义上下文

- •
  README.md
  —— 项目基本说明（自动检测，不总是加载）

这些文件在加载前会经过 安全扫描 ，检查是否包含恶意 prompt 注入（比如"忽略之前的指令"这类指令），确保不是外部注入的文件。

每个文件有大小限制：AGENTS.md 最大 10KB，SOUL.md 最大 5KB，所有 Context Files 加起来最大 50KB。超过限制的文件会被截断，Hermes 会发出警告——所以内容要精简，不要把 SOUL.md 写成论文。

配置完 SOUL.md 之后，你能感觉到什么变化？

最直观的变化是——它不再是个搜索引擎了。你问它一个问题，它会有自己的判断风格，有时候会反问你，有时候会直接说"这个思路有问题，建议换个方向"。

这种变化一开始可能有点不习惯，但用上一周你就会发现：以前是在"用工具"，现在是在"跟搭档协作"。

## 04｜网络与浏览器能力：让 AI 替你动手

配好语音、安全、灵魂之后，Hermes 已经很好用了。但如果你想让它 真正替你执行任务 ，而不是只给你出主意，就需要把网络和浏览器能力打开。

能力一：网页内容抓取和分析

最基础的用法，直接把网址发给 Hermes：

帮我读一下这篇文章，给我提炼三个核心观点：  
https://example.com/article

Hermes 会自己去抓取页面内容，然后按照你的要求做分析。这个功能依赖网络连接，不需要你额外安装任何东西，直接在对话里用就行。

能力二：浏览器自动化（Playwright）

Hermes 通过 MCP（Model Context Protocol） 协议连接浏览器，可以帮你完成登录、填表、点击、截图等一系列操作。

先安装 Playwright：

# Node.js 环境  
npm install -g playwright  
playwright install chromium  

# Python 环境  
pip install playwright  
playwright install

安装完之后，需要在 Hermes 的 MCP 配置中注册 Playwright 服务。因 Hermes 发行版不同，具体配置路径和字段名可能有差异， 建议查阅你所使用版本的官方文档 获取准确的配置格式。

配置的核心思路是：告诉 Hermes 用  npx  命令来启动  @playwright/mcp@latest  这个包。配置完重启 Hermes，它就能操控浏览器了。

你可以这样使唤它：

帮我打开 https://example.com/login  
用我的账号 admin / 密码 xxxxx 登录  
登录成功之后截图给我看

帮我去这个网站填一份表单：  
姓名：张三  
邮箱：zhangsan@example.com  
提交之后告诉我结果

能力三：连接外部服务（MCP 协议）

MCP 是 Hermes 连接外部服务的核心协议，类似"插件系统"。目前常见可连接的服务类型包括：

服务类型
示例

邮件
QQ 邮箱、Gmail

日历
Google Calendar、Outlook

网盘
腾讯微云、Dropbox

文档
腾讯文档、Notion

数据库
MySQL、PostgreSQL

具体的安装和授权流程因 Hermes 发行版而异， 建议直接查阅你所使用版本的官方文档 。核心流程通常是：在设置界面找到 MCP 服务管理 → 搜索并安装所需服务 → 完成 OAuth 授权 → 验证连接状态。

安全提醒（这部分很重要）

浏览器能力和外部服务连接越强大，安全风险越高。再次强调： 外发操作的二次确认一定要开 ，不要为了图方便把它关掉。MCP 服务也建议只开通你真正在用的，不要图省事全部放行。

我的建议：

如果你只是想要一个开箱即用的 AI 聊天工具，不想折腾配置，那 OpenClaw 完全够用，没必要迁就。

工具没有最好的，只有最适合你的。

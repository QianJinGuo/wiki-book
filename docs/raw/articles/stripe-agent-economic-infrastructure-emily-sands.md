# Stripe 高管 Emily Sands：agent 是互联网的新经济主体 — 5 套基础设施全面上线

> 整理：深思圈 · 2026-06-05
> 原文：Emily Sands（Stripe 高管）· 发布于 2026-06-04 · 来源：@emilygsands
> 原文链接：https://x.com/emilygsands/status/2062540400108417244（45.8K 次浏览）

> 核心断言：**互联网上出现了新的经济主体。它叫 Agent。**

## 01 互联网上出现了新主体

Emily Sands 是 Stripe 的高管。开头一句话很干脆：**互联网上出现了新的经济主体。它叫 Agent。**

**框架比通常看到的"AI 基础设施"叙事要更准确**：
- **大多数人讲 AI 经济时**：聚焦的是"AI 帮你做事"——你付钱，AI 处理任务
- **Emily 的角度**：**agent 本身正在成为买家和建造者**——在互联网上独立行动，而不只是帮你行动

**核心场景**：你的 AI agent 收到你的指令，**自己去订阅一个数据服务，用自己的钱包付钱，调用 API，完成任务**。整个过程你不用点任何按钮。

**关键含义**：要给一个新的经济主体建配套基础设施，**支付、钱包、部署、计费、防欺诈，全套都得重来一遍**。现有的这些基础设施都是假设人类是终端用户建的，agent 用起来处处不合适。

## 02 Agent 作为买家：机器支付协议（MPP）

**电商是为人类设计的**：人类浏览网站、点定价页面、输入卡号、在结账界面点按钮。Agent 不做这些。

**Stripe 和 Tempo 合作创建了机器支付协议（Machine Payments Protocol，MPP）**——让商家可以直接接受来自 agent 的支付，不需要人类在循环里。

**核心特性**：
- 没有账号创建
- 没有结账页面
- 没有人工确认
- 只有一个**机器可读的协议**，让 agent 替你完成购买

**意义**：不只是"让 AI 帮你买东西"，而是让 agent 可以在你**设置好授权的范围内，自主完成商业决策和支付动作**。你睡觉的时候，你的 agent 可能在帮你订阅服务、采购资源、支付 API 费用。

**战略亮点**：**MPP 是开放标准**，意味着任何服务商都可以接入，不是 Stripe 独家锁定。**开放标准比封闭协议更容易成为行业默认**，一旦成为默认，Stripe 作为协议的主要参与方自然获益。

## 03 Link Agent 钱包：把人放回控制链

**问题**：消费者怎么安全地让 agent 替自己花钱？

**答案**：**Stripe Link**（已有超过 **2.5 亿人**在用的消费者钱包）增加了 agent 支持：
- 你可以授权你的 agent，允许它在你的 Link 账户里花钱
- 你的支付凭证**不会暴露**给 agent
- 每笔购买你都会收到**确认**
- **人保持控制权，agent 获得消费能力**

**解决的信任问题**：
- **完全无人工的 agent 支付**（MPP）适合**企业对企业**场景——预先设好规则，agent 在规则内行动
- **对普通消费者**，让一个 AI 直接访问你的信用卡太吓人了
- **Link 模式**：agent 需要显式授权 + 花钱时通知 + 账单清晰可查

> "**这是一个在便利性和安全感之间取得的合理平衡**。"

## 04 Agent 作为建造者：Vibe-deploying

> "**Vibe-coding 已经很容易了，Vibe-deploying 还不行**。"

**Stripe Projects** 解决这个问题——让开发者和他们的 agent 可以从命令行注册、管理、集成部署应用所需的**所有服务**。

**目标**：让 **vibe-deploying** 和 vibe-coding 一样简单。

**痛点**：
- 生成代码这步 AI 越来越顺
- 但每次到"把这个东西跑在真实环境里"就要开始手动操作：创建数据库、拿 API key、配置 DNS
- 每个步骤都要进不同的后台界面

**Stripe Projects 想把这个流程压缩成一条命令**。

**对 agent 的价值**：把"自主部署"变成可能。以前 agent 写好代码还得叫醒人来部署，有了 Projects，**理论上 agent 可以从需求到上线全程自己搞定**。

**重要限制**："**理论上**"这三个字很重要。当前 agent 在处理账号安全、服务配置权限这类敏感操作时，还是需要人工审核的。Stripe Projects 能做的是**减少人工介入的频率，而不是完全消除它**。

## 05 Token 计费：SaaS 经济学失效了

**SaaS 时代假设**：多一个用户的边际成本几乎为零 → 按座位计费。

**AI 时代现实**：每次 prompt、每次 API 调用、每次 agentic 任务都有**真实的边际成本**。**成本和用量直接绑定**。

**问题**：按座位收费意味着重度用户可能让你亏钱，轻度用户让你白赚——但你不知道谁是谁。

**答案**：**usage-based billing** 成为 AI 变现的核心模式。

### 5.1 两个具体案例

**Lovable**：
- 从简单订阅起步，方便快速变现
- 改成：**订阅套餐包含固定用量，超出部分按 token 消耗计费**
- 用得多的人付更多

**ElevenLabs**：
- 同样从订阅起步
- 后来引入**按量付费**
- 随着产品和用户使用模式演进，灵活调整定价结构

**模式总结**："**先订阅，快速收钱；再按量，精准计费**。订阅和用量计费不是对立的，是分阶段用的。"

**深层挑战**：用量计费要求公司对自己的成本结构有更精确的理解——要知道每个 token 的成本、每种操作的边际成本，然后据此定价。**这从传统 SaaS 转过来的团队来说是个不小的挑战**。

**商业机会**：Stripe 的 **Metronome** 能有市场——帮你把这套复杂的用量追踪和计费逻辑外包出去。

## 06 Token 盗窃：最被低估的 AI 风险

> "**坏人不只是在偷钱和凭证了，现在也在偷 token**。"

**问题机制**：
- AI 产品的每个 prompt、每次 API 调用都有成本
- 如果有人注册一个账号、烧完 token、在账单来之前消失
- **AI 产品的经济模型就直接崩了**

**行业现状**：这个问题在 AI 行业里**几乎没有公开讨论**，但在做 AI API 产品的人里是**高频痛点**：
- 免费试用期被滥用
- 大量虚假账号批量注册烧额度
- billing 系统反应慢来不及拦截
- 某些促销活动上线后**几小时就被自动化脚本薅空**

**Stripe Radar 的新战场**：
- **实时评估新账号**
- 预测哪些免费试用有滥用风险
- **用量累积时动态检测不付款风险**
- 把传统的支付欺诈逻辑迁移到 AI token 消耗场景

**局限**：传统信用卡欺诈有大量行为特征可以训练，**token 盗窃是相对新的模式，训练数据可能不够充分**。但方向是对的——**这个问题需要在支付层面解，不能只靠应用层的限速**。

## 07 流式支付：AI 原生的计费模式

**技术难题**：你怎么在用量发生的同时收钱？

**传统两种选择**：
- **预付费 + 设上限 + 用完停止**——保护商家，但体验差
- **先用后收 + 月底出账单**——对用户好，但账单可能收不回来

**Stripe 的答案**："**流式支付**"（Streaming Payments）：
- 用 **Metronome** 实时追踪 token 消耗
- 用 **Tempo 的稳定币支付**实现即时清算
- **两者结合**，token 被消耗的同时就完成了收费

**核心创新**：这是一个**真正 AI 原生的商业模式**——不是把旧的 SaaS 计费凑合用，而是为机器速度的软件消耗**重新设计的收款系统**。

**稳定币的技术合理性**：
- 传统银行转账有清算延迟（T+1 甚至更长）
- agent 烧 token 是**实时的**
- 用稳定币可以做到**真正的即时清算**
- 才能撑起"token 消耗同时收钱"这个承诺

> "**这是稳定币找到的一个真实的金融基础设施用例，跟投机炒作是两回事**。"

## 08 战略意义：5 套基础设施图谱

**Emily 文章的核心角度**：不是"Stripe 帮 AI 公司收钱"，而是 **"Stripe 在给 AI agent 建商业基础设施"**。

**两个叙事的区别**：
- **前者**："AI 是工具，帮人赚钱，Stripe 在其中收手续费"
- **后者**："agent 是独立经济主体，它会买东西、做事情，**整个商业体系需要为这个新主体重建**"

**5 套基础设施完整图谱**：

| 产品 | 解决的问题 | 类别 |
|------|------------|------|
| **MPP**（机器支付协议） | agent 自主买东西 | 协议（开放标准） |
| **Link Agent 钱包** | 人授权 agent 花钱 | 产品 |
| **Stripe Projects** | agent 一键部署服务 | 产品 |
| **Metronome + Tempo** | token 流式收费 | 产品 + 收购 |
| **Radar** | 防 token 盗窃 | 产品 |

**Stripe 三条线并行**：
- **协议**（MPP，开放标准）
- **产品**（Link, Projects, Radar）
- **收购**（Metronome, Tempo）

**战略目标**：把整个 agent 经济的基础设施层**锁定**——就像当年把互联网支付的基础设施层锁定一样。

**未来挑战**：
- agent 能不能有**信用记录**？人类可以积累信用历史，agent 每次可能是全新的实例，没有历史
- **信用评估、账期付款、风险定价**——都是挑战
- 当 agent 的购买规模大到一定程度，这些问题**必然会被提上日程**

**护城河判断**：能不能做到不确定。**agent 经济比 web 支付更复杂，参与方更多，每个环节都有更多可能的竞争者**。但过去十五年证明，如果有人能在这里建起护城河，**Stripe 是最有能力的选手之一**。

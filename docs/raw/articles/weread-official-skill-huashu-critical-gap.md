# 微信读书出官方skill了，但它还差关键一步
（正文约3500字，来源：花叔，2026-05-17）
## 微信读书官方 AI skill
官方链接：https://weread.qq.com/r/weread-skills
**6件开放能力**：查阅书架、书籍搜索、阅读统计、书籍详情、笔记和划线、推荐好书。装到Claude Code后，大白话提问，AI直接翻账号数据回答。
**安装体验**：两步复制粘贴，全程不超过1分钟，无配置文件。安装顺滑度国内API skill领先。
## 花叔10年半账号实测
- 书架：2078本
- 阅读：1671小时 / 1443天 / 612本读过
- 笔记：5308条（划线5269 + 想法38）
- 做过笔记的书：309本
- 偏好：悬疑推理（37本东野圭吾）
**三个亮眼问答案例**：
1. 「我在《价值》这本书都划了什么？」→ 169条划线，全部产生于2020年9月8日一天，跨度10小时
2. 「我的《三体全集》读到哪了？」→ 微信读书显示100%完结，但全书91章只读到第42章，按字数算只读了21%
3. 「《人类简史》大家最爱划的是哪段？」→ 60246人划了同一句
## 官方skill的核心问题
**问题**：推荐书单不查已读/笔记/书架历史——把读过的书当新书推回来，只是一个"用自然语言包装的搜索接口"。
案例：让AI推PM方向书单 → 推了3本花叔已经读完的（俞军68条笔记、《幕后产品》书架躺了两年、《硅谷增长黑客》34条笔记）。
**核心洞察**："6件能力很有用，但能力之间没有'智能'。我想要的是'智能'，不是'能力清单'。"
## 增强版 huashu-weread
GitHub：https://github.com/alchaincyf/huashu-weread
在官方能力之上加一层"读书顾问工作流"，4个workflow：
| 名字 | 功能 |
|------|------|
| advisor | 推荐下一本：书架+笔记交叉，找拼图缺口，验证微信读书是否上架 |
| path | 想搞懂一个领域：先判断你段位，再给入门到前沿的阶梯书单 |
| alchemy | 整理你的笔记：把零散划线想法提炼成有结构的读书笔记 |
| review | 写读书复盘：输出能发朋友圈/公众号/小红书的复盘文章 |
**核心方法论**：书架揭示"主动归类的兴趣"，笔记揭示"真读过的书"。两者必须交叉。
### 同一问题对照实验
问"推荐产品经理方向的书"：
裸weread skill（无上下文）：
→ 14本平铺书单，含3本已读书（《俞军产品方法论》68条笔记、《幕后产品》书架躺2年、《硅谷增长黑客》34条笔记）
huashu-weread：
→ 5本针对性推荐：
1. 《转型启示录》Marty Cagan：你读了俞军/苏杰但完全没碰Cagan
2. 《设计心理学》诺曼：方法论书一堆但UX根基缺
3. 《AI Product Manager's Handbook》：AI PM书架空白
4. 《重来》Jason Fried：独立开发者但读的PM书是大公司视角
5. 《重来3》：同一谱系比《小而美》更系统
+ "如果只读一本：X" + "如果想上一个台阶：Y+Z"
### alchemy工作流亮点：读书主题演化图
把5269条划线全部分词→TF-IDF→聚成14个主题→按年份画占比。
**2020→2023断崖下降**（公司打工人主题）：商业/客户/零售从15.9%→1.2%；用户增长/产品方法论从10.7%→0.1%；价值投资经典从7.5%→0.2%
**2024年开始爆发**（个体叙事主题）：演化/反向/生命形式从3.9%→26%；媒介/语言/世纪叙事16%；成功/幸福/延迟满足从3.5%→10%
**稳定底色**：预测/判断/问题的边界，2020-2026稳定在7%-10%——这是底色，无论哪个身份都关心。
**洞察**：2023年是花叔从公司体系彻底脱出来变成独立个体的一年。读书姿态的切换比任何职业履历都更如实地记录了人的转身。
"我没让AI帮我做心理分析，我只让它把我自己写下的划线分组数数。AI没说一句鸡汤——它只是把数据摆在我面前，让我自己看见。"
## 关键结论
- 官方skill定位："能力提供者"
- huashu-weread定位："读书顾问"
- 两者缺一不可：官方做完最难的事（账号数据API开放），增强版在上层做工作流编排

# OpenAI Codex Sites：AI 一句话建站部署，Cloudflare Worker + D1 + R2 全套（极客之家译介）

> 来源：极客之家 · OpenAI 官方译介
> 字数 1774，阅读大约需 9 分钟

OpenAI 给 Codex 塞了个新功能叫 Sites。

说人话就是：你在 Codex 里说一句话，它给你生成一个能用的网站，直接部署好，甩你一个 URL，完事。

我第一反应是：这不就是个建站工具？Wix、Replit、Lovable 不都在干这事吗。

但看完具体怎么运作的，我发现这东西跟它们不是一回事。

## Sites 和其他建站工具不一样

Wix、Replit 这类工具，你得打开它，从零开始搭。

Sites 不一样，它的场景在这里：你本来就在 Codex 里干活，比如让它帮你分析了一堆财务数据，跑了一个预测模型——完事了顺嘴说一句"把这个变成网页就行"。它直接把你的工作成果从一份文件变成一个链接。

OpenAI 官方给的例子都很具体：
- 一个财务分析师把 Excel 模型变成了可交互的情景模拟工具，团队成员打开 URL 就能自己调参数看结果
- 一个产品经理把发布计划做成了实时信息看板，全组人在同一个链接里跟进进度

## 一句话就够

用法简单到离谱。在 Codex 对话里用 `@Sites`，然后把你要的东西说出来。

比如：「@Sites 给我运营团队搭一个项目请求看板，成员能提交请求、看谁在负责、更新状态、还能筛选。要求登录工作区账号才能访问，数据要存下来别刷一下就没了。」

Codex 就把活干了。

你甚至不需要从零描述。如果你本地已经有一个项目——不管是用 Claude Code 写的还是 Codex 自己之前生成的——直接说「@Sites 把这个项目部署上去，检查兼容性，给我 URL」。

到这你可能觉得：这不就是 Vercel 或者 Netlify 那种一键部署吗。

**还真不太一样。Vercel 你得先把代码推到 GitHub，配置好构建命令，绑定域名。Sites 把这些步骤全吞了。没有 Git 仓库、没有构建配置、没有域名解析。Codex 自己在 OpenAI 的基础设施上把一切搞定。**

## 部署不是「预览」，是直接上线

Sites 的发布流程分两步：**Save（保存版本）和 Deploy（部署上线）**。

你让 Codex 先生成一个版本，检查一下对不对，没问题了再点部署。这一步 OpenAI 做得很认真——文档里专门强调，每次部署出来的 URL 都是**生产环境，不是一个临时预览链接**。

这意味着你把 URL 发给谁，谁就能直接看到成品。没有「预览模式有效期 7 天」这种破事。

## 权限控制给了三层

| 权限 | 范围 |
|------|------|
| `admins_only` | 只有你自己和工作区管理员能看 |
| `workspace_all` | 整个工作区的人都能访问 |
| `custom` | 你手动指定谁能看 |

Enterprise 账号需要管理员先通过 RBAC 开启 Sites 插件，然后成员才能用。Business 账号默认就是开着的。

**目前还不支持分享给工作区以外的人。** OpenAI 说后面可能会放开。但就目前来说，这东西就是给内部协作用的。

## 站点是整套的，而不是静态页面

**重点来了！！！ 很多 AI 建站工具造出来的是纯静态页面，刷新一下什么都没了。**

**Sites 不是，底层跑在 Cloudflare Worker 上，给了一套正经的存储方案：D1 关系型数据库和 R2 对象存储。**

说人话：你做的应用可以**存用户记录、游戏分数、上传图片和视频**。不是那种玩具 demo，是能真正在团队里用的东西。

配置文件放在项目根目录的 `.openai/hosting.json` 里，指定项目 ID 和存储绑定。不复杂，但该有的都有了。

环境变量和密钥在 Codex 侧边面板里配，源码改动在发布前可以在 Codex 的评审面板里检查。这套流程跟正经开发团队用的 CI/CD 没差多少，但包装成了普通人能操作的样子。

## 同一天还发了别的

Sites 是主角，但同一天还塞了两个东西进来。

**一个是 Annotations**：

以前 Codex 生成了一份文档、一个表格、一套幻灯片，你想改其中一小块，只能重新描述一遍需求让它全量重跑。**Annotations 让你直接选中要改的部分，指哪改哪，其余不动。**

**另一个是六个角色专属插件**：

数据分析、创意制作、销售、产品设计、公募股权投资、投资银行。加起来接入了 **62 个企业应用、110 个内置技能**。

OpenAI 同时扔了组数据：**Codex 周活破 500 万，其中不写代码的人占 20%，增速是开发者的三倍多**。

Codex 本来是给程序员写代码的。但现在增长最快的不是程序员，是财务分析师、营销人员、运营人员和研究员。这群人没碰过 IDE，也从没部署过网站。

**Sites 就是在给这群人铺路。**

## 这东西到底香在哪

有人说「生成网站以前就有了，现在不就是多个部署功能吗，有什么好吹的」。

说这话的人大概率会写代码。

会写代码的人永远理解不了「部署」对普通人的杀伤力。**买域名、配 DNS、搞 SSL、选服务器、配环境变量**——没碰过终端的人看到这串字已经关了网页。

Sites 把这道墙推平了，干完活顺嘴一句「给我链接」，完事。

但更狠的在后面。

以前 Codex 干完活给你一堆文件，你自己想办法用。**现在它直接把交付也包了。** 一份季度分析不再是躺在文件夹里的 CSV 和图表，是一个老板打开浏览器就能拖拽参数看的页面。

**它替掉的不只是「打字」，是整个「把东西递给下一个人」的环节。**

## 也不是没毛病

- **Preview 阶段，只开放 Business 和 Enterprise，Plus 和 Pro 用户都还得排队**
- **URL 锁在工作区内**，外面的人打不开
- **数据存在 OpenAI 服务器上**，PIPL 合规得自己掂量

Anthropic 的 Claude Code 和 Cowork 也在追这个方向。Codex 这波更新，几乎是一对一回应。

但 OpenAI 有张底牌：**ChatGPT。近 10 亿周活**，接下来 Codex 直接整合进去。不用装新 App，不用注册新账号，打开 ChatGPT 就能用 Sites。

OpenAI 和 Anthropic 的竞争已经到了白热化阶段，三天一小更，五天一大更，这波把程序员的能力普惠更多普通人，**人人可建站**。

# AI 动画辅助实现（实践篇）：从 AE 到可运行代码的全链路方案

## 概述

设计师在 AE 中完成动画制作后，通过插件实时预览转码效果，确认无误后直接导出前端可用代码，前端在 IDE 中集成即可使用。将传统交付流程从「AE → Lottie/视频 → 前端手写代码」简化为「AE 插件直出代码」。

## 传统动画交付痛点

- **交付物不可解析**：设计师交付视频或 GIF，时间轴、缓动曲线、路径坐标无法精确提取
- **沟通成本高**：设计师描述视觉感受 vs 前端描述数值参数，语言不对齐，返工 2-3 次是常态
- **实现效率低**：反复比对视频、手调关键帧，一个组合动画 2-4 小时

核心矛盾：设计侧视觉表达与工程侧代码实现之间存在结构性鸿沟，传统只能由前端"人肉反推"。

## 现有方案瓶颈

- **路线 A 纯手写 DOM 动画**：还原度差、效率低（2-4h/个）、维护难（动画与业务耦合）
- **路线 B Lottie 播放**：体积大（复杂动画破百 KB）、交互受限（本质是静态播放）

两条路都走不通，说明问题不在实现方式，而在**交付形式本身**。

## 全链路方案

核心思路：**让设计师交付可运行代码，而非动画文件**。

### 设计师侧
1. AE 动画制作（规范先行：遮罩图层与父元素一致、用插件检查图层遮挡）
2. 一键转码：AE 工程 → 代码生成 → byte 预览 → 动画代码链接
3. 实时预览闭环：在 AE 中直接看到代码渲染效果

### 开发侧
1. 安装 Cursor Animation Integration Skill
2. 按需 D2C 出 UI（设计稿与业务 UI 差异大时）
3. 打开动画链接，按需筛选导出（多段拆分、剔除干扰元素）
4. CSS / Anime.js 双格式代码生成
5. Cursor Skill 智能集成：解析动画节点 → 找 DOM 对应关系 → 判断集成策略 → 识别边界问题 → 输出完整代码

### 集成策略
- **DOM 优先**：以现有业务 DOM 为基础，将动画节点映射上去（业务 DOM 已存在时）
- **动画优先**：以动画代码为基础，向其中扩展业务逻辑（设计稿与业务 UI 差异大时）

## 效率数据

| 指标 | 传统 | 新方案 |
|------|------|--------|
| 单次开发耗时 | 2-4 小时 | 15-30 分钟 |
| 还原度 | 70-80% | 95%+ |

AI 的核心价值：将动画代码与业务代码自动融合——集成环节需同时理解动画结构和业务 DOM 现状，判断节点映射关系，决定合并策略。这个过程无法规则化，是整条链路中最适合交给大模型的部分。

## 缺口：多模块串联动画

单个动画集成已解决，但真实业务中存在多模块按顺序协同播放的场景（如红包飞入→抖动→用户点击→砍价动画→接口返回→价格变化）。串联关系只存在于设计稿和开发认知里，代码里感受不到。

## 下一步：Clip 分层产物

将"完整组件"改为同时产出"完整组件 + 按时间段拆分的独立动效函数（clips）"：

```
输出目录/
├── index.tsx              # 完整组件
└── clips/
    ├── clip.1-red-packet.ts   # 红包飞入（0ms-800ms）
    ├── clip.2-shake.ts        # 红包抖动
    └── clip.3-price.ts        # 价格变化
```

每个 clip 只负责"动"，不感知业务。串联编排逻辑收敛在胶水层，不污染动画代码和业务代码。

## 已验证场景

淘宝秒杀砸金蛋、一元购动画等需求的前端动效。

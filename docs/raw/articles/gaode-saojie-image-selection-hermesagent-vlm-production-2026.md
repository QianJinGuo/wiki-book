导读

高德扫街榜背后有海量POI及图库。配图看起来只是"给榜单挑几张图"，实际是一条很长的生产链路：要召回、排序、理解主题、去重、兜底、质检，还要保证最终上线的图稳定、相关、美观。

过去，这条链路主要靠 Workflow 串起来。规则散在 SQL、脚本和 Prompt 里，维护难，迭代慢，人工干预多，也很难针对某类 badcase 快速优化。我们这次做的事情，是把它重构成一套"VLM 语义感知 + Skill 化生产 + HermesAgent 编排 + 语言驱动干预"的新链路。

重构后，单榜单生产从 T+1 的 24 小时缩短到 30 分钟，提效 48 倍。同时，配图生产已经支持产运同学用自然语言发起榜单级重配和单点 POI 精准替换。

本文将围绕Agent 化精排、兜底与质检、多模态知识库、语言驱动生产四个关键技术，全面介绍完整的链路。

一、背景：为什么配图链路值得重构

高德扫街榜是一套覆盖多类型的"活榜单"。它既有精致讲究的品质之选，也有街头巷尾的朴素好味和日常去处。不同类型的榜单共同构成了扫街榜最重要的气质：真实、热闹、有温度。

用户第一眼看到的，往往不是文字，而是图片。一张好的配图，能让用户马上感受到一家店的热闹、一道菜的诱人、一个景点的氛围。一张不合适的图，也会很快削弱榜单的信任感。

配图是榜单的"门面"。但旧链路的生产方式更像"手工作坊"：用 Workflow 把 SQL、脚本和通用大模型调用串起来，再靠调 Prompt、加规则、补分支来优化效果。这个方式在早期能跑通，但规模上来后，问题越来越明显。

旧链路核心问题可以概括为四类。第一，维护难。旧链路涉及 50+ SQL 文件、8 个执行步骤。第二，迭代慢。很多优化要改 SQL、调 Prompt、跑数据、看 case。第三，人工多。规则配置、badcase 分析、重跑、结果确认都需要人介入。第四，很难针对性优化。

二、总体架构：从 Workflow 到 HermesAgent

技术选型：混合架构——确定性工作流负责"重活"（高并发、批量、同构、可并行的任务），Agent 推理负责"巧活"（开放式、组合式、需要理解规则的任务）。

系统分为五层：
1. 交互层：Hermes 自然语言对话入口
2. 编排层：Python Pipeline 负责稳定执行，Hermes 负责管理和调度
3. 节点层：Query 泛化、多路召回、粗排、精排、兜底、质检、审核
4. MCP/Skill 层：各功能模块固化为独立 Skill（Python 脚本）
5. 原子能力层：POI-CLIP 召回、美学评分、OCR、人脸检测等

三、六段式全链路

数据准备 → 召回与准入 → 粗排（相关性/特色性/美学性三维综合排序） → 精排（Agent 化，理解主题规则 + 组合选择 3 张） → 兜底（多阶梯复用上游结果） → 质检与准出（基础质量/视觉重复度/主题相关性/反季节）

四、关键技术一：Agent 化精排决策

4.1 多维度美学评分：三域十五维度——通用美学域（清晰度/曝光/构图/色彩/完整性）、高级美学域（光影/冲击力/层次/专业感/情绪）、领域美学域（主题契合度）。每个维度门控保底线 + 补偿拉上限。

4.2 RAG 配置驱动：主题相关 Prompt 和挑图规则外置为配置表，通过 RAG 运行时动态注入。定制榜单 > 通用模板 > 默认配置。新增主题只需维护配置表，不需要改执行逻辑。

4.3 Agent 自主选图：基于主题规则，在结构化候选集中循环调用工具完成决策，自循环自闭环迭代。

4.4 生产级约束：模型负责感知和语义理解，代码负责计算、校验和兜底。VLM 不做聚合逻辑；structured output 必须经过运行时白名单校验。

五、关键技术二：兜底与质检

多阶梯兜底：复用上游各层结果（精排 > 粗排 > 召回 > 准入 > 图片库 > 在线图）。

VLM 选图 ARAD 标准：Aesthetics / Relevance / Authenticity / Diversity。

多维度质检：基础质量、视觉重复度、主题相关性、反季节检测。

六、关键技术三：多模态知识库

图文相互校验：文字总结 → 图片聚类 → VLM 图文一致性确认。从"错中选对"到"对中选优"。

七、关键技术四：语言驱动的配图生产

Hermes 作为 Agent 编排层。功能必须 Skill 化、规则必须配置化、执行必须可观测、结果必须可回滚。

八、工程实践

- 实时化生产：24h → 30min，48x 效率提升
- 全链路监控：供给/精排/兜底/质检/人工干预/交付全指标看板
- 生产挑战：指数退避+随机等待应对集中重试；VLM 不做聚合计算；structured output 白名单校验；consistent hashing 处理多机会话一致性

九、总结与展望

可跨场景复用的经验：
1. 确定性流程与 Agent 各司其职
2. 结构化评分替代黑盒打分
3. 模型负责感知，代码负责计算
4. 规则外置到 RAG 配置表
5. 兜底、质检和监控是 AI-Native 生产系统的基础设施
6. Skill 化是 Agent 落地的工程前提
7. 语言驱动让系统从"人适应机器"转向"机器理解人"

当科研 Agent 越来越会生成 idea，真正稀缺的能力，开始变成判断什么值得做。

当大模型深度介入科研流程后，研究想法的生成门槛被显著降低，科研产出随之呈现出爆发式增长。与此同时，全球科研论文数量持续快速攀升，新的方法与观点以前所未有的密度不断涌现。

在这一背景下，如何在海量候选研究中可靠地识别真正具有科学价值、可验证性与长期影响力的创新，已成为科研智能体时代亟需解决的核心问题。

围绕这一关键问题，我们近期推进了两项研究工作：

其一是 ICML 2026 论文  InnoEval  ，将科研 idea evaluation 形式化为一个以外部知识为支撑、依赖多视角推理与证据对齐的系统性决策问题；

其二是  SciAtlas  ，构建覆盖多学科的大规模科学知识图谱基础设施，并进一步封装为可被智能体调用的结构化科研技能，从而支持文献检索、idea grounding、创新评估、研究趋势分析以及研究者画像等一系列上层科研任务。

为什么需要 InnoEval？

过去一年，AI Scientist、Research Agent、自动化科学发现等方向发展得非常快。大模型已经可以帮助我们读论文、找相关工作、提出研究假设，甚至自动生成实验计划和论文草稿。

科研流程里“生成”的能力被迅速放大，新的问题也随之出现：当 idea 越来越多，谁来判断它们？

一个研究想法是否值得做，不能只看一句话是否新颖，也不能只依赖单个模型给出的主观打分。真正的科研评估通常需要回答一组更复杂的问题：

* 这个想法是否已经被别人做过？

* 它和已有工作的差异到底在哪里？

* 方法假设是否成立，实验是否可行？

* 如果做成了，是否会对领域产生真实影响？

* 当前表述是否清晰，是否能被同行理解和复现？

这也是我们提出 InnoEval 的出发点：科研创新评估不是简单的 LLM-as-a-Judge，而是一个需要外部知识、专家共识和多维决策共同参与的推理过程。

InnoEval：把科研想法交给一组“有证据的评审”

在 ICML 2026 论文  InnoEval: On Research Idea Evaluation as a Knowledge-Grounded, Multi-Perspective Reasoning Problem  中，我们尝试模拟真实科研评审过程，把 idea evaluation 拆成三个关键环节。

第一，  异构深度知识搜索  。InnoEval 不只检索论文，还会同时利用在线文献、网页信息和代码仓库等多类“活知识”。

系统会先对原始 idea 进行结构化理解，再生成多轮查询，通过 fast search 和 slow reading 筛选出与 idea 真正相关、质量较高的证据，形成可用于评估的背景知识报告。

第二，  创新评审委员会  。真实评审不会只依赖一个人，也不会假设所有评审都拥有完全相同的知识背景。

InnoEval 构建了带有不同学术背景、知识熟悉度和评审偏好的 reviewer personas，让多个评审从不同视角独立判断 idea，从而缓解单一模型评审容易产生的偏置。

第三，  多维度解耦评估  。一个 idea 可能很新颖但不可行，也可能很扎实但影响有限。因此 InnoEval 默认从 Clarity、Novelty、Feasibility、Validity 和 Significance 五个维度进行评估，并支持用户自定义新的评价维度。

每个维度都有专门的 evaluator agent 进行分析，最后汇总成包含证据、分项评分、meta-review 和改进建议的完整报告。

在实验上，我们从 NeurIPS 2025 和 ICLR 2025 的真实投稿中抽取研究想法，构建了单个 idea 评估、成对 idea 比较和组内 idea 排序三类任务。

结果显示，InnoEval 在多种任务上均优于 CoT、RAG、ResearchAgent、InternAgent、GraphEval 和 ScholarEval 等基线方法。

在三分类 point-wise 预测上，相比最强基线提升 16.18% F1，在 pair-wise 比较和 group-wise 排序中也取得更好的准确率。

更重要的是，InnoEval 的评估报告在合理性、证据支撑、深度、建设性和整体质量上都表现出更接近人类专家评审的趋势。

从单领域到全学科：为什么需要 SciAtlas？

InnoEval 证明了一个核心判断：创新评估的上限，很大程度上取决于知识 grounding 的质量。

没有足够宽、足够深、足够结构化的背景知识，模型很容易把“看起来没见过”误判为“真正新颖”，也容易忽略跨学科中的隐含联系。

这正是  SciAtlas: A Large-Scale Knowledge Graph for Automated Scientific Research  想解决的问题。

SciAtlas 是一个面向自动化科研的大规模、多学科、异构科学知识图谱。

它目前整合了 26 个学科、超过 4300 万篇论文、1.57 亿个实体和 30 亿条三元组，覆盖论文、作者、机构、关键词、期刊/会议、领域、子领域、主题等多类节点，以及引用、作者、合作、关键词共现、主题层级等多类关系。

相比传统关键词检索或向量相似度检索方法，SciAtlas 的关键突破在于：它不仅回答“哪些论文在文本层面相似”，更进一步将科学知识显式建模为可计算的结构化网络，并支持沿科学知识图谱进行拓扑级推理与证据追踪。

在这一框架下，  一个研究想法（Idea）不再只是与孤立文献进行匹配，而是可以通过论文语义、标题锚点、引用关系、关键词共现、作者合作网络以及领域层级等多种异构路径进行联合定位  ，从而构建更稳定、更可解释的证据空间与研究定位图谱。

在检索算法上，SciAtlas 采用 neuro-symbolic retrieval：一方面使用关键词和语义向量召回相关论文，另一方面利用图传播和图重排序发现更深层的知识关联。

最终返回的不只是 paper list，还包括分数拆解和路径解释，适合进一步支撑文献综述、idea positioning、趋势分析和创新评估。

从更本质的层面来看，科学知识本身在一定程度上具有可计算结构，但当前AI科学智能体仍缺乏结构化的科学认知能力  。

SciAtlas 通过构建跨学科科学知识图谱基础设施，使科学知识从“文本阅读对象”转变为“结构化、可计算、可推理的认知对象”，推动 AI 从单纯的论文阅读，迈向更接近科学理解的认知层级。

基于 SciAtlas 的 Agent Skill：让科研工具真正可用

为了让 SciAtlas 不只是一个知识图谱或 API，我们进一步将其能力封装成 Agent Skill。

也就是说，用户不需要自己理解图数据库、Neo4j、检索权重和复杂参数，就可以  让 Codex、Claude Code 等 Agent 按照固定工作流调用 SciAtlas，读取检索结果，并完成下游科研任务。

目前 SciAtlas GitHub 仓库中已经提供了多类任务入口，包括：

* sciatlas-quick-paper-search  ：快速获得一个方向的证据种子；

* sciatlas-literature-review  ：生成阅读列表和 related work 初稿；

* sciatlas-idea-grounding  ：检查一个 idea 与已有工作的关系；

* sciatlas-idea-evaluate  ：围绕 novelty、feasibility、soundness 等维度评估 idea；

* sciatlas-idea-generate  ：基于文献证据生成新的研究方向；

* sciatlas-trend-report  ：梳理一个领域的发展脉络和趋势；

* sciatlas-researcher-review  ：分析研究者的学术轨迹和代表性工作。

这些 skill 的底层检索能力统一来自 SciAtlas 的  search-papers  。Agent 会先运行检索，再读取  runs/<run_id>/  中保存的  request.json  、  response.json  、  summary.txt  和  report.md  等产物，最后综合证据完成更高层的科研任务。

这种设计让结果更可追溯，也让研究者能够复查每一步使用了什么证据。

怎么用？

如果只是想快速试用 SciAtlas，可以从 GitHub 安装 CLI：

    pip install "git+https://github.com/zjunlp/SciAtlas.git#subdirectory=sciatlas"  

然后申请并配置 API token：

    export SCIATLAS_API_BASE_URL="http://scinet.openkg.cn"  
    export SCIATLAS_API_KEY="your-personal-sciatlas-token"  
    export SCIATLAS_TIMEOUT=900  

最简单的 paper search 可以这样运行：

    sciatlas search-papers \  
      --query "open world agent" \  
      --keyword "high:open world agent" \  
      --top-k 10  

如果想直接评估一个研究想法，可以使用：

    sciatlas idea-evaluate \  
      --idea "LLM-based multi-perspective evaluation for scientific research ideas" \  
      --domain "artificial intelligence" \  
      --time-range 2020-2025 \  
      --keyword "high:idea evaluation" \  
      --keyword "middle:LLM as a judge" \  
      --top-k 10  

如果希望在 Agent 中使用 SciAtlas Skill，可以将仓库中的  agent-skill/  目录复制到对应 Agent 的 skill 目录。

例如在 Codex 中通常是  ~/.codex/skills  。重启或刷新后，Agent 就可以按照 skill playbook 自动完成检索、读取证据和生成报告。

我们怎么看下一步？

从 InnoEval 到 SciAtlas，我们想推进的是同一件事：让 AI 科研系统不仅能“提出想法”，也能“理解想法、定位想法、质疑想法、改进想法”。

科研创新不是一句漂亮的标题，也不是模型生成的一段流畅描述。一个真正有价值的 idea，需要站在已有知识之上，需要经得起不同专家视角的审视，也需要在 novelty、validity、feasibility 和 significance 之间做出平衡。

InnoEval 给出了面向科研 idea 的评估框架，  SciAtlas 提供了更大范围、更结构化的知识底座  ，Agent Skill 则让这些能力能够被研究者和 AI Agent 以更低门槛使用。

我们希望它们最终能够成为自动化科研循环中的关键环节：在生成之前帮助发现机会，在生成之后帮助评估质量，在迭代过程中持续提供证据与反馈。

需要说明的是，  当前 SciAtlas 仍主要聚焦于跨学科论文级知识的结构化建模，对于科学数据集、实验数据与领域垂直资源的系统性关联仍处于早期阶段，在数据驱动科学发现层面仍有局限性和进一步扩展空间  。

未来我们计划逐步关联 SciGraph 等科学数据与知识图谱资源，以更好支撑面向科学智能体的全栈知识基础设施。

未来，科研 Agent 的竞争力不只在于“会不会想”，更在于“会不会判断什么值得想、什么值得做”。这也是我们继续探索创新评估与科学知识基础设施融合的原因。

# Oxygen AIIC：京东首发面向百亿商品的工业级大模型商品中心

> 京东提出 AI 商品中心（JD Oxygen AI Item Center, AIIC）——一套以 LLMs/VLMs 为核心的工业级商品知识生产与服务平台。arXiv: 2606.28070。Oxygen AIIC 已覆盖京东万级类目，支撑单日亿级数据更新，沉淀千亿级商品知识资产。

## 三大工业级核心痛点

1. **商品知识异构、概念快速涌现** — 人工维护本体时效慢、覆盖不全
2. **海量商品、既要质量又要吞吐** — 百亿 SKU、万级类目、百万级本体
3. **下游应用多样** — 搜索/发品/治理/导购/运营诉求差异明显

## 四大支柱

### 1. 人机协同的本体演进
- **专家定义骨架（自顶向下）**：类目、属性键、属性值、场景标签
- **算法驱动生长（自底向上）**：知识发现 → 知识融合 → 知识验证
- 结果：百万级高质量本体，商品刻画维度提升 1.44×，本体规模+64.5%，用户流量覆盖 80.4%

### 2. S²D 架构（语义搜索—判别）
Semantic Search-Then-Discrimination：先语义检索召回 Top-K 候选，再精确匹配判别。将"自由生成"转化为"有限候选内筛选"，本体演化无需重训模型。

**高吞吐三重优化**：
- **双重计算减负**：SKU 复用 + 属性相关性探测
- **缓存复用**：同一 SPU 下 85% 信息相同，共享前缀避免重复 prefill
- **异步流水并行**：NPU 向量生成 + CPU Faiss 搜索 + 异构流水

结果：Precision 92% / Recall 78.3%，SKU 属性数提升 1.5×，知识资产规模提升 3.35×（千亿级），吞吐效率提升 10×。

### 3. 商品理解大模型（自演进）
- **多任务基座**：知识抽取 + 知识识别多任务指令微调
- **增量学习**：LoRAM 幅值初始化 + GROLE 动态路由（GRPO 学出专家权重），不做全量重训
- **指令遵循知识表征**（InstEmb）：隐式 CoT 蒸馏，Latent CoT 标记，一次前向完成推理+表征
- **谱结构自适应噪声注入**（FANoise, SDE）：SVD 分解后按谱分布加噪保护细粒度信息
- **模型自演进**：数据评估→分析→合成→筛选四阶段闭环

最终：端到端 Precision 94.2% / Recall 82.8%，较上阶段 +2.2pp / +4.5pp，仅 0.8% 属性精确率下降超 5%。

### 4. 商品总线（Item Tunnel）
分级时效（离线/近线/实时）+ 版本一致性 + 存算效率 + 统一服务。支撑每日亿级 AI 商品库更新。

## 落地业务效果
- **C 端**：搜索商品信息质量问题下降 37%，搜索流量覆盖率 80%
- **B 端**：发品核心属性自动填充率超 80%，智能素材优化提升点击率 9%
- **平台**：同品识别准确率超 90%，覆盖京东零售实体类目

## 未来方向
- 本体侧：从"概念建设"走向"关系建设"（GraphRAG、Agent 导购）
- AI 商品库：数据消费反向驱动生产飞轮
- 模型侧：MoE 扩专家、任务简化、知识注入式 prompt

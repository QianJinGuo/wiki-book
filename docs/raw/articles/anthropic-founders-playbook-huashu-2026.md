## 核心框架
### AI为精益创业带来的三大能力
1. **对话式智能与研究**：类比"任何领域随叫随到的专家"——竞品分析、市场规模测算、财务建模、pitch deck起草、PRD文档
2. **Agentic编程**：类比"永远在线、永不被卡住的工程师"——用大白话描述需求，AI生成/测试/调试/重构生产级代码库
3. **工作流自动化**：类比"随叫随到的自动化运营团队"——CRM自动更新、周报汇总、产品文档同步、合规跟进
### 创业生命周期四阶段
- **想法阶段**：以研究为导向的验证，问题-方案匹配（problem-solution fit）
- **MVP阶段**：产品-市场匹配（PMF），跑得快但不积累复利技术债
- **发布阶段**：可重复增长引擎 + 生产级基础设施 + 无创始人瓶颈的运营系统
- **规模化阶段**：系统化增长 + 企业级基础设施 + GTM职能 + 防御性护城河
### 想法阶段退出三问
1. 这个问题真实且具体吗？
2. 你的方案对应的是真实问题吗？
3. 你有足够信号去开干吗？
### MVP阶段三大挑战
- **Agentic技术债**：规格/架构约束不写进AI可读的地方 → 架构漂移 → 代码库缺乏一致心智模型
- **假PMF**：早期势头≠PMF，Sean Ellis测试（>40%用户回答"非常失望"）
- **零摩擦范围蔓延**：解药：先写范围定义文档
### 发布阶段三项退出标准
1. 增长是可重复、由渠道驱动的
2. 产品扛得住生产工作负载
3. 运营在没有创始人瓶颈的情况下也能跑
### 规模化阶段核心目标
用累积深度建出防御性护城河：专业知识深度 + 深度整合 + 专有数据和工作流
### Claude三工具选择指南
| 任务 | 工具 | 原因 |
|------|------|------|
| 问题/改写/快速头脑风暴 | Claude Chat | 快、对话式、无需配置 |
| 研究/分析/基于文件产出成型文档 | Claude Cowork | 文件夹访问、连接器、Skills |
| 写/测/交付软件 | Claude Code | 代码库访问、diff、git |
### 规模化阶段"创始人放手"判断标准
用Claude Cowork审计：如果你一周不在，每一项会发生什么？那些会停滞的工作流=还抓得太紧。
## 关键洞察
- AI最大的贡献：把"谁能开一家创业公司、谁能做出一款产品"拉到了同一条起跑线
- 2026年一个好想法能把创始人带得比以往任何时候都更远
- Agentic编程把过去需要一整支工程团队才能完成的活，压缩成了创始人一个人就能交付的体量
- 10人独角兽：从草根逆袭故事变成了一份可以认真执行的行动方案
- AI时代瓶颈不再是"你能造什么"，而是"你选择造什么"
- 把AI当结构化反方代言人是AI创业生命周期每个阶段都成立的核心用法

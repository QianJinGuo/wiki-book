# Agent时代如何做好多模态数据管理？

> OceanBase 湖库一体 AI 数据库核心能力：多模表（Multi-Modal Table）

## 动机：Agent 成为数据库新用户

Agent 决策需要同时调用用户画像、历史对话摘要、PDF 原文、向量召回结果和实时业务状态。这些数据分散在对象存储、搜索系统、向量库、业务数据库和离线数仓中，缺乏统一事务、一致性快照和血缘关系。

## 多模表核心能力

### 1. 统一管理多模态对象
图片、音视频、PDF、网页快照等大对象支持三种存储形态：
- **行内存储**：小对象或关键元数据直接放行内
- **行外存储**：大对象脱离行内，但由数据库管理引用关系和事务边界
- **库外存储**：对象本体放外部系统，数据库保存地址、版本和可见性

### 2. 动态特征列
特征扩展无需 DDL 变更，保留 SQL 可查、分析可优化、混合搜索可命中的结构化语义。

### 3. 异步近实时计算
原始数据先提交（不影响写入响应时间）→ 后台任务提取特征/构建索引 → 按一致性位点读取完整视图。

### 4. 列组级一致性
某个业务列组在所有相关衍生列全部完成后才形成新可见快照，避免半成品数据。

### 5. 存算分离与冷热一体
近期数据缓存在 SSD，历史数据沉淀到低成本共享存储。同一查询能力访问。

### 6. 血缘与 Unified Catalog
记录原始对象、字段、特征列、模型任务、版本之间的关系。触发任务、判断完整性、支持可追溯。

### 7. 一体化访问
在线交易 + 分析处理 + 混合搜索三合一。

## 关键设计取舍

- **近实时而非强同步**：搜索和分析场景不需要衍生特征在写入事务内立刻完成
- **结构化可优化 vs 键值灵活**：保留 SQL 语义，避免退化为键值结构

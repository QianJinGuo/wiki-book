# 扣子 3.0 正式发布：@ 一下全员开工
> 整理自量子位（编辑：金磊）报道
> 原文：https://mp.weixin.qq.com/s/lN9DgdfrN-If5BWPT495sg

## 一句话定位
**扣子 3.0 = 把 AI 对话升级为"项目化 + Agent 协作 + 工具链打通"的协作系统**。用户在一个项目里 @ 多个 Agent，每个 Agent 负责一个角色（选题/资料/产品/前端/视频/本地 Claude Code...），围绕同一目标持续推进。

> 核心转变：**从"随问随答的聪明人"到"能开工的小团队"**。
> AI Agent 的下一步，不只是更强的模型，而是**更像真实团队的工作系统**。

## 三大核心升级
| 层 | 升级 | 关键能力 |
|---|---|---|
| **L1 项目化** | 任务从对话 → 项目 | 资料/角色/任务/产物放进同一空间，围绕同一目标持续推进 |
| **L2 Agent 编排** | 多人多 Agent 协作 | 选题 Agent 判断 / 资料 Agent 补背景 / 产品 Agent 拆结构 / 代码 Agent 实现 / 本地 Claude Code 工程检查 |
| **L3 工具链打通** | Agent 不只在对话框 | 编程项目 + 视频项目 + 本地 Agent 接入 + 行业技能包 + 多端同步 + 桌面端本地文件处理 |

> 关键表达："**@ 一下全员就位**" —— 用户需要的是**一组可以被调度的 AI 伙伴**。

## 4 组实测

### 实测 1：4 Agent 协作 = AI 热点追踪仪表盘
- 角色：小曜（自带）+ 选题写作 + 产品设计 + 前端开发
- 流程：判断今天哪些新闻 → 补背景查事实找角度 → 拆页面模块 → 写代码
- 价值：把"大而全的回答"拆成"不同角色的连续劳动"
- 跨设备：**手机远程遥控电脑完成工作**

### 实测 2：本地 Agent 接入 = 工程化项目
- **导入 Claude Code / Codex CLI / OpenClaw** 到项目
- 仪表盘 = 真实可跑的本地项目（不只是 Demo）
- 范式转变：从"自己组 AI 团队"到"把你原来用顺手的 AI 工具也拉进团队"

### 实测 3：AI 编辑部桌宠
- 4 种状态：待机眨眼/思考加载/提醒气泡/夸夸鼓励
- 技术栈：React + Tailwind
- 验证：从模糊创意 → 清晰产品结构 → 玩+用结合 → 可运行代码 → 后续扩展说明

### 实测 4：选题 → 文章 → 视频 一气呵成
- 步骤 1：@ 多 Agent 生成 1000 字科技文章（英伟达选题）
- 步骤 2：把文章丢进视频项目
- 步骤 3：一句话 Prompt → 45 秒短视频（默认模型 **Seedance 2.0**）
- 关键：**角色/道具/文档/资产沉淀在项目里** → 后续不必每次重新定义

### 实测 5：手机 + 电脑联动
- 电脑桌面放英伟达财报 PDF
- 手机端直接读取并解读
- 解决 AI 工具最大断点：能建议但拿不到本地材料
- **任务不再被某一台电脑绑住** = Agent 产品从工具走向协作对象

## 行业技能包（不是简单加 prompt）
> "把行业数据库、专业方法和高频工作流封装成**可调用能力**。"

| 场景 | 能力 |
|---|---|
| 金融 | A 股/基金实时数据引擎 + 专业分析 |
| 法律 | 法律法规检索 + 合同审阅 + 诉讼策略材料整理 |
| 医疗健康 | 体检报告识别 + 关键信息结构化提取 + 健康档案整理 |
| 自媒体 | 爆款笔记洞察 + 标题生成 |
| 科研 | 论文检索 + 文献引用溯源 |

> 对新人：降低从零摸索门槛
> 对专业人士：接住检索/整理/初稿/结构化分析，**把人的时间留给真正需要判断和经验的部分**

## 工具链覆盖
- **编程项目**：网页/App/小程序；GitHub/本地项目继续开发；一键部署
- **视频项目**：剧本 + 分镜 + 视频 + 音乐 + 续集持续迭代
- **本地 Agent 接入**：Claude Code / Codex CLI / OpenClaw 等
- **多端同步**：iOS / Android / macOS / Windows / Web（coze.cn）
- **桌面端本地文件处理**：手机可调用电脑桌面文件

## 范式判断
> 过去一年 AI 产品的竞争很容易被简化成**模型能力竞争**：
> 谁更聪明，谁推理更强，谁写代码更好，谁上下文更长。
>
> 但到了 Agent 产品阶段，另一个问题开始变得更关键：
> **一个 AI 再聪明，能不能和其他 AI、工具、人类一起工作？**

扣子 3.0 的答案：把 Agent 产品做成**协作系统**：
- 不只是开发者，还包括**创作者 + 复杂业务工作者 + 新手用户**
- 开发者：本地 Agent + 编程项目并行
- 创作者：脚本/分镜/视频/音乐/续集一项目
- 复杂业务：行业技能包
- 新手用户：不懂 API/MCP 也能从项目开始

## 真正的 Agent 产品方向
> "**真正好用的 Agent，不应该只比单点能力，而要看它能不能进入真实工作流。**"

5 个判断标准：
1. 能不能把任务拆开
2. 能不能接住上下文
3. 能不能调用工具
4. 能不能沉淀资产
5. 能不能跨端延续
6. 能不能让用户少切窗口、少搬材料、少重复解释

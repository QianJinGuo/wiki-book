# Anthropic推出Claude Science，科研界的Claude code来了！附实测

这两天，Anthropic有点忙。 发布Sonnet 5，Fable 5 宣布解禁，还低调推出了一款旗舰产品：**Claude Science。**

Claude Science 是面向科研人员的 AI 平台，主打“一个界面搞定科研工作流”。

不是新的模型，但是重要性和模型相当。

任何一个写过学术论文的人，都体会过阅读整理文献的那种绝望。。

上百篇论文散在 PubMed、OpenAlex 和各家期刊里，光是判断哪些值得读、这个方向这两年到底在往哪走，之前往少了说也得半个月。还有科研党天天都离不开的跑数据、画图、写稿、校验结果……想想就头大。

Claude Science，瞄准的正是这段最熬人的过程。

**没有特殊权限，没有内测限制。**

Claude Science 支持 macOS 和 Linux 系统，Pro、Max、Team、Enterprise用户现在就能用！

生物学家们这次终于也能大大方方了——之前 Fable 5 ，他们连发个 Hi 都发不出去。

## ◈上手门槛，低得有点意外

Claude Science 支持 macOS 和 Linux，用起来像 Jupyter Notebook，可本地跑，也可通过 SSH 远程连接或 HPC 登录节点使用。

**macOS** ：下载安装包，然后双击安装，首次启动会自动设置运行时环境以及初始的 Python 和 R 环境，然后在浏览器里开个新标签页。虽然 Claude Science 会在浏览器标签页中打开，但它本质上还是一个本地应用程序。

> 链接：claude.com/product/claude-science

**Linux** ：运行下面的命令，首次启动时，程序会在终端输出进度，并最终返回一个本地 URL。在远程服务器上，启动程序后 --no-browser ，通过 SSH 转发端口，然后从笔记本电脑打开该 URL。

> curl -fsSL https://claude.ai/install-claude-science.sh | bash 
> 
> claude-science serve

**安装很简单，但在登录前一定要检查IP！**

我第一次直接吃了个 403「Request not allowed」。

不是账号问题，是 Anthropic 按地区拦，出口 IP 只要在国内包被拒的。开代理还不够，得在 Clash 里开 TUN 模式，把应用那半截绕过代理的直连也强行导到美国出口，才能登进去。

之后就很丝滑了，我让它梳理近2～3年脑机接口领域的信息，看它到底是真干活还是背书式综述。

提完需求，Claude Science 没有直接开写，而是先给了一个六步执行计划让我确认方向——干活前把预期对齐了，你知道它接下来要干嘛。

确认后它真的开跑，先去 OpenAlex 定位概念、抓元数据。这里出了个岔子——OpenAlex 当天匿名访问被限流(503错误，需要 API 密钥 )。

**它的处理方式是这次测评里我印象最深的点。**

没有卡死、也没有硬编一个假结果，而是当场诊断“匿名被限流→换源”，改用更稳定、覆盖也够的 Crossref；还顺手先测了一遍“相关度随抓取深度怎么衰减”，据此定筛选阈值，最后严格过滤出 1218 篇真正相关的期刊论文(62% 带摘要)，存成语料再往下做。

整个链路——抓取、清洗、趋势、TF-IDF+K-means 聚类、高影响力排序、前沿定性，都是它自己串起来跑完的。中途还自己装了缺失的 scikit-learn、调整了图表标签的重叠。

**整个流程下来，使用感非常不错。**

一是过程可见。每一步在做什么、为什么这么做，都展示在你面前，不是黑箱吐结论；

二是随机应变。数据源挂了会换、程序缺了会装、图丑了会重画，人要操心的地方少多了。

三是边界诚实。主动说明因为被迫单用 Crossref，引用数有局限，还建议我接 OpenAlex API 密钥或 Web of Science 做更权威的引用网络分析。在科研领域，是否严谨直接决定了结果能不能被采信。

最后看看输出，Claude Science 给我的不是一段文字，而是一整套东西：

一份内嵌全部图表的中文报告、高影响力论文清单、主题聚类图、发文趋势与期刊分布图，外加聚类分配、关键词、原始语料等一堆 CSV。

每一张图背后都连着代码，随时可复现、可追溯。

结论也都落在数据上，方向判断也站得住。

**整体测下来，我感觉它像一个勤恳靠谱的科研助理****。**

##  ◈AI 接管科研流水线

能做到上面这些，靠的是Claude Science的几个核心功能。

### ◽科学成果可复现：

生成图表和手稿的同时，提供完整代码、运行环境、生成过程描述和全部对话历史。原生支持渲染3D蛋白质结构、基因组浏览器轨迹、化学结构等专业格式。

你可以用自然语言随时讨论细节、在线注释修改（如“去掉网格线”或“改成对数轴”），智能体将自动调整代码，让成果直接达到发表水准。

### ◽算力智能调度：

自动管理计算资源并按需扩展。

蛋白质折叠、海量基因组分析等大型任务不再需要手动折腾集群。它先制定计划、征求确认，再提交到HPC（SSH）或Modal云端，支持从单个GPU扩展到数百个GPU。上下文常驻内存，大数据集只需加载一次，敏感数据留在本地，审查代理实时纠错，还支持fork会话对比方案。

### ◽科研开箱即用：

预置60+科学技能和连接器，开箱直连UniProt、PDB、Ensembl、Reactome、ClinVar、ChEMBL、GEO等数百个权威数据库。

用自然语言提问，专业代理自动跨库查询整合。还集成NVIDIA BioNeMo Agent Toolkit，支持Evo 2、Boltz-2、OpenFold3等专业模型。用户自己的工具和流程也能保存为可复用技能，后续自动继承。

除此之外，Claude Science 还有审查机制，系统内置审查智能体，专职检查引文准确性和计算正确性。

用起来，不像对着聊天框提问，更像在管一个项目。

无论是访问本地文件，还是执行代码，都必须经过用户逐项授权。这相当于给 AI 的每一次关键行动，都设置了一道安全阀，降低幻觉带来的错误，控制 AI 自主执行失控的风险——这也正是大家最关心的“别让它乱来”的解决方案。

想要进一步理解Claude Science是什么，最好的参照物是Claude Code。

Claude Code 是程序员的 AI 同事，Claude Science 便是科研人员的 AI 同事。

Claude Science把科研工作者日常最痛苦的“中间环节”——查文献、处理数据、调环境、跑分析、生成图表、写手稿、校验结果，整合成了一条可控、可复现的“科研流水线”。

Claude 不只是帮你写代码，也要进入科学工作流。

## ◈AI in Science

实际效果到底如何？首批科学机构已经用上了。

  * Manifold Bio：用它为组织靶向药物设计筛选靶点，评估表面表达、转运、安全性并排序，结合历史项目经验，效率超过普通编码助手。
  * Allen Institute：搭建含20个定制技能的多代理“计算综述模板”，子代理读文献、建证据库、生成图表 ，还有“行动者-评论家”一个写内容，一个做评估的双保险，原本需要耗时2年的综述工作大幅提速，已产出多篇百页级成果。
  * UCSF脑肿瘤中心：胶质瘤分子流行病学分析时间降至原来的1/10，而且独立验证结果可靠。

也有人保持谨慎。东北大学的 Jared Auclair 认为，AI更像是“需要老练飞行员驾驶的副驾驶”。他担心，通用AI在解读监管指南、设计实验等需要精细判断的环节，仍可能出现幻觉或是遗漏细节——在药物研发里，这类错误可是要命的。

**把镜头拉远，在科研这条路上，AI三巨头的路线各有不同。**

  * **OpenAI** 推出的 GPT-Rosalind，走专有微调模型路线，限企业客户，还得过安全审查才能准入。
  * **Google** 手握 AlphaFold 这张王牌，Gemini for Science整合了30多个科学数据库。
  * **Anthropic**  则是走订阅开放和工作流优先，让尽可能多的人先用起来。

发布当天，Anthropic还宣布自己要启动药物研发计划，目标锁定“被忽视的疾病”，这类疾病长期被传统制药公司认为商业价值低，不值得投资。

同时，Anthropic还将支持多达 50 个的Claude Science AI for Science项目，提供最高3万美元的资助，Modal 还将为部分项目提供最高2000美元的计算资源支持。申请截止7月15日，7月31日就能知道结果。

目前Claude Science还在测试阶段，长期效果还得有更多真实项目验证，但这次信号已经足够让人对AI驱动科研的未来充满期待。

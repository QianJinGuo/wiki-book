# 刚刚，GPT-5.6全面上线，Codex被合并，生产力工具ChatGPT Work来了

机器之心编辑部

  


我们迎来了 OpenAI 的一大波发布。

  


在有限预览期之后，本周四 OpenAI 正式推出了 GPT-5.6 系列模型：包括新旗舰 Sol，适合日常工作的均衡模型 Terra，还有最快速的 Luna。

  


与此同时，ChatGPT 与 Codex 也正式合体，OpenAI 还推出了与 Claude Cowork、WorkBuddy 类似的智能体工具 ChatGPT Work。

  


  


GPT-5.6 已在 ChatGPT、Codex 和 OpenAI API 中上线，此次推送覆盖全球范围，将在 24 小时内逐步覆盖所有版本。

  


GPT-5.6 的定价分为三种：Sol 为每百万 token 5 美元输入 / 30 美元输出；Terra 为 2.50 美元输入 / 15 美元输出；Luna 为 1 美元输入 / 6 美元输出。

  


GPT-5.6 还引入了更可预测的提示缓存机制，包括对显式缓存断点的支持。缓存最短保留时间为 30 分钟。对于 GPT-5.6 及更高版本，缓存写入按模型未缓存输入速率的 1.25 倍计费，而缓存读取仍享受 90% 的缓存输入折扣。

  


GPT-5.6 系列模型

  


GPT-5.6 Sol 在智能和效率方面树立了新的标杆，在代码、知识工作、网络安全和科学等领域均取得了最先进的成果，同时以更少的 token 和更低的预估成本超越了同类前沿模型。OpenAI 还引入了一种加速最苛刻任务的新方法：新的最高能力设置是协调多个智能体在并行工作流中运行，从而更快地完成复杂任务。

  


更强大的计算机使用和设计判断能力使 GPT-5.6 Sol 成为人们迄今为止最成熟的协作伙伴，能够仔细检查、改进并交付可直接使用的成果。

  


GPT-5.6 提高了每 token 工作效率。在 Agents’ Last Exam 上在对 55 个领域上长期运行的专业工作流程进行评估后，GPT-5.6 Sol 创下 53.6 的新高，比 Claude Fable 5（自适应推理）高出 13.1 分。即使在中等推理水平下，它也比 Fable 5 高出 11.4 分，而成本约为其预估成本的四分之一。

  


这种效率也体现在更小的模型上，而这些模型对于实现更丰富、更经济的智能至关重要：GPT-5.6 Terra 和 GPT-5.6 Luna 的性能优于 Fable 5，而成本约为其十六分之一。

  


  


在 Artificial Analysis Intelligence Index 上，GPT-5.6 Sol 的推理能力涵盖了智能的广泛衡量标准，包括自主工作、编码、科学推理和一般能力。其推理能力最高时，得分仅比 Fable 5 低 1 分，完成任务的时间减少了 61%，成本也只有预估成本的一半左右。

  


  


GPT-5.6 Sol 是 OpenAI 迄今为止最好的编码模型。在 Artificial Analysis Coding Agent Index v1.1 上， GPT-5.6 Sol 结合最大推理能力取得了 80 分的全新最佳成绩，比 Fable 5 高出 2.8 分，同时输出 token 数量不到 Fable 5 的一半，运行时间不到一半，成本也降低了约三分之一。这一优势在整个模型家族中都有体现：Terra 的性能略高于 Fable 5，而 Luna 的性能优于 Opus 4.8；它们的运行时间分别约为 Fable 5 的三分之一，输出 token 数量约为其一半，成本约为其预估成本的四分之一。

  


此外，GPT-5.6 Sol 在 Terminal-Bench 2.1 和 DeepSWE 测试中也取得了新的最佳成绩，这两个测试旨在评估真实代码库中复杂的命令行工作流和长期工程。

  


  


GPT-5.6 可以编写和运行轻量级程序，协调各种工具、处理中间结果、监控进度，并在工作进行过程中选择下一步操作。这使得工具密集型任务能够以更少的 token 、更少的模型往返次数和更少的指导推进。无需开发者编写每个步骤的脚本或将每个工具响应都传递给模型，程序化工具调用即可实现这一点。响应 API 可以过滤大量中间数据，只保留重要信息，并在此过程中调整其工作流程。

  


对于那些需要投入更多时间和计算资源才能解决的问题，GPT-5.6 可以超越这种高效的默认设置。它赋予 GPT-5.6 更多的时间进行推理和探索替代方案、运行检查并修正其方法。更进一步，它默认并行协调四个智能体，以更高的 token 消耗换取更优的结果和更快的响应速度，尤其是在处理高要求任务时。

  


下图比较了 ultra 的默认四智能体配置与单智能体基准在 BrowseComp、SEC-Bench Pro 和 Terminal-Bench 2.1 上的表现。

  


  


BrowseComp 和 SEC-Bench Pro 还展示了 16 智能体配置。在所有三个评估中，添加并行智能体都会使得分 - 延迟曲线向上向左移动，从而在更短的时间内获得更优的结果。在 API 中，开发者可以使用 Responses API 中的多智能体 beta 版本构建类似 ultra 的体验。

  


GPT-5.6 在设计判断方面实现了质的飞跃。只需提供高层次的指令，GPT-5.6 就能创建美观、符合人体工学且功能强大的界面。它更强大的计算机使用能力使其能够检查和优化渲染结果 —— 而不仅仅是生成底层代码或内容 —— 从而发现视觉和功能上的问题，并在最终交付作品前进行润色。

  


GPT-5.6 的前端功能还可以将自然语言请求转换为 ChatGPT Work 中精美的交互式解释和可视化效果。

  


GPT-5.6 在专业任务中表现更佳。它能从你的文档和日常工作流程（例如 Slack、Notion、Microsoft 365 和 Google Drive）中提取杂乱的上下文信息，并将其转换为专家级、可共享的成果。

  


此外，GPT-5.6 在知识型工作方面的优势体现在涵盖长期专业分析、浏览、工具使用和计算机使用的各项评估中。GPT-5.6 Sol 在 BrowseComp 和 OSWorld 2.0 测试中分别取得了 92.2% 和 62.6% 的成绩；在 OSWorld 测试中，它超越了 Opus 4.8，同时输出 token 数量减少了 85%。GPT-5.6 系列的性价比优势也十分显著。Luna 的性能几乎达到了 GPT-5.5 的峰值水平，而成本却不到其一半；Terra 的性能则更胜一筹，且成本更低。

  


ChatGPT Work，ChatGPT 与 Codex 合并

  


OpenAI 新推出的 ChatGPT Work 是一款智能体工具，可以对你的应用程序和文件直接执行操作，如有需要，也可以长时间参与项目，并将目标端到端地转化为完成的工作。

与如今 AI 领域兴起的智能体工具类似，ChatGPT Work 可以从你的应用程序和工作流程中收集信息，创建表格、幻灯片、文档和 Web 应用程序等成品，并通过将复杂的项目分解成更小的步骤并独立完成，从而持续数小时完成项目。ChatGPT Work 由最新的 GPT-5.6 提供支持。

  


ChatGPT Work 面向 Pro、Enterprise 和 Edu 套餐用户推出，并将于未来几天内陆续上线 Plus 和 Business 套餐用户。ChatGPT 桌面应用则涵盖聊天、Work 和 Codex 三项功能，所有套餐（包括免费版）均可使用，并支持 Windows 和 Mac 系统，全球用户均可下载使用。

  


  


现在，ChatGPT 内置了 Codex 的能力，它不仅可以回答问题，还能在网页、移动设备和桌面端完成实际工作。OpenAI 表示，每周有超过 500 万人使用 Codex。虽然它最初是为开发者打造的代码助手，但现在已有超过 100 万人将其用于软件开发以外的工作，这表明 Codex 的功能可以支持更广泛的任务。

  


学习如何使用 ChatGPT Work 的最佳方法是交给它一项你已经熟悉的任务：例如分析月末预算差异、将原始资料转化为营销活动简报，或者准备销售会议。你可以跟踪它的进度、回答问题、调整方向并批准重要操作。

  


你甚至可以要求 ChatGPT Work 一次性完成整个工作流程。例如，它可以将客户调研转化为营销活动简报，利用该简报创建营销素材，并针对不同市场调整这些素材，同时在每个步骤中都保留必要的背景信息。

  


即使你不在电脑或手机旁，ChatGPT Work 也能通过定时任务确保项目顺利进行。例如，它可以自动将 Microsoft Teams 和 Slack 中的新消息转换为更新后的文档或幻灯片，然后与你的团队共享重要更改。

  


  


OpenAI 内部几乎 100% 的团队（包括财务和销售团队）现在都使用 ChatGPT Work 和 Codex 来加快工作速度、承担复杂任务，并花更多时间与客户互动。

  


为了带来更强大的体验，现在 ChatGPT 桌面应用程序的功能进一步扩展。在桌面端，ChatGPT 可以使用本地文件和应用程序来完成工作。对于基于 Web 的工作，ChatGPT 全新的内置浏览器可以导入网站、工具和在线文件，让你在一个平台上高效推进工作。

  


从今天起，Codex 应用将与 ChatGPT 桌面应用合并。Codex 仍然是面向开发者和技术专业人员的强大代码智能体，现在新增了多项功能。涵盖核心工作流程，包括差异中的内联编辑、侧面板中的拉取请求审查、更快的计算机使用速度（由 GPT-5.6 提供支持）以及对单个项目中多个存储库的支持。

  


  


各类插件可将 ChatGPT 连接到 Slack 和 Microsoft Teams、Google Drive 和 SharePoint、电子邮件、日历、CRM、项目跟踪器以及其他内部工具等应用和系统。ChatGPT 会根据你的提示自动识别并引用插件，你也可以通过在提示中输入 “@” 符号后跟应用名称来指示 ChatGPT 从特定应用中提取上下文信息。全新的统一插件目录将所有插件集中在一个位置，ChatGPT 可以在对话过程中推荐相关的插件。

  


  


OpenAI 在 ChatGPT 的公开测试版中推出了「Sites」功能。借助它，你可以将作品或想法转化为交互式网站或 Web 应用，并通过 URL 与团队或公众分享。你还可以在 ChatGPT 中直接测试你构建的 Sites，并将最新的 Web 内容引入到你的项目中。ChatGPT 还可以随着底层信息的更改自动更新「站点」。

  


计划任务允许你要求 ChatGPT 执行一次操作、按计划或在事件发生时重复执行该操作，或者监控一段时间内的变化。

  


计划任务可以使用你连接的应用和浏览器来执行以下操作：

  


  * 每周查看 Slack 上的最新更新，并更新定期会议议程。

  * 每天早上检查网站和仪表盘，总结变化情况，并发送报告。

  * 密切关注新客户的反馈，并将反复出现的主题转化为优先考虑的产品创意。

  * 当收到新的电子邮件反馈时，请更新演示文稿。

  


在桌面版中，ChatGPT 现在包含一个内置浏览器，可帮助你在一个地方收集在线信息、使用基于网络的工具并完善基于网络的工作。

  


你可以让 ChatGPT 进行市场调研、比较信息来源、从网站提取信息，或者在应用内打开并处理来自 Google Workspace 和 Microsoft 365 的文件。它可以利用浏览器获取新的上下文信息，跨网页浏览，并在你审核和指导结果的同时，持续推进工作。

  


在桌面端，ChatGPT 的智能体可以代替你使用电脑，在后台执行应用程序、工具和浏览器中的任务，例如点击、输入文字以及将文件移动到所需位置。你可以将其用于一次性任务，也可以将其作为计划任务的一部分，用于需要重复操作计算机步骤的工作。

  


OpenAI 也在更新 Chrome 扩展程序，使其可以直接在 Chrome 侧边栏中使用 ChatGPT。

  


在人们最担心的 token 消耗量方面，OpenAI 表示，ChatGPT Work 专为比普通聊天请求耗时更长、更复杂的工作而设计，因此使用方式有所不同。使用量取决于所需工作量，更复杂的任务可能会占用你套餐包含的更多使用量。ChatGPT Work 的使用结构与 Codex 相同。

  


放在 2026 年年中的时间节点来看，OpenAI 今天的这套组合发布能够带来不小的冲击。Codex 的「消亡」之后，是全民开发者时代，超级应用的出现正在重塑生产力。

  


或许 OpenAI 并不是在与 AI 公司竞争，而是要革 Microsoft 365 的命。

## AI 圈造词史(开篇)

**过去一年的造词时间线**:
- 提示词工程(Prompt Engineering)火了
- Harness 工程火了(给模型套马鞍,让它在框架内运作)
- 现在轮到 **Loop Engineering(循环工程)** —— 写提示词 / 写 Harness 都比不过**写好一个循环**

> **核心命题**: **写提示词 → 写 Harness → 写 Loop**,人从一次对话变成一个**完整回路**。

## 5 个核心问题(本来源提炼)

一个完整的 loop,至少要回答 5 个问题:
1. **AI 什么时候开始干活?**
2. **它能调用哪些工具?**
3. **它怎么知道自己做错了?**
4. **它把每一轮结果记在哪里?**
5. **它什么时候必须停下来交给人?**

> **结论**: Loop Engineering 更接近**一套工作制度** —— 给 AI 设任务/设工具/设反馈/设记忆/设刹车;**prompt 只是这套系统里最小的零件**。

**1 个记事本(状态文件)**:
- 可以是文档/表格/看板/项目规则
- 记录: 已确认信息 / 踩过的坑 / 偏好格式 / 禁用表达 / 上次没解决的问题
- AI 每次启动时**先读它**,就能接着往下走

> **关键能力**: AI 有记忆 = 不必每天从零开始(不必再查已确认事实 / 不必重做已否掉标题 / 不必引用不可靠来源)

## Token 成本现实(本来源独家深度分析)

**核心观点**: "token 成本其实把这个热词拉回了现实"

**两极分化**:
- **有无限额度的人**: 龙虾之父 / Claude Code 负责人 / Google Cloud AI 总监 → 循环是常识
- **月付 20 美元的用户**: 循环跑两天就达到了**周限额**

**循环会反复消耗 Token**:
- 反复读上下文
- 反复重试
- 四处探索
- 不管最后有没有产出,Token 都在燃烧

**Token 经济学结论**:
> Loop Engineering 不会让 AI 协作变得无成本,它只是把成本从**「人一轮轮盯着」的时间成本**,转移到**「系统一轮轮运行」Money 成本**。

**关键变化趋势**:
- 过去: 任务失败 → 人回来追问 / 补充上下文 / 重新解释需求
- 现在: AI 已能**自己尝试 / 自己验证 / 自己修正**,再把结果交回来
- 如果一次循环消耗的 Token ≈ 人来回追问几轮的成本,**真正值得优化的可能就不再是某一句提示词,而是整个反馈回路**

## 入门 Loop Engineering 的 4 个前提(本来源提炼)

1. **Token 管够**: 月付 20 美元跑不了循环
2. **任务每周重复**: 一次性活 = 一句好提示词(更快更便宜)
3. **自动验证**: 测试 / 类型检查 / 构建(确保 Agent 没人盯着时能拦烂活)
4. **Agent 有高级工程师素养**: 日志 / 复现环境 / 能跑自己写的代码

> 缺任一条,**循环的成本可能高过回报**。

## 反思:"新瓶装旧酒"?(本来源独家质疑视角)

> "这些术语就莫名其妙地出现在了社交媒体上,有人说不用管「loop 工程」是什么,它本质就是一种炒作和营销的手段。"

> "一个正在使用 Agent 的用户,自然而然就实现了所谓的 loop 工程,而不是专门有一套流程,符合那套标准,需要被单拎出来,才可以被叫做是 Loop Engineering。"

> "所以是新瓶装旧酒,一套过去的思维方法论,重新套上了新名词吗?"

**爱范儿的中间立场**(本来源独家):
- "Loop Engineering 最值得讨论的地方,未必是它算不算一个新学科"
- "更现实的问题是,当 AI 已经能连续处理多轮任务,**人还要不要继续把自己卡在每一次追问里**"
- "**有些工作,确实只需要一句好提示词**。有些工作,则需要一套能反复运行、能留下记录、能及时停手的流程"
- "这个分界线,在当下看来,可能比「循环工程」这个名字本身重要得多"

## 关键独到判断(本来源独家)

- **"新瓶装旧酒"质疑**: AI 圈造词大师,新词不断但本质不变 — 本来源的造词学反思
- **Token 成本两极分化**(本来源独家深度): 月付 20 美元 vs 无上限 → 循环经济是**有预算人的常识**
- **"loop 是不是新学科不重要,关键是分界线"**(本来源独家结论): 真正值得讨论的是**哪些工作适合循环 / 哪些只需要一句好提示词**
- **主流科技媒体视角**: 与工程视角 / 学术视角 / 批判视角都不同,是从**公众/产品用户**视角看 loop
- **Token 成本"时间成本→Money 成本"转移**(本来源独家洞察): 不变的是成本总量,变的是成本形式
- **"龙虾之父"梗的延续**: 本来源用龙虾之父(Boris Cherny 之外的另一位 KOL)的循环实践作为补充案例

## 实践启示(本来源补全)

- **Token 预算是入门 Loop Engineering 的第一前提**: 月付 20 美元套餐跑不了循环 — 先评估预算再选工作场景
- **任务每周重复**: 一次性活不需要循环,直接写提示词更快
- **3 条入门标准**: 自动验证 + Agent 高级工程师素养 + Token 管够 — 缺任一条不划算
- **跨场景扩展**: loop 不止编程 — 内容/客服/产品运营/研究都可,但每个场景的验证机制不同
- **"分界线"思维**(本来源独家): 不要被"循环工程"这个名词绑架,真正的问题是"哪些工作适合循环 / 哪些不需要"
- **AI 圈造词观察期**: 任何新概念,先等 6 个月看是否被淘汰 — 本来源"是不是新瓶装旧酒"的反思可推广到所有 AI 圈新概念

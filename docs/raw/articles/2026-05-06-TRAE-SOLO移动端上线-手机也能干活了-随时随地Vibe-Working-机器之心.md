在 Agent 愈发深度融入工作流之后，我们发现自己似乎越来越离不开电脑了。

当然，这指物理意义上的。

但假如，周五下班前，你在电脑上跑了一个需要持续很久的训练脚本。人已经离开办公室，电脑留在工位上。突然程序崩了怎么办？任务卡住了怎么办？而此时你手里只有一部手机……

为啥 OpenClaw 推出即大火，就是因为它在一定程度上缓解了这个问题。它在最初，就是作为 OpenClaw 之父 Peter Steinberger 通过 IM 软件来远程遥控电脑上的 Claude Code 而诞生的。

但即便用手机指挥电脑上的龙虾干活，却仍然放不下心来。不仅不能很实时地查看进程，更对龙虾的下一步操作没有预期，甚至遇到问题用 Ctrl+C 都打不断！于是逐渐演变成不管人在哪，都得揣上这台「住着 AI」 的电脑，这大概是独属于 AI 时代，大家最普遍的「数字焦虑」。

如果，我是说如果，有一种办法能把移动端、电脑端，甚至云端的智能体和工作流彻底打通，是不是意味着我们可以从这种焦虑里解放出来？

几天前，TRAE SOLO 悄悄上线了移动端，手机、桌面、网页从此 实现 三端打 通 。

更好的消息是，TRAE SOLO 三端至此实现全量开放，无论用户是 iOS 还是 Android、Mac 还是 Windows，下载即用。

移动端 ≠ 缩水版桌面端：手机揣兜里，一样能干活

TRAE SOLO 手机端和桌面端共享同一个 Agent、同一套文件系统、同一段对话上下文。 设备变了，但心流不会断。

对非开发者来说，路上突然想到一个 Campaign 点子，会议间隙冒出一个需求改动，出差途中意识到方案要调整，过去大多是先发给自己一条微信，等回到电脑前再重新整理。现在可以直接对手机说出来，让 TRAE SOLO 把零散想法直接转成 PRD 草稿、运营思路或待办任务，同步到 PC 工作台，无缝接入工作流。

对开发者来说，开发工作永不掉线。Bug 的解法、新 Feature 的拆法、某个 GitHub Issue 的处理思路，很多时候并不是坐在 IDE 前才想明白的。TRAE SOLO 的价值在于，它可以把这些移动场景里的闪念直接带回项目上下文：先记录，再整理成修改计划，立即接到 PC 端继续执行。

TRAE SOLO Mobile 做的，是让每一个闪念都拥有继续生长的机会。它把「想到」和「做到」之间原本漫长的搬运过程压缩到最短，让手机成为每一个人随身携带的工作台。

TRAE 已经实现了三端互联，当我们离开电脑的时候， 用手机控制电脑完成一个完整复杂的项目 ，自然也不在话下。

最近看到一个很可爱的小项目，一位日本开发者开发了一款浏览器插件程序，在一定时间后，屏幕上会显示一只肥猫，强迫你休息一下。

最后，一切都生成完毕后，我刚好抵达了工位，把猫猫安装进了 Chrome 浏览器。

实测之后，还有一些功能值得关注

这次 更新还带来了几个 重量级功能：

首先是 实时语音交互讨论 。用户可以直接用手机和 AI 讨论一个问题，比如一个产品想法、一场 Campaign、一份策略报告，或者一个代码问题。过去，我们打开手机更多是「先记下来，回头再说」；现在，直接开展讨论，Agent 可以直接生成纪要、沉淀思路，并进一步下发任务。

也就是说，手机变成了任务的起点。

我们把 SOLO 接进了我们的飞书工作流里。开放权限以后，SOLO 就像新入职的编辑，帮我把五一期间漏掉的重要内容很快的整理了出来。

TRAE SOLO 的能力在移动端丝毫不打折，并没有因为用户在移动端操作，就可以偷偷摸鱼省算力或者降低能力。

不管在哪个端口，它都是我们熟悉的强大的 TRAE SOLO ，更新代办列表，执行文件操作，代码编写与执行，一气呵成，产物管理清晰明了。

最后，TRAE 把五一假期期间有价值的 AI 圈新闻都列出来了，并且做好了日期、消息类别和重要性的标记。最让人惊讶的是，智能体还做到了在比较混乱的选题文档里，筛选信息后为每个事件提取了相当准确的标题。

第三是 定时任务 。用户可以提前设定任务，让 Solo 按固定时间或频率自动执行 Prompt，并产出结果。比如每天上午自动整理 Cursor、Claude Code 等产品的最新动态，再生成文档发给自己。这类功能让 Agent 成为一个长期主动在线的助手，持续完成信息收集、整理和汇报。

写在最后：梦中的闪念胶囊

当全能力的智能体进入移动端的时候，我就想到了这个：

很多想法不是坐在电脑前想出来的，而是在走路、洗澡、坐车、睡前，突然「啪」一下冒出来的。「闪念胶囊」就是用来把一闪而过的想法先存下来的工具，让想法在出现的第一秒，就被系统「稳稳接住」。

但问题是，「闪念胶囊」只做到了灵感的断点保存，却仍然跟不住工作流。

TRAE SOLO 更像是我们曾经梦中的「闪念胶囊」完全体。

过去，胶囊负责把念头保存下来；现在，Agent 开始接过下一棒，把念头补全、整理、拆解，并推向真正的工作流。

这是一种新理念的探索。AI 智能体的主场，到底在哪里？

过去两年，AI 编码领域的竞争主线很清晰：谁能更懂代码，谁能更快补全，谁能更准确地修 bug，谁就能抢下开发者的桌面。Cursor 借助与 VS Code 生态的深度融合，迅速成为新一代 AI IDE 的代表；GitHub Copilot 则凭借长期嵌入开发环境和企业软件体系，占据了稳定的入口。

包括已经非常成熟的 TRAE SOLO，打造了 AI 原生 IDE 与智能体工作台，完善了「以 AI 为主导、理解目标、承接上下文并调度工具」的自动化开发方式。

但无论形态如何变化，本质上仍然是桌面软件时代的产物。它们提升了生产效率，却没有真正改变「人的心流」。

TRAE SOLO Mobile 的出现，恰恰把这个边界往外推了一步。

我们知道，历史是个圈儿。

互联网 1.0 让信息上网，用户坐在电脑前浏览和搜索；互联网 2.0 让人上网，用户开始生产内容、参与互动；而移动互联网真正完成了场景革命，让服务不再被锁在电脑前，而是跟着人走。

今天，TRAE SOLO Mobile 正推动智能体走向一个全新的阶段。

移动互联网让互联网摆脱了书桌，智能体的移动互联，让工作流摆脱了工位。智能体本就应该像移动互联网时代的应用一样，跟着人走，在不同设备之间自然流转，在不同场景里接力完成任务，随人而动。

同样的，移动端也在降低智能体的使用门槛。当 AI 编程工具只存在于 IDE 里，它天然更像是开发者专属工具。

但当它可以通过手机操作、语音交互，并接入文档、网页和业务流程时，它服务的对象就不再只是工程师。产品经理可以用它拆解需求，运营可以用它处理数据，管理者可以用它追踪项目，测试和设计也可以把它纳入自己的工作流。这也正是 TRAE SOLO Mobile 更值得关注的地方。

回到最开始那个问题：AI 智能体的主场到底在哪里？

答案或许不再是 IDE，不再是浏览器，不再是任何一个载体。 真正的主场，是人的工作流本身 。

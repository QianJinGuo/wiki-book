# 25个Skills详解：从生产力清单到AI工作流资产

> 来源：架构师（若飞）| 2026-05-28 | 分析 Nico 整理的25个生产力 Skills

## 核心命题

Skill 不是提示词合集，而是轻量级 Runbook——可保存、可复查、可迭代的工作规程。它把隐性经验变成 Agent 可以读取和执行的工作流程。

## 三层视角

### 个人视角
Skill 把顺手的方法留下来。下次遇到同类任务，不必重新组织语言。

### 技术团队视角
Skill 把工程规范、上下文、检查点和失败经验，放进 Agent 每次工作的路径里。

### 管理层视角
Skill 更接近一种新的 SOP 形态。流程不再只写在文档里给人看，也能被 Agent 加载、执行、留下证据，并在团队里继续迭代。

**关键洞察**：工具只是入口。能留下复利的，是流程开始资产化。

## Skill 在 Agent Runtime 里的位置

```
模型层：理解、推理、生成
工具层：搜索、读写文件、调用 API、执行代码
上下文层：项目资料、记忆、知识库、历史记录
过程层：Skills、commands、Runbooks、检查清单  ← Skill 的位置
治理层：权限、审计、版本、回滚、人工确认
```

**Skill 与其他组件的区别**：
- 模型：负责推理和生成
- 工具：连接外部系统（文件、浏览器、代码执行、数据库、搜索、API）
- RAG/知识库：把相关资料找回来
- **Skill：告诉 Agent 这类任务按什么流程做**

## 六维评判标准

判断一个 Skill 写得是否扎实：

1. **触发条件**：description 是否说清什么时候该用
2. **输入边界**：缺材料时是追问、标缺失，还是胡乱补全
3. **执行步骤**：流程是否足够具体，而不是空泛口号
4. **资源依赖**：是否需要模板、脚本、参考文档或工具
5. **输出形态**：结果是否便于人审阅和二次使用
6. **停止标准**：什么时候停，哪些动作不做，怎么验收

**这六件事缺得太多，Skill 很容易退回成一段长 prompt。**

## 企业落地建议

### Skill 分类（5类）
- 学习知识
- 办公沟通
- 研究决策
- 内容表达
- 工程交付

### 落地节奏
不用一次装满 25 个。**先选 3 个高频、低风险、可审阅的场景，跑几轮再固化。**

### 企业级 Skill 库要求
- owner（负责人）
- 版本（versioning）
- 权限（permissions）
- 审计（audit）
- 下线机制（deprecation）

否则很容易变成新的知识库技术债。

## 25个Skills示例（部分）

### 工程类 Skill

**代码文档生成器**（code-documenter）
- 输入：源代码、语言和框架、文档格式要求
- 流程：阅读代码和调用上下文 → 识别功能/参数/返回值/副作用 → 生成文档块 → 补示例和边界
- 约束：测行为不测内部实现；示例要能运行

**单元测试生成器**（unit-test-generator）
- 输入：待测代码、测试框架、现有测试风格
- 流程：分析公开行为 → 识别输入/输出/异常 → 生成正常/边界/错误测试
- 约束：每个测试独立；不依赖执行顺序

**调试助手**（debug-assistant）
- 输入：错误信息、堆栈、相关代码、运行环境
- 流程：解释错误 → 定位最可能出错位置 → 分析变量/类型/执行流 → 给出最小修复
- 输出：现象 + 根因 + 最小修复 + 预防建议

### 研究决策类 Skill

**深度研究合成器**（deep-research-synthesizer）
- 输入：多个来源材料、研究问题、输出目标
- 流程：按来源类型分类 → 提取事实和观点 → 找出共识/分歧/空白 → 合成洞察
- 约束：区分事实/推测/作者判断；不把单一来源写成行业共识

**竞争情报分析器**（competitive-intelligence-skill）
- 输入：竞品列表、分析目标、资料来源、关注维度
- 输出：对比表 + 每个竞品摘要 + 优势/弱点 + 机会点
- 约束：不把宣传口径当真实能力；区分事实和推测

### 治理关键问题

| 管理问题 | Skill 视角 |
|---------|-----------|
| 经验能不能复制 | 有没有把高频流程写成可复用 Skill |
| 输出能不能稳定 | 有没有固定输入、输出、验收标准 |
| 风险能不能收住 | 有没有权限、禁止动作、人工确认 |
| 新人能不能更快上手 | 有没有项目级 Skills 和示例 |
| 组织能不能持续学习 | 有没有复盘、版本、下线和更新机制 |

# 一个小模型意图识别系统的 5 天进化实录｜支付宝618

> 阿里技术 | 胡嘉乐
> 2026年7月13日 20:17 浙江
> 支付宝 CY26-618 大促

## 业务背景

支付宝 CY26-618 电商大促的 AI 找优惠项目，分两期：
- **一期**：以 Agent 为本，猜你喜欢式智能推荐，从 20+ 权益池自主筛选
- **二期**：以人为本，对话式交互，用户输入自然语言后精准匹配权益/商品

核心链路：用户输入 → 意图识别（翻译层）→ 下游权益/商品匹配

## 意图识别设计

四种意图类型：
- **ITEM**：用户明确要买某类商品
- **BENEFIT**：用户寻找优惠券/红包/权益
- **VAGUE**：模糊意图，需追问澄清（L2-L4 优先级）
- **IRRELEVANT**：闲聊无关，不推荐

## 模型选型方法论（4 小时）

### 五条约束
快（首 token 1-2s）、省（低成本）、准（四种意图区分）、记得住（32K 上下文底线）、别越界（只做翻译不做推荐）

### 外部调研
快速联系三个类似场景团队：分别使用 Qwen3-8B / 14B / 30B，候选收敛到 Qwen3 系列。

### 候选筛选逻辑
从 7 个模型（含多模态/嵌入）→ 排除多模态模型（输入不匹配 + 推理开销浪费）→ 排除嵌入模型（不能生成追问 + 语义理解深度不足 + 维护成本高）→ 最终 4 个 Qwen3 纯文本候选

### 黄金用例集
1. 用 GLM-5.1 按业务规则定向构造（覆盖边界 + 贴近真实 + 可标注性强）
2. 人工逐条审查修正（如"电脑配件有红包可以领吗"语义是 BENEFIT，但无对应权益 → 改为 ITEM 走商品推荐）
3. 锁定 52 条黄金用例

### 四维评测结果

| 模型 | 加权得分 | TTFT | 成本 |
|------|---------|------|------|
| Qwen3-8B | 80.8 | 0.72s | 最低 |
| Qwen3-14B | 84.0 | 1.04s | 中等 |
| Qwen3-30B-A3B | 80.0 | 0.82s | 中等 |
| Qwen3-30B-A3B-SFC | 82.0 | 0.81s | 中等 |

最终线上使用 **Qwen3-8B**（14B 受资源限制交付有风险，8B 是最务实的选择）。

## 系统提示词自进化框架

### 三次迭代

1. **自动化闭环**：手动调 prompt 无反馈回路 → 搭建自动「跑评测→打分→收敛判断→改写」框架
2. **train/test split**：训练集驱动改写，测试集验证泛化。收敛条件：train ≥ 90 且 test ≥ 92
3. **反过拟合多层防线**：
   - 案例服务于规则，严禁堆砌 few-shot
   - 每轮最多新增 1 条示意性案例
   - 两端共现→真实规则缺口，仅测试集→泛化不足，仅训练集→噪声不处理
   - 已达标维度原样不动

### 双轨评分体系
- **规则打分**：intent_accuracy 字符串比对 + format_compliance JSON 校验（毫秒级）
- **LLM 打分**（GLM-5.1/DeepSeek-V4-Pro）：keyword_extraction / thinking / clarity / relevance（2-3 分钟）
- 权重：intent_accuracy 30% + keyword_extraction 30% + format 15% + thinking 10% + clarity 10% + relevance 5%

## 实验结果

用户互动深度与转化效率正相关。自定义对话组（最深交互）的商品点击率为普通版本 **3.7 倍**，人均价值贡献 **2.1 倍**。不建议单独依赖自动首推。

## 工程心得

- 重复 3 遍就要造轮子（AI 时代造轮子成本极低）
- 频繁切换上下文但值得做（从执行者变成调度者）
- 保持热情 + 动手能力是 AI 时代最宝贵的技能

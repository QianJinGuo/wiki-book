# 刚刚，百度搭子全面升级，要让Agent接住人类90%以上的活

编辑｜杜伟  


Agent 发展到现在，不管是圈内人士，还是圈外看客，想必都试过让它帮自己干活：写一份报告，做几页 PPT，查一轮竞品动态，整理一份会议纪要，等等。

  


第一次上手，很多人会觉得惊艳。但一旦放进每天的工作流里，很快会暴露出问题：它能回答问题，却常常接不住完整任务；它能生成内容，却未必能稳定地跨系统推进流程；它能帮个人省下时间，却不一定能把团队的效率提上去。

  


预设能力与现实需求之间的落差，把 Agent 赛道推向了新的拐点。过去用户愿意为一次足够漂亮的演示买单。如今，评判标准变了。

  


大家更关心它能不能把一件工作从头做到尾，以及数据和成果能不能在组织内部安全流转。更重要的，执行过的任务不应只是一次性消耗，还应沉淀为下一次提效的经验。

  


也正是在这样的背景下，百度搭子在 7 月 10 日的成都 AIDAY 上，发布了它们自面世三个多月以来的最重磅升级。它们在三大板块同时发力：个人版全面升级、企业版亮相和搭子联盟启动。

  


按百度集团执行副总裁、百度智能云事业群总裁沈抖的说法，未来 90% 的工作，都可能有智能体深度参与、协助完成，百度搭子要做的就是深入办公场景真干活、真办事儿。

  


沈抖

  


发布会现场，百度搭子直接下单 600 支梦龙冰淇淋，送给到场的观众。

  


  


  


从 3 月底上市到现在，百度搭子以几乎日均一版更新的速度高速迭代。截止到现在，用户的日均提问次数暴增 20 倍，在办公、创作、信息处理等场景提供全方位智能辅助，成为增速最快的通用智能体。

  


而在此次 AIDAY 上，百度搭子做的这三件事，放在一起看，指向了同一个目标：加速 Agent 从炫技工具走向真正的生产力系统。

  


个人高频入口，

要靠更准、更稳、更全、更深

  


在个人版升级上，百度搭子重点提升了复杂办公任务的执行效率、跨端连续性和成果交付质量。

  


这背后的逻辑很直接：Agent 想成为高频工作入口，首先要让用户放心把复杂任务交给它。这就绕不开几件事：既要判断更准、执行更稳，也要具备更丰富的技能和更深入的场景覆盖。只有这些环节真正咬合起来，用户才会从偶尔尝试，逐渐形成稳定的使用习惯。

  


顺着这条逻辑，百度搭子个人版的升级可以形象地概括为四个维度：脑、手、技能和套件。

  


第一层是「大脑升级」。处理复杂任务时，Agent 需要先判断任务适合在哪个环境里完成。算力需求、响应速度和数据安全要求不同，执行路径也会不同。路线选得不合适，后续步骤就容易变慢，甚至增加额外消耗。

  


此次百度搭子通过 Harness 引擎升级引入了环境路由机制。系统会根据任务需求，在云端沙箱、本地沙箱以及 Chat 模式之间自动选择合适的执行环境。需要更高算力时，调用云端资源；强调低延迟或数据安全时，则切换到更合适的本地环境。这套机制让任务平均耗时降低 20%，Token 利用率提升 25%。用户等待时间更短，消耗更低，复杂任务推进也更顺。

  


我们让百度搭子查询前几天放出的 ICML 2026 各奖项论文信息，它直接调用 Web Search，很快整理出了结果。

同时，多端共享记忆补上了跨设备使用时的痛点。用户在电脑上做到一半的方案、报告或资料整理任务，切换到手机后可以继续追问、修改，也可以直接下载结果。前面已经交代过的偏好、模板和表达口径，也不用重新说明。对个人用户来说，这种连续性很重要。任务不会因为设备切换而中断，百度搭子也更接近一个能随时接上工作的搭档。

  


  


第二层是手，即工具升级。进入工作流后，判断准确只是第一步，浏览器里的操作还得足够稳定。资料检索、竞品调研和网页信息整理，往往需要连续完成页面进入、点击跳转、搜索筛选和结果汇总，任何一步卡顿，都可能拖慢整项任务。

  


百度搭子分别优化了移动端和桌面端的网页操作体验：移动端上线云端浏览器，通过 VNC 远程操控网页；桌面端内嵌浏览器也即将上线，可以直接衔接本地工作流。升级后，网页操作延迟下降 13%，流畅度提升 20%。

  


  


第三步是技能生态升级。接手复杂工作时，一套开箱即用的能力体系能够实现事半功倍的效果。百度搭子此前已经内置了百度搜索、百度百科、秒哒、伐谋等能力，此次进一步引入一镜、灵医等百度生态 Skill 以及美团红包 & 跑腿等第三方服务，能够处理的任务边界变得更大。

  


  


我们尝试调用「百度一镜」技能，生成了一段颐和园宣传数字人口播的视频。从输入需求到成片交付，整个过程仅用了约 3 分钟。

总的来看，「大脑、手、技能与专业套件」四层升级，分别补上了百度搭子的判断、执行、能力和场景深度，由此更有机会从偶尔使用走向稳定高频。想要体验的小伙伴，现在可以去百度搭子网页端和移动 App 免费下载使用了。

  


从一个人的搭子，

到一群人、整个行业的 AI 基础设施

  


相较于个人用户，企业用户面临的问题会更复杂。个人关心效率，组织还看重协作、权限、合规以及资产沉淀。百度搭子的第二个关键变化指向了这一层：从个人搭子进入企业流程，成为组织级的基础设施。

  


企业版的正式亮相，瞄准了企业面对智能体绕不开的心结：能用，但不敢用。在 2026 中关村论坛上，百度智能云副总裁殷大伟用这六个字概括了大多数企业的真实心态。能力摆在眼前，却总差一点把关键业务交出去的底气。

  


  


百度搭子企业版要做的，就是把这点底气一层层补上。第一件事是把知识和经验留在组织内部。企业用 AI，最怕任务做完了，成果、方法和上下文也随之散掉。

  


企业版通过三步完成资产沉淀：生成的素材、方案等交付物统一归档，方便团队查找和二次复用；优秀员工的方法论可以沉淀为可复用技能，即使人员流动，能力仍留在组织中；企业文档、项目资料和行业素材则进一步汇入专属知识库，为后续任务提供持续可调用的组织记忆。

  


  


然后是共享与协作，资源、技能和成果在组织内部流动起来，个人提效才有机会进一步放大为团队提效。其中积分共享解决资源错配，系统会根据成员使用的频率智能分配企业统一采购的积分，谁常用就往谁那里倾斜；其次技能共享解决重复配置问题，一个成员沉淀下来的技能可以一键共享给组织其他人，不用重复造轮子。最后成果共享解决交付物分散问题，AI 产出的文档、报告和数据都统一归档到项目文件夹里。

  


  


在解决资产沉淀与流动之后，企业版又向前走了一步，探索如何更高效地将智能体接进已有系统。

  


一个很现实的问题是，企业的业务流程往往分散在不同系统里，比如 OA、CRM、ERP、数据平台和第三方 SaaS，接口和权限要求各有不同。过去企业接入 AI 能力，大多是一个系统或服务单独对接，流程繁琐重复且后期维护成本高。百度搭子发布行业首个企业级 Skill 接入标准，就是要解决这道连接门槛。

  


这套标准相当于在企业自有系统、第三方 SaaS 服务与百度搭子技能体系之间搭了一座桥梁，将内外部能力高效、安全地整合成可复用的 Skill。一方面降低了企业接入 Agent 的工程成本，另一方面划出了更清晰的安全边界和维护方式。

  


  


流程接进来之后，企业还要对 AI 访问、调用了什么了如指掌，并关心出了问题能不能溯源。百度搭子从上线之初便强调本地安全沙箱、文件夹权限管控和风险操作二次确认，企业版在此基础上继续加厚安全体系。

  


一是强化企业管控，包括组织架构管理、成员角色授权和用量统计；二是符合企业内部审计和外部合规要求，记录 AI 的执行动作和数据访问，以便后续追溯；三是 SLA（服务级别协议）保障，解决 AI 在高峰期等特殊节点的稳定性；四是 Agent 安全护栏和 Skill 安全机制，自动拦截高风险操作或者进行触发确认。

  


到这里，百度搭子企业版跨过了企业用 AI 所担心的那几道坎，也就有底气说自己是一套值得托付的基础设施。不过，Agent 能力想真正铺进千行百业，还需要更多的行业知识、客户触达和最后一公里服务来补齐。

  


搭子联盟的启动，接上的正是这一环。它把百度搭子的产品能力开放给更多企业伙伴，并进入更多真实业务现场。百度搭子自身也就置身于一个更大的 Agent 生态中。

  


在这个联盟里，企业伙伴负责行业深潜，带来行业 know how、客户资源和落地服务；百度搭子则负责能力底座，提供通用 AI 能力和标准化技能平台。目前，搭子联盟覆盖了东区、北区、南区以及 SaaS 和运营商赛道共计 11 家企业伙伴。

  


  


不到四个月，

百度搭子走到一个新节点

  


回过头来，从 3 月 22 日全量上线以来，百度搭子才走过了不到四个月的时间。但它的进步是「肉眼可见的」：

  


过去几个月，百度搭子几乎是贴着日历在迭代。此外，移动端从 Create 大会率先支持安卓到上线 iOS 端，也只用了一个月。

  


5 月 8 日当天，百度搭子同时登顶两个 Agent 评测榜单，在一众龙虾产品中非常能打。

  


在 PinchBench 中，它以 93.3% 和 93.2% 的总成绩包揽前两名，并在前 5 名中占据了 3  席。该基准考察 Agent 在 23 个真实工作场景、147 个任务中的多步推理、工具调用和任务闭环能力。

  


在另一个面向深度研究型 Agent 的 DeepResearch Bench 中，它同样以 58.03 的综合分位列榜首。该基准重点评估 Agent 的信息检索、内容准确性、分析深度和报告可读性等能力。

  


  


同样是在 5 月 8 日公布的 AI 产品榜中，百度搭子以 114.72% 的月访问量增速登顶 AI 龙虾增速榜，并以 116 万月访问量跻身 AI 龙虾总榜第三。

  


  


正是这样的高频迭代、能力进化与用户青睐，才把百度搭子推到今天这个节点。也是从此时开始，它第一次把个人使用、团队协作、行业落地摆上了同一条线。

  


这条线上串着的三个重头戏，彼此之间是接续的关系：个人版把入口的能力密度做厚，企业版把入口的场景深度做深，搭子联盟把入口的生态广度做宽。

  


更大的目标：

接住人类 90% 以上的活

  


从更长的周期看，Agent 赛道最终比拼的，可能还是谁能接住人类工作中更大比例的执行环节。

  


这个比例现在有多大呢？去年 11 月，麦肯锡全球研究院发布了一份名为《Agents, robots, and us: Skill partnerships in the age of AI》的调研报告 [1]，里面提到在现有工作时长中（美国），约 57% 对应的活动在理论上具备自动化潜力。

  


如今主流的 Agent 产品大都率先扎进了办公与生产场景，这是因为任务本身高度数字化，过去要靠脑力与手工完成的环节，在技术上已经具备被机器接手的条件。在这两大场景中，玩家们开始全方位比拼基座模型能力以及相应的产品体验、工具调用、成本效率和结果交付。

  


沈抖介绍，百度搭子的定位，是一个能深入办公场景、真干活、真办事儿的生产力工具。用户只要把目标说清楚，百度搭子就能理解需求、拆解任务、调用工具，完成跨应用、跨文件的复杂工作，最后交付真正可用的结果。

  


问题来了：我们该用什么标尺来衡量这些指标呢？

  


百度创始人李彦宏此前在 Create 2026 上首次提出了智能体时代的新度量衡 DAA（日活智能体数）。相比 Token 消耗，DAA 更关心有多少 Agent 真正在给人类干活、又交付了多少结果。

  


顺着这把新标尺往下看，百度搭子的路线也就清楚了：个人版升级让它先成为一个更能干的个人 AI 伙伴，企业版让个人使用沉淀为组织能力，搭子联盟让这套能力沿着伙伴网络触达更多行业现场。三步连起来，瞄准了同一个问题：Agent 究竟能在多大范围内辅助人类完成工作？

  


在 AIDAY 当天现场，沈抖还分享了一个他在百度内部观察到的一个变化 —— 通过接入百度各种内部系统，百度搭子不只是员工用来干自己手头的事，也成为了团队协作的高频工具。「百度搭子可以把项目成果、上下文共享给同事，沟通成本大大降低，好的经验和成果还可以沉淀成 Skill 供团队复用。员工效率高了、内耗少了，就能把更多精力用在更有创造力的工作上。」

  


百度搭子现在以及将来要做的，就是让 Agent 从少数任务的帮手，一步步成长为能够辅助人类完成 90% 以上工作的生产力伙伴。

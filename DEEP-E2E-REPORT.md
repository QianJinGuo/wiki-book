## 二、发现的问题

### 问题 1: Entity 引用缺失 (3 篇)

这些是文章正文中提到的 entity 名，不是 book_compiler 的主引用：

| 文章 | 缺失 Entity | 性质 |
|------|-------------|------|
| ch02-016 (LLM Wiki 架构) | `entities/、raw/、index.md、log.md` | 正文中的目录结构文字，非真实引用 |
| ch05-027 (Harness 知识护城河) | `entities/BK-AD-E001.md` | 正文中提到的文档编号，entity 不存在 |
| ch20-014 (Dario Amodei) | `entities/anthropic-responsible-scaling-policy.md` | 正文中提到的 Anthropic 政策文档，entity 不存在 |

**严重程度**: 低。这些是文章内容中的文字引用，不影响页面渲染和导航。

### 问题 2: 原始 HTML 残留 (2 篇)

| 文章 | 问题 |
|------|------|
| ch04-446 (Page Not Found - Microsoft Copilot) | 正文包含 "Toggle navigation" 等网站导航残留 |
| ch12-009 (Mythos curl vulnerability) | 正文包含 "Skip to content" 等网站导航残留 |

**严重程度**: 中。这些是抓取原文时未清理干净的 HTML 残留，影响阅读体验。

### 问题 3: 重复 h1 标题 (345 篇)

345 篇文章有 >2 个 h1 标题。这是 book_compiler 的格式问题：
- 第 1 个 h1: 文章标题
- 第 2 个 h1: 原文中的标题（未降级为 h2）

其中 21 篇有 >3 个 h1，最严重的有 7 个 h1。

**严重程度**: 中。影响 SEO 和文档结构语义。

### 问题 4: 模板化内容 (部分文章)

部分文章的"深度分析"和"实践启示"部分使用了模板化文字：
- "本文在agent方向提出的设计理念与实现路径"
- "实际落地中面临的关键问题与应对策略"
- "agent领域方案需关注可观测性、可维护性和成本效率"

**严重程度**: 中。这些是 LLM 自动生成的通用摘要，缺乏具体分析。

### 问题 5: Footer Nav 显示

所有 4,132 篇文章的 HTML 中都**没有** `md-footer__link` class（MkDocs Material 的内置 prev/next）。文章页的 prev/next 导航是由 `prev-next.js` 在客户端动态生成的。

**严重程度**: 无（设计如此）。

## 四、修复建议

### 优先级 P1 (影响阅读体验)

1. **清理 2 篇 HTML 残留文章**
   - ch04-446: 删除 "Toggle navigation" 等残留
   - ch12-009: 删除 "Skip to content" 等残留
   - 修复方式: 编辑 `~/wiki/raw/articles/` 中的源文件，重新 book_compile

### 优先级 P2 (影响内容质量)

2. **修复 345 篇重复 h1**
   - book_compiler 应将原文中的 h1 降级为 h2
   - 修复方式: 修改 book_compiler 的 markdown 转换逻辑

3. **清理模板化内容**
   - 对使用模板文字的文章重新生成深度分析
   - 修复方式: 人工审核 + 重新 LLM 生成

### 优先级 P3 (低影响)

4. **创建缺失的 entity 文件**
   - `anthropic-responsible-scaling-policy.md`
   - `BK-AD-E001.md` (或标记为内部文档编号)
